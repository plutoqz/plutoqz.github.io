# 第 1 题：无限循环

---

## 一、题意理解

给定长度为 (n) 的数组

$$(a={a_1,a_2,\dots,a_n})$$

将这个数组完整地复制并拼接到末尾，总共再拼接 (10^9) 次。于是得到新数组 (a')，长度是：

$$[ n\times (10^9+1) ]$$

并且本质上就是原数组重复了 (10^9+1) 轮。

要求：求这个超长数组 (a') 的**最长严格递增子序列**（LIS）的长度。

---

## 二、核心观察

注意这里是**子序列**，不是子数组；也就是说可以跳着选，但选出的值必须严格递增。

而数组是同一个序列重复很多次。

---

## 三、关键结论

设原数组中不同数字去重后，从小到大有若干个值。

对于某个值 (x)，因为要求严格递增，所以在最终 LIS 中，**同一个值最多只能取一次**。

因此，重复拼接不会让“同一个值”被多次使用，因为相等不满足严格递增。

那么问题的关键变成：

> 不同轮次中的元素，能不能帮助我们按更优顺序取到更多“不同值”？

答案是：**最多只能取到原数组中能形成循环上升链的那些值，各值至多一次，所以最终答案不会随着 (10^9) 线性增长。**

进一步分析后会发现：

### 结论

超长重复数组的 LIS 长度，等于把原数组看成一个**环**后，能够按环上的顺序选出的最长严格递增子序列长度。

也就是：

-   你可以从某一轮的某个位置开始；
    
-   继续往后扫，扫到末尾后还可以进入下一轮；
    
-   但每个值只能选一次；
    
-   本质上相当于在“原数组复制两遍”的长度 (2n) 数组中，找一个**跨度不超过 (n)** 的最长严格递增子序列（按位置环形展开）。
    

不过这还需要高效做。

---

## 四、更直接的正确建模

设数组重复无限次。我们想选一个严格递增子序列：

$$[ x_1 < x_2 < \dots < x_k ]$$

对于每个值 (x\_j)，必须在某一轮中找到它的位置，且这些位置在整个展开数组中的全局位置递增。

因为严格递增，所以同值只能出现一次。

那么只关心每个值在原数组中的出现位置集合。对于一串递增值

$$(v_1<v_2<\cdots<v_k)$$

，是否能按某种轮次安排依次取到？

这等价于：

-   从小到大枚举这些值；
    
-   每次尽量在当前轮找一个位置严格大于上一个位置；
    
-   如果找不到，就跳到下一轮并取该值在原数组里的最早出现位置。
    

由于总轮数有 (10^9+1)，而值的种类最多 (n)，通常足够；只要某串值能在若干轮内安排完即可。

但是这里因为拼接轮数极大，而不同值数量最多 (n)，所以轮数不是限制。<br>所以我们只需要判断：**有哪些不同值可以按从小到大的顺序依次“接上”**。<br>因为不管跳多少轮，只要总共不超过值种数，肯定远小于 (10^9+1)。

于是得到一个非常重要的结论：

---

## 五、终极结论

由于轮数极多，而严格递增又使每个值最多取一次，

> 最终答案其实就是原数组中**不同元素的个数**。

### 为什么？

设不同元素排序后为：

$$[ b_1 < b_2 < \cdots < b_m ]$$

我们要证明这 (m) 个值都能在重复数组中依次取到。

对每个 (b\_i)，在原数组中它至少出现一次。<br>我们从 (b\_1) 开始取，记当前在某一轮中的位置为 pos。<br>对于下一个 (b\_{i+1})：

-   如果当前轮中存在位置 (>pos) 的 (b\_{i+1})，就取它；
    
-   否则就跳到下一轮，取 (b\_{i+1}) 在原数组中的最早出现位置。
    

因为轮数几乎无限，且一共只需处理

$$(m\le n)$$

个不同值，所以一定能完成。

因此所有不同值都能选到，形成严格递增子序列，长度为 (m)。

同时因为严格递增，不同值至多选一次，所以 LIS 长度不可能超过不同值个数 (m)。

故答案就是：

$$\boxed{\text{数组中不同元素的个数}}$$

---

## 六、算法

对每组数据：

-   读入 (n)
    
-   读入数组
    
-   用哈希集合统计不同元素个数
    
-   输出集合大小
    

---

## 七、复杂度

设数组长度为 (n)。

-   时间复杂度：(O(n)) 或 (O(n\\log n))（取决于 set / unordered\_set）
    
-   空间复杂度：(O(n))
    

---

## 八、C++ 代码


# 第 2 题：最少相邻交换次数使括号串平衡

---

## 一、题意理解

给定一个长度为偶数的括号串 (s)，只含 `'('` 和 `')'`。

你可以进行若干次操作：

-   选择一个位置 (i)（1≤ i<n）
    
-   交换相邻字符 (s\_i) 和 (s\_{i+1})
    

要求最少多少次相邻交换，才能把整个串变成一个**平衡括号序列**。

并保证一定可以通过相邻交换变成平衡括号串。

---

## 二、平衡括号串判定基础

一个括号串是平衡的，等价于：

1.  `'('` 与 `')'` 数量相同；
    
2.  从左到右任意前缀中，`'('` 数量不少于 `')'` 数量。
    

设把 `'('` 记作 (+1)，`')'` 记作 (-1)，则任意前缀和都不能为负，最后总和为 0。

---

## 三、最少相邻交换的经典思路

### 方法核心：

当扫描到某个位置时，如果前缀已经“失衡”了（即右括号过多），那么必须把后面最近的一个 `'('` 交换过来修复。

并且：

-   选择**最近的** `'('` 才是最优；
    
-   因为相邻交换的代价就是它跨越的距离。
    

---

## 四、贪心过程

先记录所有 `'('` 出现的位置，设为数组 `pos`。

然后从左到右扫字符串，用变量：

-   `balance` 表示当前前缀中 `'(' - ')'`
    
-   `p` 表示下一个尚未被“用来修复”的 `'('` 在 `pos` 中的下标
    

扫描字符 `s[i]`：

-   若 `s[i]=='('`，则 `balance++`，并且 `p++`
    
-   若 `s[i]==')'`，则 `balance--`
    

如果此时 `balance < 0`，说明当前位置前缀中 `')'` 太多，需要从后面拉一个 `'('` 到当前位置。

那么最近可用的 `'('` 就在 `pos[p]`：

-   交换次数增加 `pos[p] - i`
    
-   实际上把那个 `'('` 移到 `i`
    
-   修复后当前前缀平衡值应变为 1（因为原本是 -1，换过来后相当于当前位置变 `'('`）
    
-   然后 `p++`
    

这个贪心是经典正确做法。

---

## 五、为什么正确

当某个前缀第一次出现负值时，这意味着当前位置必须放 `'('`，否则任何最终平衡串都不可能满足此前缀合法。

因此必须从后面的某个 `'('` 拉过来。

为了最少相邻交换，显然应该选最近的那个 `'('`，代价最小。<br>而且这样的局部最优不会影响全局最优，因为你若选更远的 `'('`，只会增加当前代价，也不会给后面带来任何优势。

所以贪心正确。

---

## 六、示例说明

例如：

$$[ s = "))(( " ]$$

位置从 0 开始：`0 1 2 3`

`'('` 的位置是 `[2, 3]`

扫描：

-   i=0, `)`，balance = -1，失衡<br>拉最近的 `'('`（位置 2）到 0<br>代价 += 2<br>balance = 1
    
-   i=1, `)`，balance = 0
    
-   i=2, 原本被处理后等价继续看，最终下一次失衡时再修
    

最终答案为 2。

---

## 七、复杂度

每个字符只扫描一次：

-   时间复杂度：(O(n))
    
-   空间复杂度：(O(n))
    

满足

$$\sum n \le 2\times 10^5$$

---

## 八、C++ 代码

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--) {
        int n;
        string s;
        cin >> n >> s;

        vector<int> pos;
        for (int i = 0; i < n; i++) {
            if (s[i] == '(') pos.push_back(i);
        }

        long long ans = 0;
        int p = 0;          // 下一个可用 '(' 在 pos 中的位置
        int balance = 0;    // 当前前缀平衡值

        for (int i = 0; i < n; i++) {
            if (s[i] == '(') {
                balance++;
                p++;
            } else {
                balance--;
            }

            if (balance < 0) {
                ans += pos[p] - i;
                p++;
                balance = 1;
            }
        }

        cout << ans << '\n';
    }

    return 0;
}
```

# 第 3 题：树上路径二进制串翻转与查询\*

---

## 一、题意理解

有一棵 (n) 个点的树，节点编号

$$1\sim n$$

<br>每个点的值是 `0` 或 `1`。

定义：

$$u \to v$$

表示树上从 (u) 到 (v) 的简单路径；

$$g(u\to v)$$

表示路径上所有节点值按路径顺序组成的 01 串，对应的二进制数对 (10^9+7) 取模的结果。

例如路径串是 `01101`，其二进制值是 13。

有 (m) 次操作：

1.  把路径
    
    $$u \to v$$
    
    上所有节点值翻转（0 变 1，1 变 0）
    
2.  询问
    

$$g(u\to v)$$

要求对每个询问输出答案。

---

## 二、这题本质是什么

这是一个典型的：

-   **树上路径修改**
    
-   **树上路径查询**
    
-   查询信息与路径方向有关（因为二进制拼接有左右顺序）
    

所以非常适合使用：

$$\boxed{\text{树链剖分（HLD） + 线段树}}$$

关键难点在于：

1.  路径查询有方向；
    
2.  线段树节点要维护“从左到右”的二进制值；
    
3.  翻转操作如何快速维护。
    

---

---

# 三、维护什么信息

对于线段树中的一个区间，假设其长度为 `len`，对应一段二进制串。

例如串为 `b1 b2 ... bk`，其二进制值为：

$$[ val = b_1 2^{k-1} + b_2 2^{k-2} + \cdots + b_k ]$$

我们需要支持：

1.  区间翻转：所有 bit 取反
    
2.  区间查询：既要能得到左到右的值，也要能得到右到左的值<br>因为树上路径拆成多段后，可能有的段需要反向拼接。
    

所以每个线段树节点维护：

-   `len`：区间长度
    
-   `fwd`：该区间从左到右构成的二进制值
    
-   `bwd`：该区间从右到左构成的二进制值
    
-   `lazy`：是否整段翻转
    

---

## 四、区间合并公式

设左儿子为 `L`，右儿子为 `R`。

### 1）正向值 `fwd`

左区间在前，右区间在后：

$$[ fwd = L.fwd \times 2^{R.len} + R.fwd ]$$

### 2）反向值 `bwd`

如果整个区间反过来看，则先是右区间的反向，再接左区间的反向：

$$[ bwd = R.bwd \times 2^{L.len} + L.bwd ]$$

---

## 五、翻转操作怎么更新

若一个长度为 `len` 的二进制串为 `x`，翻转后每一位都变成 `1-bit`。

设全 1 的长度 `len` 串对应的值为：

$$[ ones[len] = 2^{len}-1 ]$$

那么翻转后的正向值：

$$[ fwd' = ones[len] - fwd ]$$

因为每一位从 (b\_i) 变为 (1-b\_i)，整体数值就是全1减去原值。

同理：

$$[ bwd' = ones[len] - bwd ]$$

所以区间翻转非常好做。

---

## 六、树链剖分如何处理路径方向

路径 (u→v) 经过 LCA。

在 HLD 中，路径会拆成若干条重链上的连续区间。

但要注意，(g(u→v)) 是**从 u 到 v 的顺序**，因此不能只是无脑相加，需要按路径顺序拼接。

---

## 七、路径拆分策略

我们把路径拆成两部分：

1.  从 (u) 往上到 LCA 的部分
    
2.  从 LCA 往下到 (v) 的部分
    

在 HLD 中实现时：

-   当 `top[u] != top[v]` 时，深度更深的那个链头先处理；
    
-   若处理的是 `u` 这一侧，那么查询区间 `[dfn[top[u]], dfn[u]]`，这段在线段树上是“从上到下”，但真实路径方向是“从下到上”，所以要取 `bwd`
    
-   若处理的是 `v` 这一侧，那么查询区间 `[dfn[top[v]], dfn[v]]`，真实路径方向最终是“从上到下”，所以先暂存，之后逆序拼接，或在最后统一处理
    

一种常见写法：

-   用两个序列 `leftParts`、`rightParts`
    
-   `leftParts` 收集从 `u` 往上走的段，段方向已转成路径方向
    
-   `rightParts` 收集从 `v` 往上走的段，但最终要逆序拼接
    

最后：

$$[ u\to v = leftParts + middle + reverse(rightParts) ]$$

---

## 八、如何拼接两段信息

若段 A 长度为 `lenA`，值为 `valA`；段 B 长度为 `lenB`，值为 `valB`。<br>把 A 接在 B 前面后的值：

$$[ merge(A,B) = valA \times 2^{lenB} + valB ]$$

所以我们可以定义一个结构：

```cpp
struct Info {
    long long val;
    int len;
};
```

拼接函数：

```cpp
Info mergeInfo(Info a, Info b) {
    return { (a.val * pw2[b.len] + b.val) % MOD, a.len + b.len };
}
```

---

# 九、路径翻转

路径翻转是 HLD 标准操作：

当 `top[u] != top[v]` 时，每次修改深度更深链头到该点的区间。<br>最后处理同链上的 `[dfn[l], dfn[r]]`。

这个与普通树链剖分一致，不难。

---

# 十、完整算法步骤

---

## 1\. 预处理

-   建树
    
-   DFS1：求 `fa/depth/size/son`
    
-   DFS2：求 `top/dfn/rnk`
    
-   预处理 (2^i \\mod MOD) 和 `ones[i]=2^i-1`
    

---

## 2\. 线段树维护

每个节点维护：

-   `fwd`
    
-   `bwd`
    
-   `len`
    
-   `tag`
    

支持：

-   build
    
-   push\_up
    
-   apply\_flip
    
-   push\_down
    
-   range\_flip
    
-   range\_query
    

---

## 3\. 路径查询

定义 `Info` 表示一段按路径方向排列后的结果。

### 从 u 到 v 查询：

-   `left = empty`
    
-   `right = empty`
    

循环直到 `top[u] == top[v]`：

-   若 `depth[top[u]] >= depth[top[v]]`
    
    -   查询区间 `[dfn[top[u]], dfn[u]]`
        
    -   因为真实方向是 `u -> top[u]`，与 dfn 正方向相反，所以取 `bwd`
        
    -   拼到 `left` 后面
        
    -   `u = fa[top[u]]`
        
-   否则
    
    -   查询区间 `[dfn[top[v]], dfn[v]]`
        
    -   这段真实最终方向是 `top[v] -> v`，先取 `fwd`
        
    -   存到 `right`
        
    -   `v = fa[top[v]]`
        

最后同链：

-   若 `depth[u] >= depth[v]`
    
    -   查询 `[dfn[v], dfn[u]]`
        
    -   真实方向是 `u -> v`，取 `bwd`
        
    -   拼到 `left`
        
-   否则
    
    -   查询 `[dfn[u], dfn[v]]`
        
    -   真实方向是 `u -> v`，取 `fwd`
        
    -   拼到 `left`
        

最后把 `right` 逆序拼到 `left` 后面。

---

# 十一、复杂度

HLD + 线段树：

-   建树预处理：O(n)
    
-   每次路径修改 / 查询：
    

$$O(\log^2 n)$$

总复杂度：

$$[ O((n+m)\log^2 n) ]$$

---

# 十二、C++ 参考代码

下面给出可提交版本。

```cpp
#include <bits/stdc++.h>
using namespace std;

static const long long MOD = 1000000007LL;
static const int MAXN = 200000 + 5;

int n, m;
vector<int> g[MAXN];
int val[MAXN];

int fa[MAXN], dep[MAXN], sz[MAXN], son[MAXN];
int top[MAXN], dfn[MAXN], rdfn[MAXN], timer_;
long long pw2[MAXN], ones[MAXN];

struct Node {
    long long fwd, bwd; // 正向、反向二进制值
    int len;
    bool lazy;
} seg[MAXN << 2];

struct Info {
    long long val;
    int len;
    Info(long long v = 0, int l = 0) : val(v), len(l) {}
};

Info mergeInfo(const Info &a, const Info &b) {
    return Info((a.val * pw2[b.len] + b.val) % MOD, a.len + b.len);
}

void dfs1(int u, int p) {
    fa[u] = p;
    dep[u] = dep[p] + 1;
    sz[u] = 1;
    son[u] = 0;
    for (int v : g[u]) {
        if (v == p) continue;
        dfs1(v, u);
        sz[u] += sz[v];
        if (!son[u] || sz[v] > sz[son[u]]) son[u] = v;
    }
}

void dfs2(int u, int tp) {
    top[u] = tp;
    dfn[u] = ++timer_;
    rdfn[timer_] = u;
    if (son[u]) dfs2(son[u], tp);
    for (int v : g[u]) {
        if (v == fa[u] || v == son[u]) continue;
        dfs2(v, v);
    }
}

void push_up(int p) {
    seg[p].len = seg[p << 1].len + seg[p << 1 | 1].len;
    seg[p].fwd = (seg[p << 1].fwd * pw2[seg[p << 1 | 1].len] + seg[p << 1 | 1].fwd) % MOD;
    seg[p].bwd = (seg[p << 1 | 1].bwd * pw2[seg[p << 1].len] + seg[p << 1].bwd) % MOD;
}

void apply_flip(int p) {
    seg[p].fwd = (ones[seg[p].len] - seg[p].fwd + MOD) % MOD;
    seg[p].bwd = (ones[seg[p].len] - seg[p].bwd + MOD) % MOD;
    seg[p].lazy ^= 1;
}

void push_down(int p) {
    if (seg[p].lazy) {
        apply_flip(p << 1);
        apply_flip(p << 1 | 1);
        seg[p].lazy = false;
    }
}

void build(int p, int l, int r) {
    seg[p].lazy = false;
    seg[p].len = r - l + 1;
    if (l == r) {
        int u = rdfn[l];
        seg[p].fwd = seg[p].bwd = val[u];
        return;
    }
    int mid = (l + r) >> 1;
    build(p << 1, l, mid);
    build(p << 1 | 1, mid + 1, r);
    push_up(p);
}

void update(int p, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) {
        apply_flip(p);
        return;
    }
    push_down(p);
    int mid = (l + r) >> 1;
    if (ql <= mid) update(p << 1, l, mid, ql, qr);
    if (qr > mid) update(p << 1 | 1, mid + 1, r, ql, qr);
    push_up(p);
}

Node queryNode(int p, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) return seg[p];
    push_down(p);
    int mid = (l + r) >> 1;
    if (qr <= mid) return queryNode(p << 1, l, mid, ql, qr);
    if (ql > mid) return queryNode(p << 1 | 1, mid + 1, r, ql, qr);
    Node L = queryNode(p << 1, l, mid, ql, qr);
    Node R = queryNode(p << 1 | 1, mid + 1, r, ql, qr);
    Node res;
    res.len = L.len + R.len;
    res.fwd = (L.fwd * pw2[R.len] + R.fwd) % MOD;
    res.bwd = (R.bwd * pw2[L.len] + L.bwd) % MOD;
    res.lazy = false;
    return res;
}

void path_flip(int u, int v) {
    while (top[u] != top[v]) {
        if (dep[top[u]] >= dep[top[v]]) {
            update(1, 1, n, dfn[top[u]], dfn[u]);
            u = fa[top[u]];
        } else {
            update(1, 1, n, dfn[top[v]], dfn[v]);
            v = fa[top[v]];
        }
    }
    if (dep[u] >= dep[v]) update(1, 1, n, dfn[v], dfn[u]);
    else update(1, 1, n, dfn[u], dfn[v]);
}

long long path_query(int u, int v) {
    Info left(0, 0), right(0, 0);

    while (top[u] != top[v]) {
        if (dep[top[u]] >= dep[top[v]]) {
            Node cur = queryNode(1, 1, n, dfn[top[u]], dfn[u]);
            // 路径方向: u -> top[u]，因此取 bwd
            left = mergeInfo(left, Info(cur.bwd, cur.len));
            u = fa[top[u]];
        } else {
            Node cur = queryNode(1, 1, n, dfn[top[v]], dfn[v]);
            // 暂存 top[v] -> v，之后逆序拼接
            right = mergeInfo(Info(cur.fwd, cur.len), right);
            v = fa[top[v]];
        }
    }

    if (dep[u] >= dep[v]) {
        Node cur = queryNode(1, 1, n, dfn[v], dfn[u]);
        left = mergeInfo(left, Info(cur.bwd, cur.len));
    } else {
        Node cur = queryNode(1, 1, n, dfn[u], dfn[v]);
        left = mergeInfo(left, Info(cur.fwd, cur.len));
    }

    Info ans = mergeInfo(left, right);
    return ans.val;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> m;
    string s;
    cin >> s;
    for (int i = 1; i <= n; i++) val[i] = s[i - 1] - '0';

    for (int i = 1; i < n; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
        g[v].push_back(u);
    }

    pw2[0] = 1;
    for (int i = 1; i <= n; i++) pw2[i] = pw2[i - 1] * 2 % MOD;
    for (int i = 0; i <= n; i++) ones[i] = (pw2[i] - 1 + MOD) % MOD;

    dfs1(1, 0);
    dfs2(1, 1);
    build(1, 1, n);

    while (m--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            path_flip(u, v);
        } else {

        }
    }

}
```

# 最后做一个总结

---

## 第 1 题结论

重复 (10^9+1) 次后，最长严格递增子序列长度就是：

$$\boxed{\text{原数组不同元素个数}}$$

原因：严格递增中同值最多取一次，而重复轮数足够多，可以把所有不同值按升序依次取出。

---

## 第 2 题结论

最少相邻交换次数可以用经典贪心：

-   记录所有 `'('` 的位置
    
-   扫描字符串
    
-   一旦前缀失衡，就把后面最近的 `'('` 拉到当前位置
    

复杂度 (O(n))。

---

## 第 3 题结论

属于树上路径修改与方向相关路径查询问题：

-   用 **树链剖分 HLD**
    
-   配合 **线段树**
    
-   每段维护：
    
    -   正向二进制值
        
    -   反向二进制值
        
    -   长度
        
    -   翻转懒标记
        

复杂度

$$O((n+m)\log^2 n)$$
