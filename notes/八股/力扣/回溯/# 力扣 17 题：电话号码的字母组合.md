# 力扣 17 题：电话号码的字母组合

## 题目描述

给定一个仅包含数字 `2-9` 的字符串，返回所有它能表示的字母组合。答案可以按任意顺序返回。

数字到字母的映射如下（与电话按键相同）：

-   2: "abc"
    
-   3: "def"
    
-   4: "ghi"
    
-   5: "jkl"
    
-   6: "mno"
    
-   7: "pqrs"
    
-   8: "tuv"
    
-   9: "wxyz"
    

**示例：**

-   输入：digits = "23"
    
-   输出：["ad","ae","af","bd","be","bf","cd","ce","cf"]
    
-   输入：digits = ""
    
-   输出：[]
    

---

## 解法一：迭代法（队列实现）

### 思路

使用队列逐步构建所有组合。初始时队列中有一个空字符串，然后遍历每个数字，将队列中的每个组合与当前数字对应的所有字母进行组合，生成新的组合放入队列 [2]。

### 代码实现

```java
import java.util.*;

class Solution {
    public List<String> letterCombinations(String digits) {
        // 边界条件：空字符串直接返回空列表
        if (digits == null || digits.length() == 0) {
            return new ArrayList<>();
        }
        
        // 数字到字母的映射表
        String[] digitMap = {
            "", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"
        };
        
        // 使用队列存储中间组合结果
        Queue<String> queue = new LinkedList<>();
        queue.offer("");  // 初始放入空字符串
        
        // 遍历每个数字
        for (int i = 0; i < digits.length(); i++) {
            int digit = digits.charAt(i) - '0';  // 字符转数字
            String letters = digitMap[digit];     // 获取对应字母
            int queueSize = queue.size();         // 当前队列大小（本轮需要处理的组合数）
            
            // 处理队列中现有的所有组合
            for (int j = 0; j < queueSize; j++) {
                String current = queue.poll();  // 取出队首元素
                
                // 为当前组合添加新的字母
                for (int k = 0; k < letters.length(); k++) {
                    queue.offer(current + letters.charAt(k));  // 新组合入队
                }
            }
        }
        
        return new ArrayList<>(queue);
    }
}
```

### 复杂度分析

-   **时间复杂度**：O(3^m × 4^n)，其中 m 是对应 3 个字母的数字个数，n 是对应 4 个字母的数字个数
    
-   **空间复杂度**：O(3^m × 4^n)，队列需要存储所有中间组合
    

### 适用场景

-   输入长度较长，担心递归栈溢出
    
-   需要避免递归的场景
    
-   迭代思维更直观的开发者

## 解法二：回溯算法（递归 + StringBuilder）⭐ 最优解

### 思路

回溯算法是解决组合问题的经典方法。将问题抽象为树形结构，树的深度等于输入数字的长度，每个节点的分支数等于该数字对应的字母数 [1][5]。

**回溯三要素：**

1.  **路径**：当前已选择的字母组合
    
2.  **选择列表**：当前数字对应的所有字母
    
3.  **终止条件**：路径长度等于输入数字长度
    

### 代码实现

```java
import java.util.*;

class Solution {
    // 结果列表
    private List<String> result = new ArrayList<>();
    // 当前路径（使用 StringBuilder 提高字符串拼接效率）
    private StringBuilder path = new StringBuilder();
    // 数字到字母的映射表
    private String[] digitMap = {
        "", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"
    };
    
    public List<String> letterCombinations(String digits) {
        // 边界条件处理
        if (digits == null || digits.length() == 0) {
            return result;
        }
        
        // 开始回溯，从第 0 个数字开始
        backtrack(digits, 0);
        return result;
    }
    
    /**
     * 回溯函数
     * @param digits 输入的数字字符串
     * @param index 当前处理的数字索引（也是树的深度）
     */
    private void backtrack(String digits, int index) {
        // 终止条件：路径长度等于数字字符串长度
        if (index == digits.length()) {
            result.add(path.toString());  // 收集结果
            return;
        }
        
        // 获取当前数字对应的字母集合
        int digit = digits.charAt(index) - '0';  // 字符转数字
        String letters = digitMap[digit];         // 获取对应字母串
        
        // 遍历当前数字的所有字母（树的分支）
        for (int i = 0; i < letters.length(); i++) {
            // 做选择：添加当前字母到路径
            path.append(letters.charAt(i));
            
            // 递归：处理下一个数字
            backtrack(digits, index + 1);
            
            // 撤销选择：回溯，移除最后一个字母
            path.deleteCharAt(path.length() - 1);
        }
    }
}
```

### 复杂度分析

-   **时间复杂度**：O(3^m × 4^n)，必须遍历所有可能的组合
    
-   **空间复杂度**：O(m + n)，递归调用栈的深度，m+n 为输入数字长度 [2][5]
    

### 适用场景

-   **面试首选**：代码清晰，体现回溯算法理解
    
-   中等长度输入（题目提示 digits.length <= 4）
    
-   需要展示算法功底的技术面试

## 解法三：回溯算法（参数传递路径）

### 思路

将当前路径作为参数传递，避免使用全局变量，代码更加函数式，但字符串拼接开销稍大 [5]。

### 代码实现

```java
import java.util.*;

class Solution {
    // 数字到字母的映射表
    private String[] digitMap = {
        "", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"
    };
    
    public List<String> letterCombinations(String digits) {
        List<String> result = new ArrayList<>();
        
        // 边界条件处理
        if (digits == null || digits.length() == 0) {
            return result;
        }
        
        // 开始回溯，初始路径为空字符串
        backtrack(digits, 0, "", result);
        return result;
    }
    
    /**
     * 回溯函数
     * @param digits 输入的数字字符串
     * @param index 当前处理的数字索引
     * @param currentPath 当前已构建的组合字符串
     * @param result 结果列表
     */
    private void backtrack(String digits, int index, String currentPath, List<String> result) {
        // 终止条件
        if (index == digits.length()) {
            result.add(currentPath);
            return;
        }
        
        // 获取当前数字对应的字母集合
        int digit = digits.charAt(index) - '0';
        String letters = digitMap[digit];
        
        // 遍历所有字母
        for (int i = 0; i < letters.length(); i++) {
            // 递归时直接传递新字符串（隐式回溯）
            backtrack(digits, index + 1, currentPath + letters.charAt(i), result);
        }
    }
}
```

### 复杂度分析

-   **时间复杂度**：O(3^m × 4^n)
    
-   **空间复杂度**：O(3^m × 4^n)，每次递归都创建新字符串
    

### 适用场景

-   追求代码简洁性
    
-   函数式编程风格偏好
    
-   输入规模较小的场景

## 关键细节与技巧

### 1\. 数字到字母的映射技巧

```java
// 方式一：数组映射（推荐，访问速度快）
String[] digitMap = {"", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"};
int digit = digits.charAt(index) - '0';  // 字符'2'转数字2

// 方式二：HashMap 映射
Map<Character, String> digitMap = new HashMap<>();
digitMap.put('2', "abc");
digitMap.put('3', "def");
// ...
String letters = digitMap.get(digits.charAt(index));
```

### 2\. StringBuilder vs String 拼接

```java
// 推荐：StringBuilder（可变，原地修改）
StringBuilder path = new StringBuilder();
path.append('a');      // O(1)
path.deleteCharAt(...); // O(1)

// 不推荐：String 拼接（不可变，每次创建新对象）
String path = "";
path = path + 'a';     // O(n)，创建新字符串对象
```

### 3\. 回溯 vs 普通递归的区别

| 特征  | 回溯算法 | 普通递归 |
| --- | --- | --- |
| 状态恢复 | 需要撤销选择（pop/delete） | 不需要 |
| 路径存储 | 共享同一路径对象 | 参数传递新路径 |
| 空间效率 | 更高  | 较低  |
| 代码可读性 | 需要理解回溯过程 | 更直观 |

### 4\. 树形结构可视化

以输入 "23" 为例：

```text
                    root
                   / | \
                  a  b  c    (数字2对应的字母)
                 /|\ /|\ /|\
                d e f d e f d e f  (数字3对应的字母)
                
结果：ad, ae, af, bd, be, bf, cd, ce, cf
```

-   **树的深度** = 输入数字的长度
    
-   **每层分支数** = 当前数字对应的字母数
    
-   **叶子节点** = 最终组合结果 [1][5]

## 总结对比

| 解法  | 时间复杂度 | 空间复杂度 | 优点  | 缺点  | 推荐度 |
| --- | --- | --- | --- | --- | --- |
| 迭代法（队列） | O(3^m × 4^n) | O(3^m × 4^n) | 避免递归栈溢出 | 空间占用大 | ⭐⭐⭐ |
| 回溯（StringBuilder） | O(3^m × 4^n) | O(m+n) | 空间效率高，代码清晰 | 需理解回溯 | ⭐⭐⭐⭐⭐ |
| 回溯（参数传递） | O(3^m × 4^n) | O(3^m × 4^n) | 代码简洁，函数式 | 字符串开销大 | ⭐⭐⭐⭐ |

**核心要点：**

1.  本题是**多集合求组合**，不同于单集合的组合问题 [5]
    
2.  回溯算法是解决"所有可能"类问题的利器
    
3.  理解树形结构有助于掌握回溯的本质
    
4.  StringBuilder 在频繁字符串修改场景下性能更优
