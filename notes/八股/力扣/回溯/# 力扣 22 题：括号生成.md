# 力扣 22 题：括号生成

## 题目描述

给定 `n` 对括号，要求生成所有**有效的括号组合**。

例如：

```text
输入：n = 3
输出：["((()))","(()())","(())()","()(())","()()()"]
```

---

## 一、题目本质

这道题本质上是一个**构造型搜索问题**：

-   每一步只能放 `'('` 或 `')'`
    
-   但不是所有长度为 `2n` 的括号串都合法
    
-   合法括号串必须满足两个条件：
    

### 条件 1：左括号总数等于右括号总数

最终必须有 `n` 个 `'('` 和 `n` 个 `')'`

### 条件 2：任意前缀中，右括号数量不能超过左括号数量

比如：

-   `"())("` 不合法，因为在第三个字符时，右括号已经比左括号多了
    
-   `"(()())"` 合法
    

这条“前缀合法性”是这题最关键的剪枝条件。

---

## 二、结果数量的理论背景

这题的结果数量实际上是**第 n 个卡特兰数（Catalan Number）**：

$$[ C_n = \frac{1}{n+1}\binom{2n}{n} ]$$

例如：

-   `n = 1`，结果数 = 1
    
-   `n = 2`，结果数 = 2
    
-   `n = 3`，结果数 = 5
    
-   `n = 4`，结果数 = 14
    

这意味着：

-   结果本身就可能很多
    
-   所以最优算法也不可能比“输出所有答案”更快
    
-   因此这类题通常追求的是**尽量少走无效分支**
    

---

## 三、解法一：暴力枚举 + 合法性校验

### 适用场景

-   刚接触这题，想从最朴素思路理解
    
-   帮助理解“合法括号”的定义
    
-   不适合作为面试主解法
    

---

### 思路

长度固定为 `2n`，每个位置只有两种选择：

-   `'('`
    
-   `')'`
    

所以总共有：

$$[ 2^{2n} = 4^n ]$$

种可能。

我们把所有长度为 `2n` 的括号串全部生成出来，然后逐个判断是否合法。

---

### 合法性判断方法

维护一个变量 `balance`：

-   遇到 `'('`，`balance++`
    
-   遇到 `')'`，`balance--`
    

规则：

-   如果过程中 `balance < 0`，说明某个前缀右括号太多，不合法
    
-   最后 `balance == 0` 才合法
    

---

### 代码实现

```java
import java.util.ArrayList;
import java.util.List;

class Solution {
    private List<String> ans = new ArrayList<>();

    public List<String> generateParenthesis(int n) {
        char[] path = new char[2 * n];
        dfs(path, 0);
        return ans;
    }

    private void dfs(char[] path, int index) {
        if (index == path.length) {
            // 枚举完成后，检查当前括号串是否合法
            if (isValid(path)) {
                ans.add(new String(path));
            }
            return;
        }

        // 当前位置放左括号
        path[index] = '(';
        dfs(path, index + 1);

        // 当前位置放右括号
        path[index] = ')';
        dfs(path, index + 1);
    }

    private boolean isValid(char[] path) {
        int balance = 0;

        for (char c : path) {
            if (c == '(') {
                balance++;
            } else {
                balance--;
            }

            // 某个前缀中右括号多于左括号，直接不合法
            if (balance < 0) {
                return false;
            }
        }

        // 最终左右括号数量必须相等
        return balance == 0;
    }
}
```

---

### 复杂度分析

-   总共有 `2^(2n)` 个序列
    
-   每个序列校验一次，校验成本 `O(2n)`
    

所以总时间复杂度：

$$[ O(2^{2n} \cdot n) = O(4^n \cdot n) ]$$

空间复杂度：

-   递归深度 `O(n)`
    
-   结果集不算在辅助空间内
    

---

### 优缺点

#### 优点

-   思路极其直观
    
-   便于理解合法括号的定义
    

#### 缺点

-   枚举了大量无效结果
    
-   性能差，面试里通常不会作为最终答案
    

---

## 四、解法二：动态规划

### 适用场景

-   想理解这题和**卡特兰数递推**之间的关系
    
-   面试中如果被问“除了回溯还能怎么做”
    
-   适合作为补充解法，不是最常见主解
    

---

### 核心思路

设 `dp[i]` 表示由 `i` 对括号能组成的所有合法结果。

考虑 `i` 对括号中的一个合法串，一定可以写成这种结构：

```text
( A ) B
```

其中：

-   `A` 是一段合法括号串，使用了 `p` 对括号
    
-   `B` 也是一段合法括号串，使用了 `i - 1 - p` 对括号
    

所以可以得到递推：

$$dp[i] = \bigcup_{p=0}^{i-1} "(" + dp[p] + ")" + dp[i-1-p]$$

---

### 举例理解

以 `n = 3` 为例：

-   `p = 0`：`()` + `dp[2]`
    
-   `p = 1`：`(dp[1])` + `dp[1]`
    
-   `p = 2`：`(dp[2])` + `dp[0]`
    

把这些组合起来，就能得到所有结果。

---

### 代码实现

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

class Solution {
    public List<String> generateParenthesis(int n) {
        List<List<String>> dp = new ArrayList<>();

        // dp[0] = [""]
        // 表示 0 对括号只有一种“空结构”
        dp.add(new ArrayList<>(Arrays.asList("")));

        for (int i = 1; i <= n; i++) {
            List<String> current = new ArrayList<>();

            for (int p = 0; p < i; p++) {
                List<String> leftPart = dp.get(p);          // 括号内部 A 使用 p 对
                List<String> rightPart = dp.get(i - 1 - p); // 后半部分 B 使用 i-1-p 对

                for (String left : leftPart) {
                    for (String right : rightPart) {
                        // 按照 (A)B 的结构拼接
                        current.add("(" + left + ")" + right);
                    }
                }
            }

            dp.add(current);
        }

        return dp.get(n);
    }
}
```

---

### 复杂度分析

由于必须生成所有合法结果，时间复杂度与结果规模同阶。

通常可以写为：

-   时间复杂度：`O(Cn * n)`
    
-   空间复杂度：`O(Cn * n)`
    

其中 `Cn` 是第 `n` 个卡特兰数。

> 严格来说，DP 还会保存 `0...n` 的所有中间结果，所以工程上的常数通常会比回溯更大。

---

### 优缺点

#### 优点

-   递推关系非常优美
    
-   能很好体现你对“卡特兰结构”的理解
    
-   面试中属于加分思路
    

#### 缺点

-   不如回溯直观
    
-   中间状态保存较多
    
-   实现上常数较大
    

---

## 五、解法三：回溯 + 剪枝（最优，面试首选）

### 适用场景

-   面试主解法
    
-   实战最常用解法
    
-   所有“构造所有合法结果”的题目都很适合用这种模式
    

---

### 核心思想

从左到右构造字符串。

在任意时刻，维护：

-   已经用了多少个左括号 `left`
    
-   已经用了多少个右括号 `right`
    

那么有两个决策：

### 1）什么时候可以放 `'('`

只要 `left < n`，就还能放左括号

### 2）什么时候可以放 `')'`

只有当 `right < left` 时，才能放右括号

为什么？

因为右括号不能在任何前缀里超过左括号数量。

这就是最关键的剪枝。

---

### 搜索树理解

以 `n = 3` 为例：

-   一开始只能放 `'('`
    
-   放到某一步后，如果左括号没用完，可以继续放 `'('`
    
-   如果当前右括号数量小于左括号数量，说明可以闭合，就可以放 `')'`
    

也就是说，我们在搜索过程中就把所有不合法分支砍掉了，而不是生成完再检查。

这就是回溯优于暴力的根本原因。

---

### 为什么这是最优思路

因为：

-   每个合法答案都必须生成一次
    
-   无效答案尽早剪掉
    
-   不需要额外保存很多中间状态
    

这类“列举所有合法组合”的问题，回溯往往就是最优选择。

---

### 代码实现（推荐写法：char[]）

```java
import java.util.ArrayList;
import java.util.List;

class Solution {
    private List<String> ans = new ArrayList<>();

    public List<String> generateParenthesis(int n) {
        char[] path = new char[2 * n];
        dfs(n, 0, 0, 0, path);
        return ans;
    }

    private void dfs(int n, int left, int right, int index, char[] path) {
        // 当构造长度达到 2n，说明得到一个完整解
        if (index == path.length) {
            ans.add(new String(path));
            return;
        }

        // 只要左括号还没用完，就可以继续放 '('
        if (left < n) {
            path[index] = '(';
            dfs(n, left + 1, right, index + 1, path);
        }

        // 只有当前右括号数量少于左括号数量，才能放 ')'
        if (right < left) {
            path[index] = ')';
            dfs(n, left, right + 1, index + 1, path);
        }
    }
}
```

---

### 复杂度分析

设合法结果数量为 `Cn`。

-   每个合法结果长度为 `2n`
    
-   每个结果最终都要生成一次
    

所以：

-   时间复杂度：`O(Cn * n)`
    
-   空间复杂度：递归栈 `O(n)`，结果集不计入辅助空间时是 `O(n)`
    

这是这题最合理的复杂度。

## 六、三种解法对比

| 解法  | 核心思想 | 时间复杂度 | 空间复杂度 | 适用场景 |
| --- | --- | --- | --- | --- |
| 暴力枚举 + 校验 | 先生成所有可能，再筛合法 | `O(4^n * n)` | `O(n)` | 入门理解，不推荐面试主答 |
| 动态规划 | 利用 `(A)B` 结构递推 | `O(Cn * n)` | `O(Cn * n)` | 理解卡特兰递推、补充回答 |
| 回溯 + 剪枝 | 构造过程中直接保证合法 | `O(Cn * n)` | `O(n)` | 面试首选、最优解 |

## 七、为什么回溯里条件是 `right < left`，不是 `right < n`

这是这题最容易犯错的地方之一。

如果只写：

```java
if (right < n) {
    ...
}
```

那么会允许在左括号不足时提前放右括号，例如：

```text
")("
```

这样的串显然不合法。

真正正确的限制是：

```java
if (right < left)
```

含义是：

-   当前已有的左括号比右括号多
    
-   所以还有一个未闭合的左括号可以拿来匹配
    
-   这时才允许放右括号
    

---

## 八、为什么 `char[]` 往往比 `String` 拼接更好

有些人会这样写回溯：

```java
dfs(current + "(", ...)
dfs(current + ")", ...)
```

这样写逻辑没问题，但每次字符串拼接都会创建新对象，性能较差。

### 更推荐两种方式

#### 写法 1：`StringBuilder`

优点是直观，回溯时 append / deleteCharAt 即可

#### 写法 2：`char[]`

优点是：

-   常数更小
    
-   不需要回删
    
-   面试里更显得你对性能敏感
    

这题用 `char[]` 是非常合适的。

## 九、面试时如何讲这题

推荐讲法：

### 第一步：先说暴力思路，再指出问题

“最直接的方法是枚举所有长度为 `2n` 的括号串，再判断是否合法，但这样会产生大量无效结果，复杂度太高。”

### 第二步：引出剪枝条件

“合法括号串有一个关键性质：任意前缀中右括号数量不能超过左括号数量。因此在构造过程中，如果当前 `right >= left`，就不能再放右括号。”

### 第三步：给出回溯状态定义

“我用回溯维护当前用了多少左括号、多少右括号，并用一个长度为 `2n` 的路径数组来构造答案。”

### 第四步：说明复杂度

“最终复杂度与合法结果数量，即卡特兰数，同阶，时间复杂度可以认为是 `O(Cn * n)`。”

如果面试官继续追问“还有别的方法吗”，再补一个 DP：

-   `dp[i]` 表示 `i` 对括号的所有结果
    
-   每个结果都可以拆成 `(A)B`
    

这样回答会很完整。

## 十、常见错误

### 1\. 终止条件写错

正确终止条件通常是：

```java
if (index == 2 * n)
```

或者等价地：

```java
if (left == n && right == n)
```

---

### 2\. 放右括号条件写错

错误写法：

```java
if (right < n)
```

正确写法：

```java
if (right < left)
```

---

### 3\. 忘记 `dp[0] = [""]`

动态规划里这个初始化非常关键。

因为 `(A)B` 的递推中，`A` 或 `B` 都可能为空结构。

---

### 4\. 回溯时字符串回退处理不干净

如果用 `StringBuilder`，必须在递归返回后删除最后一个字符，否则路径会污染兄弟分支。

## 十一、这题和哪些经典题是同一套路

这题很典型，属于：

> **“枚举所有结果 + 在生成过程中进行合法性剪枝”**

同类题有：

-   全排列
    
-   子集
    
-   组合总和
    
-   复原 IP 地址
    
-   N 皇后
    
-   括号相关表达式生成
    

一旦你掌握了这题的模式，以后看到这类“列举所有合法结构”的题，第一反应就应该是：

1.  定义状态
    
2.  定义选择
    
3.  定义约束
    
4.  搜索过程中剪枝

## 十二、实际工作和面试中的建议

### 面试建议

1.  **主答回溯，不要主答暴力**
    
    -   面试官最想看到的是你能否发现“前缀合法性”这个剪枝条件
        
2.  **主动提卡特兰数是加分项**
    
    -   不一定要求写公式，但能说“结果数量是卡特兰数级别”会显得你理解更深
        
3.  **写代码时优先保证边界清晰**
    
    -   `left < n`
        
    -   `right < left`
        
    -   `index == 2 * n`
        
4.  **如果时间充足，再补 DP**
    
    -   这能体现你不是只会模板，而是真的理解结构递推
        

---

### 实际工作建议

这题在真实业务里不会原样出现，但它训练的是非常通用的能力：

-   构造类搜索
    
-   递归状态设计
    
-   剪枝优化
    
-   输出型问题的复杂度分析
    

在工程里，很多“生成所有配置”“枚举所有合法路径”“表达式构造”“规则组合”问题，本质上都和这题类似。

真正有价值的不是“括号生成”本身，而是你是否掌握了：

> **不要先生成一大堆无效候选，再过滤；而是在构造过程中尽早排除无效分支。**

这是回溯题的核心思想。

## 十四、一句话总结

这题的核心不是“生成括号”，而是：

> **利用“任意前缀中右括号不能多于左括号”这个约束，在搜索过程中做剪枝。**

所以最佳解法是：

-   **回溯**
    
-   **状态：left、right、index**
    
-   **剪枝：right < left**
    
-   **面试首选：回溯 +** `char[]`
