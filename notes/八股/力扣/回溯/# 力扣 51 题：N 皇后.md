# 力扣 51 题：N 皇后

## 题目核心

给定一个整数 `n`，要求在 `n x n` 的棋盘上放置 `n` 个皇后，使得任意两个皇后都不能互相攻击。

皇后的攻击范围是：

-   同一行
    
-   同一列
    
-   同一条对角线
    

要求返回 **所有不同的摆法**，每种摆法用字符串棋盘表示：

-   `'Q'` 表示皇后
    
-   `'.'` 表示空位
    

这题的本质是一个非常经典的 **回溯搜索** 问题。

---

## 一、问题本质与搜索模型

N 皇后最重要的建模方式是：

-   **按行放皇后**
    
-   第 `row` 行只尝试放一个皇后
    
-   决定第 `row` 行的列位置后，递归处理下一行
    

这样做有一个很大的好处：

-   **同行冲突天然不存在**
    
-   只需要检查：
    
    -   列是否冲突
        
    -   左上到右下对角线是否冲突
        
    -   右上到左下对角线是否冲突
        

所以搜索树结构是：

-   第 0 行尝试每一列
    
-   第 1 行尝试每一列
    
-   ...
    
-   第 `n-1` 行尝试每一列
    

一旦某个位置冲突，就直接剪枝，不再继续往下搜索。

---

## 二、解法一：朴素回溯 + 每次现场检查棋盘

### 适用场景

-   刚学回溯，想先把题做出来
    
-   面试里先快速讲清楚基本思路
    
-   强调“回溯框架”而不是极致性能
    

### 思路

我们用一个 `char[][] board` 保存当前棋盘状态。

递归到第 `row` 行时：

1.  尝试把皇后放在每一列 `col`
    
2.  调用 `isValid(row, col)` 检查当前位置是否合法
    
3.  如果合法：
    
    -   放置皇后
        
    -   递归下一行
        
    -   回溯撤销
        

### 为什么 `isValid` 只需要检查上方？

因为我们是 **一行一行从上往下放** 的：

-   当前行下面还没放皇后
    
-   当前行自己只会放一个皇后
    
-   所以只需要看“之前已经放好的行”
    

也就是：

-   同列向上检查
    
-   左上对角线检查
    
-   右上对角线检查
    

### 复杂度

-   时间复杂度：`O(n! * n)`
    
    -   搜索本身是指数级
        
    -   每次合法性检查最多还要扫 `O(n)`
        
-   空间复杂度：`O(n^2)`
    
    -   棋盘存储需要 `n x n`
        
    -   递归栈深度 `O(n)`
        

### 代码实现

```java
import java.util.ArrayList;
import java.util.List;

class Solution {
    private List<List<String>> ans = new ArrayList<>();
    private int n;
    private char[][] board;

    public List<List<String>> solveNQueens(int n) {
        this.n = n;
        this.board = new char[n][n];

        // 初始化棋盘
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                board[i][j] = '.';
            }
        }

        dfs(0);
        return ans;
    }

    private void dfs(int row) {
        // 所有行都放完，记录一种可行解
        if (row == n) {
            ans.add(buildBoard());
            return;
        }

        // 尝试在当前行的每一列放皇后
        for (int col = 0; col < n; col++) {
            if (!isValid(row, col)) {
                continue; // 不合法则剪枝
            }

            board[row][col] = 'Q'; // 放置皇后
            dfs(row + 1);          // 递归下一行
            board[row][col] = '.'; // 回溯撤销
        }
    }

    private boolean isValid(int row, int col) {
        // 检查同列是否已有皇后
        for (int i = 0; i < row; i++) {
            if (board[i][col] == 'Q') {
                return false;
            }
        }

        // 检查左上对角线
        for (int i = row - 1, j = col - 1; i >= 0 && j >= 0; i--, j--) {
            if (board[i][j] == 'Q') {
                return false;
            }
        }

        // 检查右上对角线
        for (int i = row - 1, j = col + 1; i >= 0 && j < n; i--, j++) {
            if (board[i][j] == 'Q') {
                return false;
            }
        }

        return true;
    }

    private List<String> buildBoard() {
        List<String> path = new ArrayList<>();

        for (int i = 0; i < n; i++) {
            path.add(new String(board[i]));
        }

        return path;
    }
}
```

## 三、解法二：回溯 + 列/对角线状态数组优化

### 适用场景

-   面试中最推荐写的版本
    
-   代码可读性和性能都比较平衡
    
-   大多数情况下，这是最合适的提交方案
    

### 核心优化

解法一最大的问题是：

-   每次尝试放皇后时，都要扫描棋盘判断是否合法
    

其实没必要每次“现场巡逻”，可以直接用状态记录冲突信息。

我们维护三个布尔数组：

1.  `cols[col]`
    
    -   表示某一列是否已经有皇后
        
2.  `diag1[row - col + n - 1]`
    
    -   表示主对角线（左上到右下）是否已有皇后
        
    -   因为 `row - col` 可能是负数，所以加上 `n - 1` 做偏移
        
3.  `diag2[row + col]`
    
    -   表示副对角线（右上到左下）是否已有皇后
        

这样，某个位置 `(row, col)` 是否可放，能在 **O(1)** 时间内判断。

### 对角线编号技巧

#### 1）主对角线

同一条主对角线满足：

```text
row - col = 常数
```

例如：

-   `(0, 0)` -> `0`
    
-   `(1, 1)` -> `0`
    
-   `(2, 2)` -> `0`
    

范围是：

```text
-(n - 1) ~ (n - 1)
```

为了变成数组下标：

```text
row - col + (n - 1)
```

范围变成：

```text
0 ~ 2n - 2
```

#### 2）副对角线

同一条副对角线满足：

```text
row + col = 常数
```

范围天然就是：

```text
0 ~ 2n - 2
```

### 为什么这比扫描棋盘更好？

因为每次冲突判断从：

-   扫描一整列/对角线
    

变成：

-   直接查数组
    

属于典型的 **用空间换时间**。

### 复杂度

-   时间复杂度：`O(n!)`
    
-   空间复杂度：`O(n)`
    
    -   不算结果集时，主要是：
        
        -   列数组
            
        -   两条对角线数组
            
        -   递归栈
            
        -   `queens` 记录每行皇后列号
            

### 代码实现

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

class Solution {
    private List<List<String>> ans = new ArrayList<>();
    private int n;

    // cols[col] = true 表示第 col 列已被占用
    private boolean[] cols;

    // diag1[row - col + n - 1] = true 表示主对角线已被占用
    private boolean[] diag1;

    // diag2[row + col] = true 表示副对角线已被占用
    private boolean[] diag2;

    // queens[row] = col，表示第 row 行的皇后放在第 col 列
    private int[] queens;

    public List<List<String>> solveNQueens(int n) {
        this.n = n;
        this.cols = new boolean[n];
        this.diag1 = new boolean[2 * n - 1];
        this.diag2 = new boolean[2 * n - 1];
        this.queens = new int[n];

        Arrays.fill(queens, -1);

        dfs(0);
        return ans;
    }

    private void dfs(int row) {
        // 所有行都已经合法放置
        if (row == n) {
            ans.add(buildBoard());
            return;
        }

        // 尝试当前行的每一列
        for (int col = 0; col < n; col++) {
            int d1 = row - col + n - 1; // 主对角线编号
            int d2 = row + col;         // 副对角线编号

            // 只要列或任一对角线被占用，就不能放
            if (cols[col] || diag1[d1] || diag2[d2]) {
                continue;
            }

            // 做选择：放置皇后，并标记占用状态
            queens[row] = col;
            cols[col] = true;
            diag1[d1] = true;
            diag2[d2] = true;

            dfs(row + 1); // 递归处理下一行

            // 撤销选择：恢复现场
            queens[row] = -1;
            cols[col] = false;
            diag1[d1] = false;
            diag2[d2] = false;
        }
    }

    private List<String> buildBoard() {
        List<String> board = new ArrayList<>();

        for (int row = 0; row < n; row++) {
            char[] line = new char[n];
            Arrays.fill(line, '.');
            line[queens[row]] = 'Q'; // 当前行皇后所在列
            board.add(new String(line));
        }

        return board;
    }
}
```

## 四、解法三：位运算回溯优化

### 适用场景

-   面试高阶加分项
    
-   追求性能
    
-   熟悉位运算剪枝
    
-   需要进一步展示“状态压缩”能力
    

### 核心思想

前面的数组方案，本质上已经把冲突判断优化到了 O(1)。

还能继续优化吗？

可以。把：

-   列占用状态
    
-   主对角线占用状态
    
-   副对角线占用状态
    

都压缩成一个二进制数。

假设 `n = 4`：

```text
1111  表示 4 列都纳入考虑范围
```

某一位为 `1` 表示该列可用 / 已占用，具体看语境。

### 状态定义

在递归某一行时，维护三个掩码：

-   `columns`：当前已经被占用的列
    
-   `diag1`：当前行上会被主对角线攻击的位置
    
-   `diag2`：当前行上会被副对角线攻击的位置
    

那么当前行可放的位置就是：

```java
available = limit & ~(columns | diag1 | diag2);
```

其中：

-   `limit = (1L << n) - 1`
    
-   `limit` 的低 `n` 位全是 `1`
    

### lowbit 技巧

如果 `available` 中有多个可选位置，我们可以每次取出最右侧的一个 `1`：

```java
position = available & -available;
```

这就是经典的 **lowbit** 操作。

然后：

-   放置这个位置
    
-   从 `available` 中删除这个位置
    
-   继续递归
    

### 为什么对角线要位移？

如果当前行某列放了皇后，那么对下一行来说：

-   主对角线攻击位置要 **左移一位**
    
-   副对角线攻击位置要 **右移一位**
    

这是因为皇后的对角线攻击会随着“行数 +1”而左右偏移。

### 复杂度

-   时间复杂度：仍然是指数级，通常写作 `O(n!)`
    
-   空间复杂度：`O(n)`
    

虽然大 O 级别没变，但 **常数更小**，实际运行通常更快。

### 代码实现

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

class Solution {
    private List<List<String>> ans = new ArrayList<>();
    private int n;

    public List<List<String>> solveNQueens(int n) {
        this.n = n;
        int[] queens = new int[n];
        Arrays.fill(queens, -1);

        // limit 的低 n 位为 1，例如 n=4 时，limit=1111
        long limit = (1L << n) - 1;

        dfs(0, limit, 0L, 0L, 0L, queens);
        return ans;
    }

    private void dfs(int row, long limit, long columns, long diag1, long diag2, int[] queens) {
        // 所有行都已成功放置
        if (row == n) {
            ans.add(buildBoard(queens));
            return;
        }

        // 当前行所有可放置的位置：未被列、主对角线、副对角线占用
        long available = limit & (~(columns | diag1 | diag2));

        while (available != 0) {
            // 取出最右侧的一个可放位置
            long position = available & (-available);

            // 从候选集合中删除该位置
            available -= position;

            // 根据二进制位置求出列下标
            int col = Long.numberOfTrailingZeros(position);

            queens[row] = col; // 记录当前行皇后所在列

            dfs(
                row + 1,
                limit,
                columns | position,                  // 当前列被占用
                ((diag1 | position) << 1) & limit,  // 主对角线影响到下一行时左移
                (diag2 | position) >>> 1,           // 副对角线影响到下一行时右移
                queens
            );

            queens[row] = -1; // 回溯撤销
        }
    }

    private List<String> buildBoard(int[] queens) {
        List<String> board = new ArrayList<>();

        for (int row = 0; row < n; row++) {
            char[] line = new char[n];
            Arrays.fill(line, '.');
            line[queens[row]] = 'Q';
            board.add(new String(line));
        }

        return board;
    }
}
```

## 五、三种解法对比

| 解法  | 冲突判断方式 | 时间复杂度 | 空间复杂度 | 特点  |
| --- | --- | --- | --- | --- |
| 朴素回溯 + 扫描棋盘 | 每次都现场检查 | `O(n! * n)` | `O(n^2)` | 最容易理解 |
| 回溯 + 状态数组 | `O(1)` 查列和对角线 | `O(n!)` | `O(n)` | 面试最推荐 |
| 位运算回溯 | 位掩码压缩状态 | `O(n!)` | `O(n)` | 性能最好，进阶加分 |

## 六、这题最容易写错的地方

### 1\. 对角线下标映射写错

这是最常见错误。

-   主对角线：`row - col + n - 1`
    
-   副对角线：`row + col`
    

很多人会漏掉 `+ n - 1`，导致数组越界。

---

### 2\. 回溯时忘记恢复状态

比如只写了：

```java
cols[col] = true;
```

递归完却没恢复：

```java
cols[col] = false;
```

这会导致后续搜索受到污染。

---

### 3\. 结果集里直接引用同一个棋盘对象

如果你把当前 `board` 直接塞进答案，而不是做快照，那么后续回溯修改会影响之前结果。

正确做法：

-   要么每次 `buildBoard()` 重新生成字符串列表
    
-   要么复制当前状态
    

---

### 4\. 位运算版本里移位方向搞反

常见混淆：

-   主对角线影响下一行：左移
    
-   副对角线影响下一行：右移
    

如果反了，答案就不对。

---

### 5\. 位运算版本没做 `& limit`

左移之后高位可能溢出到棋盘范围外，所以通常要：

```java
((diag1 | position) << 1) & limit
```

这样能保证只保留低 `n` 位。

## 七、为什么这题天然适合回溯

因为它有非常典型的“试探 + 撤销”结构：

1.  在当前行选一个位置
    
2.  检查是否合法
    
3.  合法就继续递归
    
4.  不合法就剪枝
    
5.  回来后撤销这个选择，尝试别的位置
    

这就是标准回溯模板：

```text
for 每个选择:
    if 不合法:
        continue
    做选择
    递归
    撤销选择
```

所以 N 皇后、全排列、子集、组合、数独，这些题本质上都属于同一类搜索题。

## 八、面试中应该怎么讲这题

### 推荐表达顺序

#### 第一层：先讲基础建模

“我会按行放置皇后。因为每一行只放一个，所以天然避免同行冲突。每次只需要判断当前列、主对角线、副对角线是否冲突。”

#### 第二层：再讲状态优化

“如果每次都扫描棋盘判断是否合法，效率不高。可以用三个数组分别记录列、主对角线、副对角线是否被占用，把合法性判断优化到 O(1)。”

#### 第三层：最后讲位运算进阶

“如果还想进一步优化常数，可以把列和对角线状态压成位掩码，通过 lowbit 快速枚举可放位置。”

这样会显得你的思路是：

-   先有正确建模
    
-   再做性能优化
    
-   最后给出高级解法
    

这个层次感在面试里非常重要。

## 九、实际工作或面试场景建议

### 1\. 真正写代码时，优先写“状态数组版”

原因很简单：

-   容易写对
    
-   代码可读性强
    
-   性能已经很好
    
-   面试官也容易跟上你的思路
    

也就是说，**解法二是综合最优解**。

---

### 2\. 位运算更适合当“优化补充”

如果你一上来就写位运算，可能存在两个问题：

-   面试官不一定第一时间看懂
    
-   自己更容易写错
    

更好的策略是：

-   先写解法二
    
-   如果面试官追问“还能优化吗”
    
-   再补充位运算方案
    

这会更稳。

---

### 3\. 这题和力扣 52 题强相关

-   **51 题**：返回所有棋盘方案
    
-   **52 题**：只返回方案总数
    

如果只是统计数量，不需要保存棋盘，那么：

-   状态数组版会更轻量
    
-   位运算版会更有优势
    

所以这两题通常是一起掌握的。

---

### 4\. 回溯题里“记录路径”比“实时拷贝大对象”更重要

这题有个非常实用的经验：

-   搜索过程中，尽量只记录必要状态
    
-   比如只记录 `queens[row] = col`
    
-   到叶子节点时再一次性生成结果
    

这是回溯题里非常常见的性能优化思想。

在实际开发中也类似：

-   中间态尽量轻量
    
-   输出时再做格式化/对象构建

## 十一、一个很值得记住的总结

N 皇后并不只是“摆棋子”问题，它其实是一个非常标准的搜索模型：

-   **决策顺序**：按行决策
    
-   **约束条件**：列、主对角线、副对角线
    
-   **剪枝方式**：冲突即停止向下搜索
    
-   **优化手段**：
    
    -   扫描棋盘
        
    -   状态数组
        
    -   位压缩
        

所以学会这题以后，很多回溯题的套路都会更清晰：

-   为什么按层递归
    
-   为什么要恢复现场
    
-   为什么要做状态压缩
    
-   为什么剪枝能大幅减少搜索空间
    

如果你后面继续刷回溯，N 皇后是一个非常关键的模板题。
