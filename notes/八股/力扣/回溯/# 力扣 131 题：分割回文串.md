# 力扣 131 题：分割回文串

## 题目描述

给定一个字符串 `s`，请你将 `s` 分割成一些子串，使得**每个子串都是回文串**，返回 `s` 所有可能的分割方案。

例如：

-   输入：`s = "aab"`
    
-   输出：`[["a","a","b"],["aa","b"]]`
    

---

## 题目本质

这题本质上是一个经典的：

-   **枚举所有切割方案**
    
-   并且每一步都要判断当前切出来的子串是否合法（是否为回文）
    

因为题目要求的是 **“所有方案”**，所以天然就有**搜索 / 回溯**的特征。

同时，字符串长度很小（这题长度上限通常是 `16`），这也在暗示：

-   可以接受指数级搜索
    
-   但可以通过优化“回文判断”来减少重复计算
    

---

## 回溯树怎么理解

以 `s = "aab"` 为例：

从下标 `0` 开始，可以切：

-   `"a"`，剩下 `"ab"`
    
-   `"aa"`，剩下 `"b"`
    
-   `"aab"`，但不是回文，不能继续
    

形成的搜索过程大致是：

```text
start = 0
├── "a"
│   start = 1
│   ├── "a"
│   │   start = 2
│   │   └── "b"
│   │       start = 3 -> 收集答案 ["a","a","b"]
│   └── "ab" 不是回文，剪枝
└── "aa"
    start = 2
    └── "b"
        start = 3 -> 收集答案 ["aa","b"]
```

所以核心问题只有两个：

1.  如何枚举每一个切割位置
    
2.  如何快速判断 `s[i...j]` 是否为回文
    

---

## 解法一：回溯 + 每次实时判断回文

### 思路

定义一个 `dfs(start)`：

-   表示从下标 `start` 开始，去枚举所有切割方案
    
-   枚举区间 `[start, end]`
    
-   如果 `s[start...end]` 是回文串，就把它加入当前路径 `path`
    
-   然后继续递归 `dfs(end + 1)`
    
-   递归回来后再回溯删除最后一个子串
    

### 为什么可行

因为每一层递归，其实就是在决定：

-   当前这一刀切到哪里结束
    

只要切出来的这一段是回文串，这一刀就是合法的。

---

### 代码实现（独立提交版本）

```java
import java.util.ArrayList;
import java.util.List;

class Solution {
    private List<List<String>> result = new ArrayList<>();
    private List<String> path = new ArrayList<>();

    public List<List<String>> partition(String s) {
        dfs(s, 0);
        return result;
    }

    private void dfs(String s, int start) {
        // 已经切到字符串末尾，说明当前 path 是一组合法方案
        if (start == s.length()) {
            result.add(new ArrayList<>(path)); // 注意要拷贝
            return;
        }

        // 枚举当前这一段的结束位置
        for (int end = start; end < s.length(); end++) {
            // 如果 s[start..end] 不是回文串，直接跳过
            if (!isPalindrome(s, start, end)) {
                continue;
            }

            // 当前这一段合法，加入路径
            path.add(s.substring(start, end + 1));

            // 继续处理后面的部分
            dfs(s, end + 1);

            // 回溯，撤销本次选择
            path.remove(path.size() - 1);
        }
    }

    private boolean isPalindrome(String s, int left, int right) {
        // 双指针判断回文
        while (left < right) {
            if (s.charAt(left) != s.charAt(right)) {
                return false;
            }
            left++;
            right--;
        }
        return true;
    }
}
```

---

### 复杂度分析

#### 时间复杂度

最坏情况下接近：

`O(n^2 * 2^n)`

原因：

-   所有切割方案本身就是指数级
    
-   每次尝试切割时，还要做一次回文判断
    
-   回文判断最坏要 `O(n)`
    

如果字符串像 `"aaaaaa..."` 这种，回文子串很多，搜索树会很大。

#### 空间复杂度

-   递归栈深度：`O(n)`
    
-   路径 `path`：`O(n)`
    
-   不算答案集时额外空间：`O(n)`
    

---

### 适用场景

这是**最容易在面试里先写出来的版本**：

-   思路直观
    
-   代码短
    
-   不容易写错
    

如果面试官先看你是否具备基础搜索能力，这一版已经足够说明问题。

## 解法二：动态规划预处理回文 + 回溯

### 思路升级点

解法一的瓶颈是：

-   每次 DFS 都重复判断很多子串是否为回文
    

比如 `s[2...5]` 可能会被反复检查很多次。

于是可以先用动态规划预处理一个二维数组：

`isPal[i][j]`

表示：

-   `s[i...j]` 是否为回文串
    

这样在 DFS 的时候，就不用再双指针现查了，而是可以 `O(1)` 判断。

---

### 回文 DP 的状态定义

定义：

```java
isPal[i][j] = true 表示 s[i..j] 是回文串
```

转移公式：

```java
isPal[i][j] = (s.charAt(i) == s.charAt(j)) && (j - i <= 2 || isPal[i + 1][j - 1])
```

### 为什么这个公式成立

一个字符串是回文，需要满足：

1.  两端字符相同
    
2.  去掉两端之后，中间那段仍然是回文
    

也就是：

-   `s[i] == s[j]`
    
-   并且 `s[i+1...j-1]` 是回文
    

但是对于长度 `1 / 2 / 3` 的情况，可以直接判断：

-   长度 1：一定是回文
    
-   长度 2：只要两个字符相同
    
-   长度 3：只要两端相同，中间自然不用再判断
    

所以用 `j - i <= 2` 作为边界条件非常方便。

---

### DP 为什么要从后往前填

因为 `isPal[i][j]` 依赖 `isPal[i + 1][j - 1]`。

所以必须保证：

-   在计算 `isPal[i][j]` 之前
    
-   `isPal[i + 1][j - 1]` 已经算好
    

因此 `i` 要从大到小遍历。

---

### 代码实现（独立提交版本，推荐掌握）

```java
import java.util.ArrayList;
import java.util.List;

class Solution {
    private List<List<String>> result = new ArrayList<>();
    private List<String> path = new ArrayList<>();
    private boolean[][] isPal;

    public List<List<String>> partition(String s) {
        int n = s.length();
        isPal = new boolean[n][n];

        // 预处理所有子串是否为回文
        for (int i = n - 1; i >= 0; i--) {
            for (int j = i; j < n; j++) {
                // 两端字符相等，且中间部分是回文
                if (s.charAt(i) == s.charAt(j)) {
                    // 长度 <= 3 时，只要两端相等即可
                    // 否则要看内部子串是否为回文
                    if (j - i <= 2 || isPal[i + 1][j - 1]) {
                        isPal[i][j] = true;
                    }
                }
            }
        }

        dfs(s, 0);
        return result;
    }

    private void dfs(String s, int start) {
        // 切到末尾，记录当前方案
        if (start == s.length()) {
            result.add(new ArrayList<>(path)); // 必须拷贝
            return;
        }

        // 枚举当前这一刀的结束位置
        for (int end = start; end < s.length(); end++) {
            // 直接 O(1) 判断是否回文
            if (!isPal[start][end]) {
                continue;
            }

            // 当前子串合法
            path.add(s.substring(start, end + 1));

            // 递归处理下一段
            dfs(s, end + 1);

            // 回溯撤销选择
            path.remove(path.size() - 1);
        }
    }
}
```

---

### 复杂度分析

#### 时间复杂度

-   预处理回文表：`O(n^2)`
    
-   DFS 搜索所有方案：最坏接近 `O(n * 2^n)`
    

整体可以写成：

`O(n^2 + n * 2^n)`

更准确地说，这题本身是**输出敏感**的：

-   因为你必须把所有方案都生成出来
    
-   答案数量本身在最坏情况下就是指数级
    

所以这已经是这类题非常标准、非常优秀的解法了。

#### 空间复杂度

-   回文 DP 表：`O(n^2)`
    
-   递归栈和路径：`O(n)`
    
-   总额外空间：`O(n^2)`
    

---

### 适用场景

这是这题最推荐、最常见、也最容易获得面试官认可的版本：

-   思路完整
    
-   有优化意识
    
-   时间复杂度明显优于“每次现查回文”

1.  先说“我会先用回溯做”
    
2.  再补一句“回文判断存在重复，可以用 DP 预处理优化到 O(1) 查询”
    

这样层次感会非常好。

适合：

-   想写出标准高质量答案

## 细节技巧与易错点

### 1\. 到达末尾时要拷贝 path

错误写法：

```java
result.add(path);
```

这样会把同一个 `path` 引用直接放进结果里，后续回溯时会被修改。

正确写法：

```java
result.add(new ArrayList<>(path));
```

---

### 2\. 回溯一定要删除最后一个元素

```java
path.add(...);
dfs(...);
path.remove(path.size() - 1);
```

这三步必须配套出现。

---

### 3\. DP 填表顺序不能写反

因为：

```java
isPal[i][j] 依赖 isPal[i + 1][j - 1]
```

所以 `i` 必须从后往前枚举。

正确：

```java
for (int i = n - 1; i >= 0; i--) {
    for (int j = i; j < n; j++) {
        ...
    }
}
```

---

### 4\. `j - i <= 2` 是非常高频的技巧

这个判断一口气覆盖了长度：

-   1
    
-   2
    
-   3
    

减少很多边界分支。

---

### 5\. 这题无法避免指数级结果

很多人会担心：

“有没有线性或多项式解法？”

没有。

因为题目要你返回**所有分割方案**，而答案数量本身就可能是指数级的。<br>所以真正能优化的是：

-   减少重复判断
    
-   降低常数
    
-   让代码结构更清晰
    

---

## 面试时的推荐表达方式

面试里可以按下面这个顺序说，逻辑会很清楚：

### 第一步：先定性

“这题要求返回所有方案，所以是典型回溯题。每次从当前位置尝试切一个前缀，如果这个前缀是回文串，就继续递归处理剩余部分。”

### 第二步：说明回文判断的优化点

“如果每次都临时判断回文，会有大量重复计算，所以可以先用 DP 预处理 `isPal[i][j]`，之后 DFS 中就能 `O(1)` 判断某段是否合法。”

### 第三步：补充复杂度

“因为要返回所有结果，最坏答案规模本身就是指数级，所以总复杂度不可能降到多项式。优化重点是减少回文判断的重复开销。”

这种回答方式会让面试官觉得你不只是会写代码，而是对问题的本质也有认识。

## 实际工作中的建议

虽然这题是算法题，但它背后的思想在工作中也很常见：

### 1\. “先预处理，再搜索”是很实用的套路

很多业务问题里也会出现：

-   搜索过程需要频繁判断某个条件
    
-   条件本身可以提前计算并缓存
    

这时就可以用类似“DP表 / 缓存表 / 预处理索引”的方式优化。

---

### 2\. 生成所有方案时要关注内存

这题因为要返回所有分割方案，所以结果集可能很大。<br>在真实业务里，如果方案数可能爆炸，通常不会一次性全部存到内存，而是：

-   分页生成
    
-   流式输出
    
-   回调消费
    
-   只保留 Top-K
    

算法题通常默认你把结果全存起来，但工程里不一定这么做。

---

### 3\. Java 中字符串切片要关注对象创建成本

这题里频繁使用：

```java
s.substring(start, end + 1)
```

由于字符串长度很小，问题不大。<br>但在工程里，如果字符串非常大、切片非常频繁，就要关注：

-   临时对象数量
    
-   GC 压力
    
-   是否可以只存索引区间，最后再统一构造结果
    

---

## 一句话总结

这题的核心不是“回文”，而是：

-   **回溯枚举分割**
    
-   **DP 预处理合法性**
    

最推荐掌握的模板是：

1.  `isPal[i][j]` 预处理所有回文子串
    
2.  `dfs(start)` 枚举从 `start` 开始的所有切法
    
3.  用 `path` 维护当前方案
    
4.  到终点时加入答案
    

如果你要在面试中写出质量更高的版本，优先写 **解法二：动态规划预处理回文 + 回溯**。
