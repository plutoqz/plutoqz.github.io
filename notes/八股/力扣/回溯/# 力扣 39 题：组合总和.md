# 力扣 39 题：组合总和

## 题目描述

给定一个**无重复元素**的正整数数组 `candidates` 和一个目标数 `target`，找出 `candidates` 中所有可以使数字和为 `target` 的组合。`candidates` 中的数字可以**无限制重复被选取**。

**约束条件：**

-   所有数字（包括 target）都是正整数
    
-   解集不能包含重复的组合
    
-   `1 <= candidates.length <= 30`
    
-   `1 <= candidates[i] <= 200`
    
-   `1 <= target <= 500`
    

**示例：**

```text
输入: candidates = [2,3,6,7], target = 7
输出: [[7], [2,2,3]]

输入: candidates = [2,3,5], target = 8
输出: [[2,2,2,2], [2,3,3], [3,5]]
```

---

## 解法一：DFS 枚举使用次数（次优解）

### 算法思路

枚举每个数字可以使用的次数，递归处理下一个数字。对于每个位置 `u`，尝试使用 0 次、1 次、2 次...直到超过 target。

### 核心要点

-   外层循环枚举当前数字的使用次数
    
-   递归处理下一个数字
    
-   回溯时恢复现场
    

### 代码实现

```java
import java.util.ArrayList;
import java.util.List;

class Solution {
    private List<List<Integer>> result = new ArrayList<>();
    private List<Integer> path = new ArrayList<>();
    
    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        dfs(candidates, 0, target);
        return result;
    }
    
    /**
     * @param candidates 候选数组
     * @param u 当前处理到第几个数字
     * @param target 剩余需要凑的目标值
     */
    private void dfs(int[] candidates, int u, int target) {
        // 终止条件：凑出目标值
        if (target == 0) {
            result.add(new ArrayList<>(path));  // 复制当前路径
            return;
        }
        
        // 终止条件：遍历完所有数字或目标值为负
        if (u == candidates.length || target < 0) {
            return;
        }
        
        // 枚举当前数字 candidates[u] 的使用次数
        for (int i = 0; candidates[u] * i <= target; i++) {
            // 选择 i 次当前数字
            for (int j = 0; j < i; j++) {
                path.add(candidates[u]);
            }
            
            // 递归处理下一个数字
            dfs(candidates, u + 1, target - candidates[u] * i);
            
            // 回溯：恢复现场
            for (int j = 0; j < i; j++) {
                path.remove(path.size() - 1);
            }
        }
    }
}
```

### 复杂度分析

-   **时间复杂度：** O(2^n × n)，指数级别
    
-   **空间复杂度：** O(target/min(candidates))，递归栈深度
    

### 适用场景

-   理解回溯的基本框架
    
-   数字可重复使用的组合问题

## 解法二：回溯 + 剪枝（最优解）⭐

### 算法思路

使用**回溯算法**遍历所有可能的组合，通过**排序 + 剪枝**优化搜索过程。关键技巧是使用 `start` 参数控制搜索起点，避免产生重复组合。

### 核心要点

1.  **排序**：便于剪枝判断，提前终止无效搜索
    
2.  **start 索引**：保证组合不重复（只向后搜索，不向前）
    
3.  **剪枝**：当当前值超过剩余 target 时，直接 break
    
4.  **可重复使用**：递归时传入 `i` 而非 `i+1`
    

### 代码实现

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

class Solution {
    private List<List<Integer>> result = new ArrayList<>();
    private List<Integer> path = new ArrayList<>();
    
    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        // 排序：便于剪枝，小的数字先尝试
        Arrays.sort(candidates);
        backtrack(candidates, target, 0);
        return result;
    }
    
    /**
     * @param candidates 候选数组（已排序）
     * @param target 剩余需要凑的目标值
     * @param start 当前搜索的起始索引（避免重复组合）
     */
    private void backtrack(int[] candidates, int target, int start) {
        // 终止条件：找到合法组合
        if (target == 0) {
            result.add(new ArrayList<>(path));  // 必须新建列表，否则后续修改会影响结果
            return;
        }
        
        // 从 start 开始遍历，避免重复组合
        for (int i = start; i < candidates.length; i++) {
            // 剪枝：当前数字已超过剩余目标值，后续更大的数字也不用尝试
            if (target - candidates[i] < 0) {
                break;  // 因为已排序，后续数字更大，直接跳出循环
            }
            
            // 做选择：将当前数字加入路径
            path.add(candidates[i]);
            
            // 递归：注意传入 i 而非 i+1，因为数字可以重复使用
            backtrack(candidates, target - candidates[i], i);
            
            // 撤销选择：回溯
            path.remove(path.size() - 1);
        }
    }
}
```

### 复杂度分析

-   **时间复杂度：** O(2^n × n)，但实际运行远优于理论值（剪枝优化）
    
-   **空间复杂度：** O(target/min(candidates))，递归栈深度 + 路径存储
    

### 递归树示例（candidates = [2,3,6,7], target = 7）

```text
                    target=7
                   /   |   \   \
                  2    3    6   7
                 /     |         \
            target=5  target=4   target=0✓
            /   |      /   \
           2    3     2    3
          /           |
     target=3      target=1(剪枝)
        |
        3
        |
    target=0✓
    
结果：[7], [2,2,3]
```

-   **面试首选解法**
    
-   需要输出所有组合的搜索问题
    
-   元素可重复使用的组合问题

## 解法三：回溯 + 剪枝优化版（进阶）

### 算法思路

在解法二基础上进一步优化：

1.  提前计算路径和，避免重复减法
    
2.  使用数组代替 List 减少对象创建开销
    

### 代码实现

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

class Solution {
    private List<List<Integer>> result = new ArrayList<>();
    
    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        Arrays.sort(candidates);
        backtrack(candidates, target, 0, new ArrayList<>());
        return result;
    }
    
    private void backtrack(int[] candidates, int target, int start, List<Integer> path) {
        if (target == 0) {
            result.add(new ArrayList<>(path));
            return;
        }
        
        for (int i = start; i < candidates.length; i++) {
            // 优化剪枝：提前判断
            if (candidates[i] > target) {
                break;
            }
            
            path.add(candidates[i]);
            // 关键：传入 i 允许重复使用当前元素
            backtrack(candidates, target - candidates[i], i, path);
            path.remove(path.size() - 1);
        }
    }
}
```

## 关键技巧总结

### 1\. 避免重复组合的核心

```java
// 正确：从 start 开始，保证组合有序
for (int i = start; i < candidates.length; i++) {
    backtrack(candidates, target - candidates[i], i, path);
}

// 错误：会产生重复组合 [2,3] 和 [3,2]
for (int i = 0; i < candidates.length; i++) {
    backtrack(candidates, target - candidates[i], i, path);
}
```

### 2\. 允许重复使用 vs 不允许重复使用

```java
// 本题：可重复使用，传入 i
backtrack(candidates, target - candidates[i], i, path);

// 力扣40题：不可重复使用，传入 i+1
backtrack(candidates, target - candidates[i], i + 1, path);
```

### 3\. 剪枝优化对比

```java
// 无剪枝：遍历所有数字
for (int i = start; i < candidates.length; i++) {
    // ...
}

// 有剪枝：排序后提前终止
Arrays.sort(candidates);
for (int i = start; i < candidates.length; i++) {
    if (candidates[i] > target) break;  // 剪枝
    // ...
}
```

### 4\. 路径复制的必要性

```java
// 正确：创建新列表
result.add(new ArrayList<>(path));

// 错误：引用同一对象，后续修改会影响已添加的结果
result.add(path);
```
