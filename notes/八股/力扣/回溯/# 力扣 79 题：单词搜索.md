# 力扣 79 题：单词搜索

题目要求在一个 `m x n` 的字符网格中，判断一个单词是否能由**上下左右相邻**的字符依次组成，且**同一个单元格不能重复使用**。这是一道非常典型的 **DFS（深度优先搜索）+ 回溯** 题。[1][3][5]

---

## 一、核心思路

这题最本质的事情只有一句话：

**从每一个格子出发，尝试把它当作单词起点，然后往四个方向递归搜索下一个字符。**

因为：

-   起点不确定，所以要枚举每个位置；
    
-   路径不确定，所以要往四个方向试；
    
-   同一个格子不能重复用，所以要做“访问标记”；
    
-   当前路径失败后，还要恢复现场继续试别的路径，所以需要“回溯”。
    

---

## 二、解法一：DFS + visited 数组（次优，最容易理解）

### 适用场景

-   刚开始学回溯；
    
-   面试里想先写一个**清晰、稳定、好解释**的版本；
    
-   不希望直接修改原始 `board`。
    

---

### 思路拆解

定义递归函数：

```java
dfs(x, y, index)
```

表示：

-   当前来到 `board[x][y]`
    
-   需要匹配 `word[index]`
    

递归逻辑：

1.  越界，失败；
    
2.  当前位置已经访问过，失败；
    
3.  当前字符和 `word[index]` 不相等，失败；
    
4.  如果 `index` 已经是单词最后一个字符，并且当前字符匹配，成功；
    
5.  标记当前格子已访问；
    
6.  往上、下、左、右四个方向递归；
    
7.  如果四个方向都失败，则撤销访问标记，返回失败。
    

---

### 复杂度分析

设网格大小为 `m * n`，单词长度为 `L`。

-   **时间复杂度**：`O(m * n * 3^L)`<br>枚举每个起点后，第一步最多 4 个方向，之后因为不能立刻走回头路，通常最多 3 个有效分支，因此常写成 `O(m * n * 3^L)`。[1][2]
    
-   **空间复杂度**：`O(m * n + L)`<br>`visited` 数组是 `O(m * n)`，递归栈深度最大 `L`。
    

---

### Java 代码

```java
class Solution {
    private int rows;
    private int cols;
    private char[][] board;
    private char[] target;

    // 四个方向：上、下、左、右
    private final int[] dx = {-1, 1, 0, 0};
    private final int[] dy = {0, 0, -1, 1};

    public boolean exist(char[][] board, String word) {
        // 健壮性判断
        if (board == null || board.length == 0 || board[0].length == 0) {
            return false;
        }
        if (word == null || word.length() == 0) {
            return false;
        }

        this.board = board;
        this.rows = board.length;
        this.cols = board[0].length;
        this.target = word.toCharArray();

        // visited[i][j] 表示当前搜索路径中，这个格子是否已被使用
        boolean[][] visited = new boolean[rows][cols];

        // 枚举每一个格子作为起点
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                if (dfs(i, j, 0, visited)) {
                    return true;
                }
            }
        }

        return false;
    }

    private boolean dfs(int x, int y, int index, boolean[][] visited) {
        // 1. 越界，直接失败
        if (x < 0 || x >= rows || y < 0 || y >= cols) {
            return false;
        }

        // 2. 当前格子已在本路径中使用过，不能重复使用
        if (visited[x][y]) {
            return false;
        }

        // 3. 字符不匹配，直接失败
        if (board[x][y] != target[index]) {
            return false;
        }

        // 4. 当前字符匹配，且已经匹配到最后一个字符，成功
        if (index == target.length - 1) {
            return true;
        }

        // 5. 标记当前格子已使用
        visited[x][y] = true;

        // 6. 尝试四个方向
        for (int k = 0; k < 4; k++) {
            int nx = x + dx[k];
            int ny = y + dy[k];

            if (dfs(nx, ny, index + 1, visited)) {
                visited[x][y] = false; // 回溯恢复，保持状态整洁
                return true;
            }
        }

        // 7. 四个方向都失败，恢复现场
        visited[x][y] = false;
        return false;
    }
}
```

## 三、解法二：DFS + 原地标记 + 剪枝优化（更优，面试更加分）

### 适用场景

-   已经掌握基本回溯；
    
-   想减少额外空间；
    
-   面试中希望展示“我不仅会写，还会优化”。
    

---

### 优化点

这个版本在解法一的基础上做了 3 个优化：

#### 优化 1：原地标记，省掉 `visited`

不用额外的 `boolean[][] visited`，直接把当前格子暂时改成特殊字符，比如 `'#'`，表示“已使用”，递归返回时再恢复。

因为题目里的字符只包含英文字母，所以 `'#'` 不会和合法字符冲突，用它做标记是安全的。[4]

---

#### 优化 2：字符频次剪枝

先统计 `board` 中每个字符的出现次数，再统计 `word` 所需字符次数。

如果 `word` 里某个字符出现次数比 `board` 里还多，那就**不可能组成**，直接返回 `false`，不必搜索。

这个剪枝在一些无解用例上会非常有效。

例如：

-   `board` 里只有 1 个 `'A'`
    
-   `word = "AA"`
    

那根本不用 DFS。

---

#### 优化 3：从更稀有的一端开始搜

如果 `word` 的首字符在 `board` 中出现很多次，而尾字符出现很少，那么从尾字符开始搜通常更省分支。

为什么可以反过来搜？

因为路径本质上是无向相邻的： 如果 `A -> B -> C -> D` 是一条合法路径， 那么反过来 `D -> C -> B -> A` 也是同一条路径。

所以把单词反转后搜索，答案不会变，但搜索树可能会小很多。

---

### 复杂度分析

-   **时间复杂度**：最坏仍然是 `O(m * n * 3^L)`，因为极端情况下还是要回溯搜索。[1][2]
    
-   **空间复杂度**：`O(L)`<br>主要是递归栈，额外频次数组大小视为常数。
    

也就是说，这个解法的**理论最坏时间复杂度不变**，但在实际测试和面试交流里，通常会显得更成熟。

---

### Java 代码

```java
class Solution {
    private int rows;
    private int cols;

    // 用一个长度为 5 的方向数组来表示四个方向
    // (dirs[i], dirs[i + 1]) 分别对应一个方向
    private final int[] dirs = {-1, 0, 1, 0, -1};

    public boolean exist(char[][] board, String word) {
        // 健壮性判断
        if (board == null || board.length == 0 || board[0].length == 0) {
            return false;
        }
        if (word == null || word.length() == 0) {
            return false;
        }

        rows = board.length;
        cols = board[0].length;

        // 剪枝 1：单词长度比总格子数还大，不可能
        if (word.length() > rows * cols) {
            return false;
        }

        // 统计 board 中每个字符出现次数
        int[] boardFreq = new int[128];
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                boardFreq[board[i][j]]++;
            }
        }

        char[] target = word.toCharArray();

        // 剪枝 2：如果 board 中某个字符数量不够，直接返回 false
        int[] remain = boardFreq.clone();
        for (char c : target) {
            remain[c]--;
            if (remain[c] < 0) {
                return false;
            }
        }

        // 剪枝 3：优先从更稀有的一端开始搜，减少起点数和分支数
        if (boardFreq[target[0]] > boardFreq[target[target.length - 1]]) {
            reverse(target);
        }

        // 枚举所有可能起点
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                // 起点字符不匹配，直接跳过
                if (board[i][j] != target[0]) {
                    continue;
                }

                if (dfs(board, i, j, target, 0)) {
                    return true;
                }
            }
        }

        return false;
    }

    private boolean dfs(char[][] board, int x, int y, char[] target, int index) {
        // 当前字符不匹配，失败
        if (board[x][y] != target[index]) {
            return false;
        }

        // 已经匹配到最后一个字符，成功
        if (index == target.length - 1) {
            return true;
        }

        // 原地标记当前格子为“已使用”
        char saved = board[x][y];
        board[x][y] = '#';

        // 向四个方向继续搜索
        for (int d = 0; d < 4; d++) {
            int nx = x + dirs[d];
            int ny = y + dirs[d + 1];

            // 越界，跳过
            if (nx < 0 || nx >= rows || ny < 0 || ny >= cols) {
                continue;
            }

            // 已使用，跳过
            if (board[nx][ny] == '#') {
                continue;
            }

            // 搜索下一个字符
            if (dfs(board, nx, ny, target, index + 1)) {
                board[x][y] = saved; // 回溯恢复
                return true;
            }
        }

        // 当前路径失败，恢复现场
        board[x][y] = saved;
        return false;
    }

    private void reverse(char[] arr) {
        int left = 0;
        int right = arr.length - 1;

        while (left < right) {
            char temp = arr[left];
            arr[left] = arr[right];
            arr[right] = temp;
            left++;
            right--;
        }
    }
}
```

## 四、为什么这题一定是“回溯”思维

这题和普通 DFS 不同的地方在于：

-   你不是单纯“遍历所有点”；
    
-   你是在尝试“构造一条满足条件的路径”；
    
-   这条路径如果走错了，要撤销刚才的选择，换个方向重试。
    

这正是回溯的典型特征：

1.  做选择；
    
2.  递归深入；
    
3.  失败则撤销选择；
    
4.  尝试下一个选择。
    

可以把它记成一个固定模板：

```text
for 每个起点:
    尝试搜索

搜索函数:
    先判断当前状态是否合法
    如果已经达到目标，返回成功
    标记当前点
    递归尝试所有下一步
    撤销标记
    返回失败
```

---

## 五、这题最容易写错的地方

### 1\. 忘记回溯恢复

这是最常见错误。

你如果把某个格子标记为已访问，却没有在失败后恢复，那么后续其他路径会错误地认为这个格子不能用，导致漏解。

---

### 2\. 终止条件顺序写错

正确顺序通常是：

1.  先判断当前位置是否合法；
    
2.  再判断是否已经匹配到最后一个字符。
    

因为如果你先判断 `index == word.length() - 1`，但当前字符其实不匹配，就会误判成功。

---

### 3\. 只从 `(0, 0)` 开始搜

单词起点可能在任意位置，所以一定要双层循环遍历所有格子。[5]

---

### 4\. 访问标记和边界判断混乱

如果使用 `visited` 数组，建议把判断顺序统一成：

-   越界？
    
-   已访问？
    
-   字符匹配？
    

这样逻辑最稳。

如果使用原地标记，建议先确保坐标合法，再判断字符。

---

### 5\. 提前剪枝没做，导致代码能过但不够漂亮

例如：

-   `word.length() > m * n`
    
-   字符数量不够
    

这些都属于非常典型的面试加分点。

---

## 六、面试时怎么讲，比较容易拿高评价

### 建议讲法

可以按这个顺序表达：

#### 第一步：先给朴素思路

“因为起点不确定，所以我会枚举每个格子作为起点；因为路径不确定，所以从当前格子往四个方向做 DFS；因为不能重复使用格子，所以需要 visited 或原地标记；搜索失败时进行回溯恢复。”

这一步能说明你已经抓住本质。

---

#### 第二步：给复杂度

“时间复杂度最坏是 `O(m * n * 3^L)`，空间复杂度如果用 visited 是 `O(m * n + L)`，如果原地标记则是 `O(L)`。”[1][2]

---

#### 第三步：主动补优化

“我还可以加两个剪枝：<br>1）如果 `word.length() > m * n`，直接 false；<br>2）如果 `board` 里某个字符数量不够，也直接 false；<br>3）还可以从 `word` 更稀有的一端开始搜，减少分支。”

面试官通常会觉得你不只是会背模板，而是会真正优化搜索问题。

---

## 七、实际工作里的建议

这题本身偏算法题，但里面的思维在工程里也很常见：

### 1\. 搜索类问题先想“状态 + 剪枝”

很多业务问题虽然不叫“回溯”，但本质一样：

-   路径规划
    
-   权限链路检查
    
-   表单联动路径验证
    
-   流程节点可达性判断
    

先明确：

-   当前状态是什么；
    
-   下一步能走到哪；
    
-   哪些状态应立即剪掉。
    

---

### 2\. 面试代码和生产代码的取舍不同

#### 面试里

优先写：

-   清晰；
    
-   边界稳；
    
-   能解释复杂度；
    
-   有一两个剪枝点。
    

#### 生产里

要额外考虑：

-   是否允许临时修改输入数据；
    
-   是否有并发访问；
    
-   是否需要防止递归栈过深；
    
-   是否要支持多次重复查询。
    

如果同一个 `board` 要查询很多个 `word`，那 79 题的单词逐个搜索就不够优了，通常会升级成 **Trie + DFS**，这其实就是力扣 212 的方向。

---

### 3\. 不要为了“省空间”牺牲可维护性

原地标记更省空间，但如果：

-   输入数据不允许改；
    
-   代码共享给别的线程；
    
-   需要调试中保留原始状态；
    

那么用 `visited` 反而更稳。

实际开发中，“更优”不一定总是“更适合”。

---

## 八、这题可以顺便掌握的技巧

### 技巧 1：方向数组

比写四段重复代码更整洁：

```java
int[] dx = {-1, 1, 0, 0};
int[] dy = {0, 0, -1, 1};
```

或者：

```java
int[] dirs = {-1, 0, 1, 0, -1};
```

这类写法在网格题里非常高频。[5]

---

### 技巧 2：原地标记

适合这种：

-   数据结构允许临时修改；
    
-   回溯结束后可以恢复；
    
-   目标是减少额外空间。
    

---

### 技巧 3：频次剪枝

这是搜索题里性价比很高的一类优化：

-   能提前判断无解；
    
-   代码量小；
    
-   常常非常加分。

## 九、结论

这题的标准答案就是：

-   **核心算法：DFS + 回溯**
    
-   **次优实现：visited 数组**
    
-   **更优实现：原地标记 + 剪枝优化**
    

如果你是为了真正掌握这题，建议按下面顺序练习：

1.  先独立写出 `visited` 版本；
    
2.  再写 `原地标记` 版本；
    
3.  最后补上 `长度剪枝 + 频次剪枝 + 反向搜索优化`。
