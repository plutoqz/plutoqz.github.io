# 力扣 3751 题：区间内数字的波动值之和

## 题意理解

一个数字的 **波动值** 定义为这个数字十进制表示中：

-   **峰** 的数量 + **谷** 的数量
    

其中：

-   若某一位数字 **严格大于** 左右相邻数字，则它是一个 **峰**
    
-   若某一位数字 **严格小于** 左右相邻数字，则它是一个 **谷**
    
-   **首位和末位** 不能作为峰或谷
    
-   **位数小于 3** 的数字，波动值为 `0`
    

要求返回区间 `[num1, num2]` 中所有数字的波动值总和。

---

## 核心观察

对于一个数字 `d0 d1 d2 ... d(k-1)`，只有中间位 `di (1 <= i <= k-2)` 才可能贡献波动值。

某个中间位 `di` 的贡献是：

-   `1`，如果 `(di > d(i-1) && di > d(i+1)) || (di < d(i-1) && di < d(i+1))`
    
-   `0`，否则
    

所以问题本质上是：

> 统计区间内所有数字中，所有中间位满足“与前一位、后一位构成峰/谷”的次数总和。

这类“统计区间内满足某种数位性质的总和”问题，标准做法是：

-   先求 `f(num)`：`[0, num]` 中所有数字的波动值总和
    
-   最终答案为：`f(num2) - f(num1 - 1)`
    

这就是典型的 **数位 DP**。

---

## 解法一：记忆化搜索数位 DP（推荐，最优）

### 适用场景

-   区间很大，不能逐个枚举
    
-   需要统计“所有数字某种局部结构的总贡献”
    
-   面试中看到“区间 + 数位性质 + 求总数/总和”，优先想到数位 DP
    

---

## 数位 DP 设计

我们设计一个 DFS，从高位到低位逐位填数字。

### 状态定义

设数字上界为字符串 `s`，定义：

`dfs(pos, prev2, prev1, started, tight)`

表示当前处理到第 `pos` 位时：

-   `pos`：当前处理的位置
    
-   `prev2`：前前位数字
    
-   `prev1`：前一位数字
    
-   `started`：前面是否已经开始构造数字（是否遇到过非前导 0）
    
-   `tight`：当前是否仍受上界约束
    

返回值不是单个整数，而是一个二元组：

-   `cnt`：当前状态下能构造出的合法数字个数
    
-   `sum`：这些数字的波动值总和
    

---

## 为什么要返回 `cnt` 和 `sum`

因为当前填入一个新数字 `d` 时，如果已经有至少两位真实数字了，那么：

-   可以判断 `prev1` 这一位是否是峰/谷
    
-   如果是，那么 **后续所有构造出的数字** 都要多加 `1`
    

所以：

-   假设递归下层返回 `(nextCnt, nextSum)`
    
-   当前新增贡献 `add` 可能为 `0` 或 `1`
    
-   那么当前总和应为：
    

```text
sum += nextSum + add * nextCnt
```

因为当前这一次形成的峰/谷，对所有下层构造出来的数字都生效。

这个技巧是数位 DP 中“统计总和”的关键。

---

## 前导零怎么处理

前导零不能真的算作数字的一部分，否则会错误制造峰谷。

例如数字 `12`，如果补成 `00012`，显然不能让前面的 `0` 参与判断。

所以要有 `started`：

-   `started = false`：还没遇到第一个非 0 数字
    
-   此时继续填 `0`，不算进入数字主体
    
-   一旦填入非 0，`started = true`
    

同时，为了知道已经真实填了几位，我们再用一个 `len` 更直观。

---

## 更清晰的状态设计

实际编码中，用下面这个状态更稳：

`dfs(pos, prev2, prev1, len, tight)`

其中：

-   `pos`：当前处理下标
    
-   `prev2`：倒数第二个真实数字
    
-   `prev1`：倒数第一个真实数字
    
-   `len`：当前已经构造出的真实位数
    
-   `tight`：是否贴着上界
    

这里：

-   `len = 0`：还没开始
    
-   `len = 1`：只有一个真实数字
    
-   `len >= 2`：已经可以在加入新数字时判断 `prev1` 是否为峰/谷
    

注意：前导零阶段不更新 `prev1/prev2`。

---

## 状态转移

枚举当前位填入数字 `d`：

### 情况 1：还没开始，且 `d == 0`

说明继续跳过前导零：

-   `len` 仍然为 `0`
    
-   不产生任何新贡献
    

### 情况 2：开始构造数字

-   若 `len == 0`，填入第一位真实数字
    
-   若 `len == 1`，填入第二位真实数字
    
-   若 `len >= 2`，这时可以判断 `prev1` 是否是峰/谷：
    

```java
add = ((prev1 > prev2 && prev1 > d) || (prev1 < prev2 && prev1 < d)) ? 1 : 0;
```

然后把状态推进为：

-   新的 `prev2 = prev1`
    
-   新的 `prev1 = d`
    
-   `len + 1`
    

---

## 递归终点

当 `pos == s.length()` 时：

说明一个数字已经构造完毕。

无论这个数字长度是几，只返回：

-   `cnt = 1`
    
-   `sum = 0`
    

为什么 `sum = 0`？

因为所有中间位的峰谷判断都已经在“加入下一位时”完成了，最后一位本身不能是峰谷，所以不需要额外处理。

## Java 实现

```java
import java.util.HashMap;
import java.util.Map;

class Solution {
    private static class Node {
        long cnt; // 当前状态下可构造的数字个数
        long sum; // 当前状态下这些数字的波动值总和

        Node(long cnt, long sum) {
            this.cnt = cnt;
            this.sum = sum;
        }
    }

    private char[] digits;
    private Map<String, Node> memo;

    public long countWavyNumbers(long num1, long num2) {
        return calc(num2) - calc(num1 - 1);
    }

    private long calc(long x) {
        if (x < 0) {
            return 0;
        }
        digits = Long.toString(x).toCharArray();
        memo = new HashMap<>();
        Node ans = dfs(0, 0, 0, 0, true);
        return ans.sum;
    }

    private Node dfs(int pos, int prev2, int prev1, int len, boolean tight) {
        if (pos == digits.length) {
            // 构造完成，一个数字对应一种方案，额外贡献为 0
            return new Node(1, 0);
        }

        String key = null;
        if (!tight) {
            key = pos + "#" + prev2 + "#" + prev1 + "#" + len;
            if (memo.containsKey(key)) {
                return memo.get(key);
            }
        }

        int up = tight ? digits[pos] - '0' : 9;
        long totalCnt = 0;
        long totalSum = 0;

        for (int d = 0; d <= up; d++) {
            boolean nextTight = tight && (d == up);

            if (len == 0 && d == 0) {
                // 继续处于前导零阶段，不进入真实数字
                Node next = dfs(pos + 1, 0, 0, 0, nextTight);
                totalCnt += next.cnt;
                totalSum += next.sum;
            } else if (len == 0) {
                // 填入第一位真实数字
                Node next = dfs(pos + 1, 0, d, 1, nextTight);
                totalCnt += next.cnt;
                totalSum += next.sum;
            } else if (len == 1) {
                // 填入第二位真实数字，还无法判断峰/谷
                Node next = dfs(pos + 1, prev1, d, 2, nextTight);
                totalCnt += next.cnt;
                totalSum += next.sum;
            } else {
                // 已有至少两位真实数字，可以判断 prev1 是否是峰或谷
                int add = ((prev1 > prev2 && prev1 > d) || (prev1 < prev2 && prev1 < d)) ? 1 : 0;

                Node next = dfs(pos + 1, prev1, d, len + 1, nextTight);
                totalCnt += next.cnt;
                totalSum += next.sum + (long) add * next.cnt; // 当前新贡献作用于所有后续方案
            }
        }

        Node res = new Node(totalCnt, totalSum);
        if (!tight) {
            memo.put(key, res);
        }
        return res;
    }
}
```

## 复杂度分析

设 `L` 为数字位数，`L <= 19`（如果输入是 `long`）。

### 状态数

状态维度大致为：

-   `pos`: `L`
    
-   `prev2`: `10`
    
-   `prev1`: `10`
    
-   `len`: `0 ~ L`
    
-   `tight`: 2 种（但只有 `tight = false` 才记忆化）
    

所以状态数大约：

```text
O(L * 10 * 10 * L) = O(L^2 * 100)
```

每个状态枚举 `0~9` 十个数字。

### 总时间复杂度

```text
O(L^2 * 100 * 10)
```

实际非常快。

### 空间复杂度

```text
O(L^2 * 100)
```

## 例子推演

以 `130` 为例：

数字序列 `1 3 0`

处理中间位 `3`：

-   `3 > 1`
    
-   `3 > 0`
    

所以是峰，贡献 `1`

而像 `123`：

-   中间位 `2`
    
-   `2 > 1` 但 `2 < 3`
    
-   既不是峰也不是谷
    
-   贡献 `0`

## 易错点

### 1\. 前导零不能参与峰谷判断

例如 `12` 不能看成 `012`，中间的 `1` 不能因为与 `0` 和 `2` 比较而变成谷。

所以必须区分：

-   还没开始构造真实数字
    
-   已经开始构造
    

---

### 2\. 峰谷判断必须是“严格”大小关系

不能写成：

```java
prev1 >= prev2 && prev1 >= d
```

必须是：

```java
prev1 > prev2 && prev1 > d
```

谷同理。

---

### 3\. 新加入一位时，判断的是“前一位”是否成为峰/谷

不是判断当前位 `d`。

因为是否成为峰/谷，必须同时知道它左右两边的数字。

所以当加入 `d` 时，才能确定 `prev1` 的身份。

---

### 4\. 区间答案要用前缀和方式

不能直接从 `num1` 遍历到 `num2`。

正确方式：

```java
answer = calc(num2) - calc(num1 - 1);
```

## 面试时怎么讲

这题非常适合按下面这个逻辑说：

### 第一步：把问题转成“按位贡献”

每个数字的波动值，是其中所有中间位是否满足峰/谷的贡献和。

### 第二步：转成“求 [0, x] 的总贡献”

区间和通常用前缀思想：

```text
sum(num1..num2) = f(num2) - f(num1-1)
```

### 第三步：想到数位 DP

因为要统计 `[0, x]` 内所有数字，并且性质依赖相邻位关系，所以需要从高位到低位构造。

### 第四步：状态只需记录最近两位

因为判断一个位置是否为峰/谷，只依赖：

-   左邻位
    
-   当前位
    
-   右邻位
    

在从左往右构造时，加入新位 `d` 后，只能判断前一位 `prev1`，因此状态保留最近两位即可。

这就是最核心的建模能力。

## 实际工作/竞面建议

### 1\. 看到“区间内所有整数 + 数位性质统计”，优先检查是否能用数位 DP

典型关键词：

-   `[L, R]`
    
-   所有整数
    
-   十进制表示
    
-   某种位之间关系
    
-   统计个数 / 总和 / 最大最小值
    

---

### 2\. 统计“总和”时，常常需要在状态返回中同时维护“方案数”和“总贡献”

这题中如果只返回 `sum` 不够，因为当前新增的 `add` 要乘以后续方案数。

很多数位 DP、树形 DP、状态 DP 都有这个技巧。

---

### 3\. 不要害怕状态里带“长度信息”

前导零问题非常常见，若逻辑不清楚，直接加一个 `len` 往往比硬推 `started` 更稳、更容易调试。

---

## 解法二：数组记忆化数位 DP（同思路，常数更优）

### 适用场景

-   追求更高性能
    
-   面试中希望展示更强的实现能力
    
-   已经确认位数上限较小，适合开数组

## Java 实现

```java
import java.util.Arrays;

class Solution {
    private static class Node {
        long cnt;
        long sum;

        Node(long cnt, long sum) {
            this.cnt = cnt;
            this.sum = sum;
        }
    }

    private char[] digits;
    private long[][][][] cntMemo;
    private long[][][][] sumMemo;
    private boolean[][][][] vis;

    public long countWavyNumbers(long num1, long num2) {
        return calc(num2) - calc(num1 - 1);
    }

    private long calc(long x) {
        if (x < 0) {
            return 0;
        }

        digits = Long.toString(x).toCharArray();
        int n = digits.length;

        cntMemo = new long[n][11][11][n + 1];
        sumMemo = new long[n][11][11][n + 1];
        vis = new boolean[n][11][11][n + 1];

        Node ans = dfs(0, 10, 10, 0, true);
        return ans.sum;
    }

    private Node dfs(int pos, int prev2, int prev1, int len, boolean tight) {
        if (pos == digits.length) {
            return new Node(1, 0);
        }

        if (!tight && vis[pos][prev2][prev1][len]) {
            return new Node(cntMemo[pos][prev2][prev1][len], sumMemo[pos][prev2][prev1][len]);
        }

        int up = tight ? digits[pos] - '0' : 9;
        long totalCnt = 0;
        long totalSum = 0;

        for (int d = 0; d <= up; d++) {
            boolean nextTight = tight && (d == up);

            if (len == 0 && d == 0) {
                // 继续跳过前导零
                Node next = dfs(pos + 1, 10, 10, 0, nextTight);
                totalCnt += next.cnt;
                totalSum += next.sum;
            } else if (len == 0) {
                // 第一位真实数字
                Node next = dfs(pos + 1, 10, d, 1, nextTight);
                totalCnt += next.cnt;
                totalSum += next.sum;
            } else if (len == 1) {
                // 第二位真实数字
                Node next = dfs(pos + 1, prev1, d, 2, nextTight);
                totalCnt += next.cnt;
                totalSum += next.sum;
            } else {
                // 判断 prev1 是否为峰或谷
                int add = ((prev1 > prev2 && prev1 > d) || (prev1 < prev2 && prev1 < d)) ? 1 : 0;
                Node next = dfs(pos + 1, prev1, d, len + 1, nextTight);
                totalCnt += next.cnt;
                totalSum += next.sum + (long) add * next.cnt;
            }
        }

        if (!tight) {
            vis[pos][prev2][prev1][len] = true;
            cntMemo[pos][prev2][prev1][len] = totalCnt;
            sumMemo[pos][prev2][prev1][len] = totalSum;
        }

        return new Node(totalCnt, totalSum);
    }
}
```

如果题目数据范围只是比如：

-   `num2 - num1 <= 10^5`

那可以直接暴力枚举每个数并逐位判断，代码简单很多。

## 解法三：暴力枚举（仅适合小范围校验，不适合大数据）

### 适用场景

-   本地对拍
    
-   小数据版本
    
-   面试中先给朴素思路，再优化到数位 DP
    

---

## Java 实现

```java
import java.util.*;

class Solution {
    public long countWavyNumbers(long num1, long num2) {
        long ans = 0;
        for (long x = num1; x <= num2; x++) {
            ans += getWavyValue(x);
        }
        return ans;
    }

    private int getWavyValue(long x) {
        char[] s = Long.toString(x).toCharArray();
        int n = s.length;
        if (n < 3) {
            return 0;
        }

        int res = 0;
        for (int i = 1; i < n - 1; i++) {
            int a = s[i - 1] - '0';
            int b = s[i] - '0';
            int c = s[i + 1] - '0';

            // 当前位严格大于左右两侧，为峰
            if (b > a && b > c) {
                res++;
            }
            // 当前位严格小于左右两侧，为谷
            else if (b < a && b < c) {
                res++;
            }
        }
        return res;
    }
}
```

## 暴力到最优的思维升级

### 暴力做法

-   枚举区间内每个数字
    
-   检查每个数字的每个中间位是否为峰/谷
    

复杂度大致：

```text
O((num2 - num1 + 1) * 位数)
```

当区间很大时必然超时。

---

### 优化思路

不要一个个数字算，而是：

-   一次性统计 `[0, x]` 中所有数字的总贡献
    
-   用数位 DP 按位“批量计算”
    

这就是这题的本质优化。

## 总结

这题是很标准、很有代表性的 **数位 DP + 局部模式统计总和** 问题。

关键点只有三个：

1.  **区间转前缀**
    
    ```java
    calc(num2) - calc(num1 - 1)
    ```
    
2.  **状态保留最近两位** 因为峰/谷判断只依赖三元组
    
3.  **DFS 返回** `(方案数, 总贡献)` 因为新形成的贡献要乘以后续方案数
