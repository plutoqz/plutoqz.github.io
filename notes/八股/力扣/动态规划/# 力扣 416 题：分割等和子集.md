# 力扣 416 题：分割等和子集

## 题目描述

给你一个 **只包含正整数** 的 **非空** 数组 `nums` ，请你判断是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

### 示例 1

```text
输入：nums = [1,5,11,5]
输出：true
解释：数组可以分割成 [1, 5, 5] 和 [11]
```

### 示例 2

```text
输入：nums = [1,2,3,5]
输出：false
解释：数组不能分割成两个元素和相等的子集
```

---

## 核心思路

这道题本质上是一个经典的 **0/1 背包问题**。

### 转化过程

如果整个数组元素和为 `sum`：

-   若 `sum` 是奇数，不可能分成两个和相等的子集，直接返回 `false`
    
-   若 `sum` 是偶数，则目标变成：
    

> 是否能从数组中选出若干个数，使它们的和恰好等于 `sum / 2`

因为一旦找到了和为 `sum / 2` 的一个子集，那么剩下的元素和自然也是 `sum / 2`。

所以问题变成：

-   每个数只能选一次
    
-   是否能恰好装满容量为 `target = sum / 2` 的背包
    

这就是标准 **0/1 背包可行性判断**。

---

## 解法一：二维 DP

### 适用场景

-   刚学习背包问题
    
-   想清晰理解状态定义与转移过程
    
-   面试中需要先讲出“最直观”的写法
    

### 状态定义

设：

```java
dp[i][j]
```

表示：

-   从前 `i` 个元素中选取若干个
    
-   是否可以凑出和 `j`
    

则答案是：

```java
dp[n][target]
```

### 状态转移

对于第 `i` 个数 `nums[i - 1]`：

-   不选它：`dp[i][j] = dp[i - 1][j]`
    
-   选它：若 `j >= nums[i - 1]`，则看 `dp[i - 1][j - nums[i - 1]]`
    

因此：

```java
dp[i][j] = dp[i - 1][j] || dp[i - 1][j - nums[i - 1]]
```

### 初始化

-   `dp[0][0] = true`：前 0 个数可以凑出和 0
    
-   `dp[0][j] = false`：前 0 个数无法凑出正数
    

### 代码实现

```java
import java.util.Arrays;

public class Solution {
    public boolean canPartition(int[] nums) {
        int sum = 0;
        for (int num : nums) {
            sum += num;
        }

        // 总和为奇数，不可能平分
        if ((sum & 1) == 1) {
            return false;
        }

        int target = sum / 2;
        int n = nums.length;

        // dp[i][j] 表示前 i 个数是否能凑出和 j
        boolean[][] dp = new boolean[n + 1][target + 1];

        // 和为 0 时，一定可以不选任何元素得到
        dp[0][0] = true;

        for (int i = 1; i <= n; i++) {
            int num = nums[i - 1];
            for (int j = 0; j <= target; j++) {
                // 不选当前数
                dp[i][j] = dp[i - 1][j];

                // 选当前数
                if (j >= num) {
                    dp[i][j] = dp[i][j] || dp[i - 1][j - num];
                }
            }
        }

        return dp[n][target];
    }
}
```

### 复杂度分析

-   时间复杂度：`O(n * target)`
    
-   空间复杂度：`O(n * target)`
    

### 优缺点

#### 优点

-   状态表达清晰
    
-   适合讲解与推导
    
-   不容易写错
    

#### 缺点

-   空间开销较大
    
-   在数据规模变大时不够优雅

## 解法二：一维 DP（状态压缩，最优）

### 适用场景

-   面试主推写法
    
-   需要空间优化
    
-   对 0/1 背包已经比较熟悉
    

---

## 为什么可以压缩成一维

二维转移：

```java
dp[i][j] = dp[i - 1][j] || dp[i - 1][j - num]
```

可以发现第 `i` 层只依赖第 `i - 1` 层，因此可以把第一维压掉。

设：

```java
dp[j]
```

表示当前处理到某个阶段时，是否可以凑出和 `j`。

转移为：

```java
dp[j] = dp[j] || dp[j - num]
```

但这里有一个非常关键的细节：

### `j` 必须从大到小遍历

```java
for (int j = target; j >= num; j--)
```

这是因为每个元素只能使用一次。

如果从小到大遍历，那么当前轮刚更新出来的 `dp[j - num]` 可能会再次被当前元素使用，导致一个数被重复选择，变成了“完全背包”的效果。

---

## 举个反例说明为什么要倒序

假设当前元素是 `num = 2`：

如果你从小到大更新：

```java
dp[2] = dp[2] || dp[0]
dp[4] = dp[4] || dp[2]
```

这里的 `dp[2]` 可能就是当前这个 `2` 刚刚更新出来的，于是 `dp[4]` 等于又用了一次 `2`，相当于当前数字被重复使用了。

所以 **0/1 背包必须倒序遍历容量**。

---

## 一维 DP 代码实现

```java
import java.util.Arrays;

public class Solution {
    public boolean canPartition(int[] nums) {
        int sum = 0;
        for (int num : nums) {
            sum += num;
        }

        // 总和为奇数，不可能分割成两个等和子集
        if ((sum & 1) == 1) {
            return false;
        }

        int target = sum / 2;

        // 如果某个元素已经大于 target，则它不可能出现在任一等和划分中
        for (int num : nums) {
            if (num > target) {
                return false;
            }
        }

        // dp[j] 表示是否可以凑出和 j
        boolean[] dp = new boolean[target + 1];
        dp[0] = true; // 和为 0 一定可达

        for (int num : nums) {
            // 倒序遍历，保证每个数只使用一次
            for (int j = target; j >= num; j--) {
                dp[j] = dp[j] || dp[j - num];
            }

            // 提前剪枝：如果已经能凑出 target，直接返回
            if (dp[target]) {
                return true;
            }
        }

        return dp[target];
    }
}
```

### 复杂度分析

-   时间复杂度：`O(n * target)`
    
-   空间复杂度：`O(target)`
    

### 优点

-   空间更优
    
-   面试中非常常见
    
-   是这道题的标准最优解

## 解法三：DFS + 记忆化搜索

### 适用场景

-   喜欢从“选 / 不选”角度理解问题
    
-   递归思维更自然
    
-   想把它和子集搜索类题目串起来理解
    

这道题也可以看作：

-   从下标 `index` 开始
    
-   当前还需要凑出 `remain`
    
-   每个数字都有“选”或“不选”两种决策
    

### 状态定义

记忆化状态：

```java
dfs(index, remain)
```

表示从 `index` 开始，是否能选出若干个数，使和恰好为 `remain`。

### 递归终止

-   `remain == 0`：找到可行方案，返回 `true`
    
-   `index == nums.length` 或 `remain < 0`：失败，返回 `false`
    

### 为什么要记忆化

暴力 DFS 会重复搜索大量相同状态。

例如：

-   到位置 `i`
    
-   还剩 `remain`
    

这个状态可能从很多路径到达，如果不缓存结果，会超时。

### 代码实现

```java
import java.util.HashMap;
import java.util.Map;

public class Solution {
    public boolean canPartition(int[] nums) {
        int sum = 0;
        for (int num : nums) {
            sum += num;
        }

        // 和为奇数，不可能平分
        if ((sum & 1) == 1) {
            return false;
        }

        int target = sum / 2;
        Map<String, Boolean> memo = new HashMap<>();

        return dfs(nums, 0, target, memo);
    }

    private boolean dfs(int[] nums, int index, int remain, Map<String, Boolean> memo) {
        // 恰好凑出目标
        if (remain == 0) {
            return true;
        }

        // 越界或超过目标
        if (index == nums.length || remain < 0) {
            return false;
        }

        String key = index + "_" + remain;
        if (memo.containsKey(key)) {
            return memo.get(key);
        }

        // 选择当前元素
        boolean choose = dfs(nums, index + 1, remain - nums[index], memo);

        // 不选择当前元素
        boolean notChoose = dfs(nums, index + 1, remain, memo);

        boolean ans = choose || notChoose;
        memo.put(key, ans);
        return ans;
    }
}
```

### 复杂度分析

-   时间复杂度：理论上接近 `O(n * target)`
    
-   空间复杂度：`O(n * target)`，包含记忆化和递归栈
    

### 优缺点

#### 优点

-   更符合“搜索决策树”的直觉
    
-   容易迁移到回溯 / 记忆化搜索问题
    

#### 缺点

-   实战中通常不如 DP 直接
    
-   Java 中字符串 key 的写法有额外开销

## 进一步优化 DFS 的写法

如果想减少 `String` 作为 key 的损耗，可以改成二维包装类型数组：

```java
import java.util.Arrays;

public class Solution {
    public boolean canPartition(int[] nums) {
        int sum = 0;
        for (int num : nums) {
            sum += num;
        }

        if ((sum & 1) == 1) {
            return false;
        }

        int target = sum / 2;
        Boolean[][] memo = new Boolean[nums.length][target + 1];
        return dfs(nums, 0, target, memo);
    }

    private boolean dfs(int[] nums, int index, int remain, Boolean[][] memo) {
        if (remain == 0) {
            return true;
        }

        if (index == nums.length || remain < 0) {
            return false;
        }

        if (memo[index][remain] != null) {
            return memo[index][remain];
        }

        boolean ans = dfs(nums, index + 1, remain - nums[index], memo)
                || dfs(nums, index + 1, remain, memo);

        memo[index][remain] = ans;
        return ans;
    }
}
```

这个版本更适合面试，因为：

-   状态更直接
    
-   常数更小
    
-   面试官更容易认可

## 三种解法对比

| 解法  | 时间复杂度 | 空间复杂度 | 推荐程度 | 特点  |
| --- | --- | --- | --- | --- |
| DFS + 记忆化 | `O(n * target)` | `O(n * target)` | 中   | 适合理解决策过程 |
| 二维 DP | `O(n * target)` | `O(n * target)` | 高   | 易讲清楚、易推导 |
| 一维 DP | `O(n * target)` | `O(target)` | 最高  | 面试和实战最优 |

按照“次优到最优”排序时，通常可以这样讲：

1.  DFS + 记忆化：帮助建立“选或不选”的模型
    
2.  二维 DP：把搜索状态表格化
    
3.  一维 DP：最终空间压缩优化

## 这题为什么是 0/1 背包，而不是完全背包

因为每个元素只能使用一次。

题目给的是一个数组中的元素，每个元素要么选，要么不选，不能重复拿同一个位置的数。

所以：

-   **0/1 背包**：每件物品最多选一次
    
-   **完全背包**：每件物品可以选无限次
    

这题显然属于前者。

## 面试里如何讲这题

### 推荐表达顺序

#### 第一步：先做数学判断

-   先求总和 `sum`
    
-   如果 `sum` 是奇数，直接 `false`
    

#### 第二步：问题转化

-   如果能分成两个等和子集，那么其中一个子集的和一定是 `sum / 2`
    
-   所以问题变成：能否从数组中选出若干数，使其和为 `target`
    

#### 第三步：识别模型

-   每个数只能选一次
    
-   典型 0/1 背包
    

#### 第四步：给出状态定义

-   `dp[j]`：是否能凑出和 `j`
    

#### 第五步：强调倒序遍历

-   倒序是为了防止同一轮重复使用同一个数
    
-   这是 0/1 背包的一大关键点
    

---

## 容易出错的细节

### 1\. 忘记判断总和是否为奇数

如果 `sum` 是奇数，不可能平分。

### 2\. 一维 DP 写成正序遍历

错误写法：

```java
for (int j = num; j <= target; j++)
```

这样会让一个数字被重复使用。

### 3\. `dp[0]` 没有初始化为 `true`

因为和为 0 一定可以由“什么都不选”得到，这是很多背包题的基础初始化。

### 4\. 状态定义混乱

很多人会把 `dp[j]` 理解成“最大和”而不是“是否可达”，导致转移写错。

这题最自然的是定义成布尔型可达性。

## 一个手动推演例子

以 `nums = [1,5,11,5]` 为例：

-   总和 `sum = 22`
    
-   `target = 11`
    

初始：

```text
dp[0] = true
其余为 false
```

### 处理 1

可以得到和 1：

```text
dp[1] = true
```

### 处理 5

基于已有状态：

-   `0 -> 5`
    
-   `1 -> 6`
    

于是：

```text
dp[5] = true
dp[6] = true
```

### 处理 11

直接可以得到：

```text
dp[11] = true
```

既然已经可以凑出 `11`，说明可以平分，返回 `true`。

## Java 面试版推荐写法

这是最建议记住的一版：

```java
import java.util.Arrays;

public class Solution {
    public boolean canPartition(int[] nums) {
        int sum = 0;
        for (int num : nums) {
            sum += num;
        }

        // 奇数和无法平分
        if ((sum & 1) == 1) {
            return false;
        }

        int target = sum / 2;
        boolean[] dp = new boolean[target + 1];
        dp[0] = true; // 不选任何元素可以组成 0

        for (int num : nums) {
            // 0/1 背包必须倒序，避免一个元素被重复使用
            for (int j = target; j >= num; j--) {
                dp[j] = dp[j] || dp[j - num];
            }
        }

        return dp[target];
    }
}
```

## 实际工作中的建议

这题在实际业务中不会直接原样出现，但它背后的思想非常常用：

### 1\. 资源分配 / 负载均衡

例如把若干任务分成两组，让两组工作量尽量均衡。<br>这题是“恰好均分”的判定版，现实中常常会扩展为“尽量接近一半”的优化版。

### 2\. 背包模型识别能力

很多需求表面看不是背包，但本质是：

-   每个元素选或不选
    
-   目标是凑某个和 / 达成某个容量 / 满足某种约束
    

能快速识别成 0/1 背包，是很重要的工程抽象能力。

### 3\. 面试中不要只会背答案

更重要的是你要讲清楚：

-   为什么能转化成 `sum / 2`
    
-   为什么是一维 DP
    
-   为什么必须倒序
    

如果这三点能讲顺，面试官通常会认为你是真的理解了。

## 一句话总结

这题的本质是：

> 判断数组中是否存在一个子集，其元素和恰好等于总和的一半。

因此直接套用 **0/1 背包的可达性判断**，其中 **一维 DP + 倒序遍历** 是最推荐、最常考、最实用的解法。
