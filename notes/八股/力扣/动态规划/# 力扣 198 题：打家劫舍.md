
# 力扣 198 题：打家劫舍

## 题目理解

给定一个整数数组 `nums`，`nums[i]` 表示第 `i` 间房屋存放的金额。

限制条件：**相邻的房屋不能在同一晚被偷**，否则会触发警报。

目标：在不触发警报的前提下，返回一晚能够偷到的**最大金额**。

---

## 一、核心本质

这道题本质上是一个非常经典的 **线性动态规划** 问题。

对于每一间房屋，都只有两种选择：

1. **不偷当前房屋**：那么最大收益就是前一间房屋的最优结果
2. **偷当前房屋**：那么前一间房屋就不能偷，只能加上前前一间房屋的最优结果

于是转移方程就出来了：

\[
dp[i] = \max(dp[i-1], dp[i-2] + nums[i])
\]

含义：

- `dp[i]`：表示偷到下标 `i` 这间房屋为止，能够获得的最大金额

---

## 二、先从最直观的思路开始

---

## 解法一：记忆化搜索（自顶向下 DP）

### 思路

站在下标 `i` 位置时，有两种决策：

- 不偷第 `i` 间，去考虑 `i - 1`
- 偷第 `i` 间，那么只能去考虑 `i - 2`

递归表达式：

\[
dfs(i) = \max(dfs(i-1), dfs(i-2) + nums[i])
\]

如果直接暴力递归，会有大量重复计算，所以要加一个 `memo` 数组做缓存。

### 适用场景

- 适合理解“状态转移”的来源
- 面试中用于先讲出递归模型，再自然过渡到动态规划
- 如果你对 DP 的定义还不够熟，自顶向下通常更容易想出来

### 代码实现

```java
import java.util.Arrays;

class Solution {
    public int rob(int[] nums) {
        int n = nums.length;
        int[] memo = new int[n];
        Arrays.fill(memo, -1); // -1 表示该状态还没有计算过
        return dfs(nums, n - 1, memo);
    }

    private int dfs(int[] nums, int i, int[] memo) {
        if (i < 0) {
            return 0; // 越界时说明没有房屋可偷，收益为 0
        }

        if (memo[i] != -1) {
            return memo[i]; // 直接返回缓存结果，避免重复递归
        }

        int notRob = dfs(nums, i - 1, memo); // 不偷当前房屋
        int rob = dfs(nums, i - 2, memo) + nums[i]; // 偷当前房屋

        memo[i] = Math.max(notRob, rob); // 记录最优解
        return memo[i];
    }
}
```

### 复杂度分析

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`
  - `memo` 数组占 `O(n)`
  - 递归栈最深也可能到 `O(n)`

### 细节技巧

1. `memo` 初始化为 `-1`，因为题目中金额非负，`-1` 可以作为“未计算”的标记
2. `i < 0` 返回 0，这是递归边界
3. 这个写法特别适合先把“选 or 不选”的决策树讲清楚

## 解法二：动态规划数组（自底向上）

### 思路

既然有状态转移方程：

\[
dp[i] = \max(dp[i-1], dp[i-2] + nums[i])
\]

那就可以直接从前往后迭代计算。

### 状态定义

- `dp[i]`：表示从第 `0` 间房到第 `i` 间房，能偷到的最大金额

### 初始化

- `dp[0] = nums[0]`
- `dp[1] = max(nums[0], nums[1])`

为什么第二个是这样？

因为对于前两间房，你只能选一间偷，所以取两者最大值。

### 适用场景

- 最标准、最容易写对的写法
- 面试中如果时间紧，直接写这个成功率很高
- 如果后续需要还原偷了哪些房子，数组 DP 更方便扩展

### 代码实现

```java
import java.util.Arrays;

class Solution {
    public int rob(int[] nums) {
        int n = nums.length;

        if (n == 0) {
            return 0; // 没有房屋
        }
        if (n == 1) {
            return nums[0]; // 只有一间房，直接偷
        }

        int[] dp = new int[n];
        dp[0] = nums[0]; // 只看第一间房
        dp[1] = Math.max(nums[0], nums[1]); // 看前两间房时的最优解

        for (int i = 2; i < n; i++) {
            int notRob = dp[i - 1]; // 不偷当前房屋
            int rob = dp[i - 2] + nums[i]; // 偷当前房屋
            dp[i] = Math.max(notRob, rob); // 取两种选择中的最大值
        }

        return dp[n - 1];
    }
}
```

### 复杂度分析

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

### 推演示例

以 `nums = [2, 7, 9, 3, 1]` 为例：

- `dp[0] = 2`
- `dp[1] = max(2, 7) = 7`
- `dp[2] = max(dp[1], dp[0] + 9) = max(7, 11) = 11`
- `dp[3] = max(dp[2], dp[1] + 3) = max(11, 10) = 11`
- `dp[4] = max(dp[3], dp[2] + 1) = max(11, 12) = 12`

最终答案：`12`

对应选择的是第 `0、2、4` 间房，总金额 `2 + 9 + 1 = 12`

### 细节技巧

很多人会把转移写错成：

```java
dp[i] = Math.max(dp[i - 1] + nums[i], dp[i - 2] + nums[i]);
```

这是错误的，因为：

- 如果已经用了 `dp[i - 1]`，就意味着前 `i-1` 间房的最优解里**可能已经偷了第 `i-1` 间**
- 这时再加 `nums[i]` 就可能违反“不能偷相邻房屋”的限制

所以正确写法只能是：

```java
dp[i] = Math.max(dp[i - 1], dp[i - 2] + nums[i]);
```

## 解法三：状态压缩 DP（最优写法）

### 思路

观察状态转移：

\[
dp[i] \text{ 只依赖 } dp[i-1] \text{ 和 } dp[i-2]
\]

这说明没必要保留整个 `dp` 数组，只需要两个变量滚动更新即可。

设：

- `prev2` 表示 `dp[i-2]`
- `prev1` 表示 `dp[i-1]`

那么当前值：

- `cur = max(prev1, prev2 + nums[i])`

更新：

- `prev2 = prev1`
- `prev1 = cur`

### 适用场景

- 追求最优空间复杂度
- 面试中写完数组 DP 后，可进一步优化展示你的 DP 熟练度
- 实际工作里如果只需要最终值，不需要路径还原，优先用这种写法

### 代码实现

```java
import java.util.Arrays;

class Solution {
    public int rob(int[] nums) {
        int n = nums.length;

        if (n == 0) {
            return 0; // 没有房屋
        }
        if (n == 1) {
            return nums[0]; // 只有一间房
        }

        int prev2 = nums[0]; // 相当于 dp[0]
        int prev1 = Math.max(nums[0], nums[1]); // 相当于 dp[1]

        for (int i = 2; i < n; i++) {
            int cur = Math.max(prev1, prev2 + nums[i]); // 当前最优解
            prev2 = prev1; // 原 dp[i-1] 成为下一轮的 dp[i-2]
            prev1 = cur;   // 当前结果成为下一轮的 dp[i-1]
        }

        return prev1;
    }
}
```

### 复杂度分析

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 三、进一步统一理解：选与不选模型

这题可以理解成一个经典模板：

> 在数组中选择若干元素，但不能选相邻元素，使得总和最大。

这是非常重要的模型，很多题都可以归到这里。

比如当前下标 `i`：

- **不选 `nums[i]`**：答案来自 `i-1`
- **选 `nums[i]`**：答案来自 `i-2 + nums[i]`

所以这道题和“爬楼梯”的思维有点像，都是当前状态依赖前几个状态，只不过这里做的是**取最大值决策**。

## 四、另一种更简洁的状态压缩写法

有些人更喜欢这种“双状态”写法：

- `take`：偷当前房屋时的最大收益
- `skip`：不偷当前房屋时的最大收益

转移：

- 新的 `take = skip + nums[i]`
- 新的 `skip = max(skip, take)`

注意更新顺序需要借助临时变量。

### 代码实现

```java
import java.util.Arrays;

class Solution {
    public int rob(int[] nums) {
        int take = 0; // 偷当前房屋时的最大收益
        int skip = 0; // 不偷当前房屋时的最大收益

        for (int num : nums) {
            int newTake = skip + num; // 当前偷，则上一间必须不偷
            int newSkip = Math.max(skip, take); // 当前不偷，则上一间可偷可不偷

            take = newTake;
            skip = newSkip;
        }

        return Math.max(take, skip);
    }
}
```

### 这个写法的优点

- 抽象层次更高
- 容易迁移到股票、打家劫舍 II、树形 DP 等题目
- 如果你擅长“状态机 DP”，这个写法很漂亮

### 这个写法的缺点

- 对初学者来说没有 `dp[i] = max(...)` 那么直观
- 面试时如果解释不到位，反而可能让表达变复杂

## 五、为什么这题是动态规划

动态规划通常有两个标志：

### 1. 最优子结构

偷到第 `i` 间房的最优解，可以由更小规模问题推出：

- 第 `i-1` 间的最优解
- 第 `i-2` 间的最优解

### 2. 重叠子问题

无论是递归过程还是决策过程，很多状态会重复使用，例如 `dfs(i-2)` 会被多次依赖。

这正是 DP 的典型特征。

---

## 六、容易出错的点

### 1. 边界条件没处理全

必须注意：

- `n == 0`
- `n == 1`
- `n == 2`

虽然有些写法通过通用初始化也能绕过 `n == 2`，但写清楚更稳。

---

### 2. 把“偷当前”理解错了

“偷当前房屋”不是：

```java
dp[i - 1] + nums[i]
```

而是：

```java
dp[i - 2] + nums[i]
```

因为相邻房屋不能同时偷。

---

### 3. 状态压缩更新顺序错误

例如：

```java
prev2 = prev1;
prev1 = Math.max(prev1, prev2 + nums[i]);
```

这样就是错的，因为更新后的 `prev2` 已经不是原来的 `dp[i-2]` 了。

必须先算 `cur`，再更新：

```java
int cur = Math.max(prev1, prev2 + nums[i]);
prev2 = prev1;
prev1 = cur;
```

---

## 七、面试中怎么讲最清楚

推荐表达顺序：

### 第一步：说决策

“对于每个位置，我只有偷和不偷两种选择。”

### 第二步：定义状态

“令 `dp[i]` 表示考虑前 `i` 间房屋时的最大收益。”

更准确一些可以说：

“令 `dp[i]` 表示从 `0` 到 `i` 的房屋中，能获得的最大金额。”

### 第三步：写转移

“如果不偷第 `i` 间，则收益为 `dp[i-1]`；如果偷第 `i` 间，则收益为 `dp[i-2] + nums[i]`，因此：
`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`。”

### 第四步：给边界

- `dp[0] = nums[0]`
- `dp[1] = max(nums[0], nums[1])`

### 第五步：优化空间

“由于每次只用到前两个状态，可以压缩到 `O(1)` 空间。”

这一套表达非常标准，面试官很容易跟上。

## 八、实际工作中的建议

这道题在业务里不一定以“偷房子”的形式出现，但它代表一种常见抽象：

> 在一维序列上做选择，且相邻选择存在冲突，求最大收益。

实际常见变形包括：

1. **任务调度**
   - 相邻时间段任务冲突
   - 不能同时选相邻任务，求最大收益

2. **资源分配**
   - 相邻资源位互斥
   - 选择某个资源会导致相邻资源不可用

3. **广告投放/优惠策略**
   - 相邻位置互相干扰
   - 在约束下做收益最大化

### 工程实践建议

- 如果只要结果，优先写 **状态压缩**
- 如果还需要恢复具体选择了哪些位置，使用 **DP 数组**
- 如果题目后面可能扩展成环形、树形结构，建议一开始就把“选/不选”的状态思想想清楚

## 九、和后续题目的联系

### 1. 力扣 213：打家劫舍 II

区别：房屋首尾相连，形成环。

处理方式：

- 要么不偷第一间，求 `[1...n-1]`
- 要么不偷最后一间，求 `[0...n-2]`
- 两者取最大值

本质上就是把环拆成两个线性问题。

---

### 2. 力扣 337：打家劫舍 III

区别：房屋排成了一棵树。

处理方式：

- 对每个节点维护两个状态：
  - 偷当前节点
  - 不偷当前节点

这就是把本题的一维 DP 扩展到树形 DP。

## 十、推荐掌握的最终版本

如果是面试实战，最推荐写下面这个版本：**简洁、正确、空间最优、表达标准**。

```java
import java.util.Arrays;

class Solution {
    public int rob(int[] nums) {
        int n = nums.length;

        if (n == 0) {
            return 0;
        }
        if (n == 1) {
            return nums[0];
        }

        int prev2 = nums[0]; // dp[0]
        int prev1 = Math.max(nums[0], nums[1]); // dp[1]

        for (int i = 2; i < n; i++) {
            int cur = Math.max(prev1, prev2 + nums[i]); // 选或不选当前房屋
            prev2 = prev1; // 滚动更新前两个状态
            prev1 = cur;
        }

        return prev1;
    }
}
```

## 十一、如果想写得更稳一些，可以用统一模板

这个模板甚至能避免单独处理很多边界：

```java
import java.util.Arrays;

class Solution {
    public int rob(int[] nums) {
        int pre = 0; // 相当于 dp[i-2]
        int cur = 0; // 相当于 dp[i-1]

        for (int num : nums) {
            int next = Math.max(cur, pre + num); // 当前最优
            pre = cur; // 滚动到下一轮
            cur = next;
        }

        return cur;
    }
}
```

### 为什么这个版本很好

假设一开始：

- `pre = 0`
- `cur = 0`

然后遍历每个房屋：

- 当前房屋不偷：收益还是 `cur`
- 当前房屋偷：收益是 `pre + num`

于是：

```java
next = Math.max(cur, pre + num);
```

这个写法天然兼容：

- 空数组
- 只有一个元素
- 多个元素

代码量非常小，面试里很受欢迎。

## 十二、总结

### 状态转移核心

\[
dp[i] = \max(dp[i-1], dp[i-2] + nums[i])
\]

### 解法对比

| 解法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|---|---:|---:|---|
| 记忆化搜索 | `O(n)` | `O(n)` | 便于理解递归决策树 |
| DP 数组 | `O(n)` | `O(n)` | 最标准，容易写对，方便扩展路径恢复 |
| 状态压缩 | `O(n)` | `O(1)` | 最优写法，面试和工程都常用 |

### 一句话理解

这题的本质就是：

> 每个位置都在做“选当前，还是不选当前”的决策，而一旦选了当前，就必须放弃前一个位置。

如果你把这道题彻底吃透，后面的：

- 打家劫舍 II
- 打家劫舍 III
- 最大不相邻子序列和
- 一些带互斥约束的线性 DP

都会顺很多。
