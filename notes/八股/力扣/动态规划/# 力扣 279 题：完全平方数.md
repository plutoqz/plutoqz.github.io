# 力扣 279 题：完全平方数

## 题目描述

给定正整数 `n`，找到若干个完全平方数（比如 `1, 4, 9, 16, ...`）使得它们的和等于 `n`。你需要让组成和的完全平方数的个数最少 [1][2]。

**示例：**

-   输入：n = 12，输出：3（12 = 4 + 4 + 4）
    
-   输入：n = 13，输出：2（13 = 4 + 9）
    

**约束：** `1 <= n <= 10^4`

---

## 解法一：动态规划（基础版）

### 算法思路

这是最直观的解法，本质是**完全背包问题**[1][2]。

-   `dp[i]` 表示组成数字 `i` 所需的最少完全平方数个数
    
-   状态转移方程：`dp[i] = min(dp[i - j*j] + 1)`，其中 `j*j <= i`
    
-   初始状态：`dp[0] = 0`
    

### 代码实现

```java
import java.util.Arrays;

class Solution {
    public int numSquares(int n) {
        // dp[i] 表示组成数字 i 的完全平方数的最小个数
        int[] dp = new int[n + 1];
        // 初始化为最大值
        Arrays.fill(dp, Integer.MAX_VALUE);
        // 基础情况：组成0需要0个完全平方数
        dp[0] = 0;
        
        // 预先生成所有小于等于n的完全平方数
        int maxSquareIndex = (int) Math.sqrt(n) + 1;
        int[] squareNums = new int[maxSquareIndex];
        for (int i = 1; i < maxSquareIndex; i++) {
            squareNums[i] = i * i;  // 生成平方数 1, 4, 9, 16...
        }
        
        // 自底向上计算每个数字的最少平方数个数
        for (int i = 1; i <= n; i++) {
            // 遍历所有不大于i的完全平方数
            for (int s = 1; s < maxSquareIndex; s++) {
                if (i < squareNums[s]) {
                    break;  // 剪枝：平方数大于当前数字，后续也不用计算
                }
                // 状态转移：使用当前平方数后，剩余部分的最优解+1
                dp[i] = Math.min(dp[i], dp[i - squareNums[s]] + 1);
            }
        }
        
        return dp[n];
    }
}
```

### 复杂度分析

-   **时间复杂度：** O(n × √n)
    
-   **空间复杂度：** O(n)
    

### 适用场景

-   面试中容易想到，适合展示动态规划基本功
    
-   n 较小时性能可接受

## 解法二：动态规划（优化版）

### 算法思路

与解法一本质相同，但**交换循环顺序**，先遍历平方数再遍历数字 [2]。

-   外层循环遍历每个完全平方数
    
-   内层循环更新所有能被该平方数影响的数字
    

### 代码实现

```java
import java.util.Arrays;

class Solution {
    public int numSquares(int n) {
        // dp[i] 初始化为最坏情况：全部用1组成，需要i个
        int[] dp = new int[n + 1];
        for (int i = 0; i <= n; i++) {
            dp[i] = i;  // 最坏情况：全部用1^2组成
        }
        
        // 外层遍历每个完全平方数
        for (int i = 2; i * i <= n; i++) {
            int square = i * i;  // 当前完全平方数
            // 内层更新所有能被当前平方数优化的数字
            for (int j = square; j <= n; j++) {
                // 使用当前平方数是否更优
                dp[j] = Math.min(dp[j], dp[j - square] + 1);
            }
        }
        
        return dp[n];
    }
}
```

### 复杂度分析

-   **时间复杂度：** O(n × √n)
    
-   **空间复杂度：** O(n)
    

### 适用场景

-   代码更简洁
    
-   实际运行可能略快于解法一（缓存友好）

## 解法三：BFS（广度优先搜索）

### 算法思路

将问题转化为**图的最短路径问题**[1]：

-   每个数字是图中的一个节点
    
-   如果两个数字相差一个完全平方数，则它们之间有边
    
-   求从 `n` 到 `0` 的最短路径长度
    

使用 BFS 逐层搜索，第一次遇到完全平方数时的层数即为答案。

### 代码实现

```java
import java.util.*;

class Solution {
    public int numSquares(int n) {
        // 生成所有小于等于n的完全平方数
        List<Integer> squares = new ArrayList<>();
        for (int i = 1; i * i <= n; i++) {
            squares.add(i * i);
        }
        
        // BFS队列，存储当前层需要处理的余数
        Queue<Integer> queue = new LinkedList<>();
        queue.offer(n);
        
        // 记录已访问的数字，避免重复计算
        Set<Integer> visited = new HashSet<>();
        visited.add(n);
        
        int level = 0;  // 记录当前层数（即完全平方数的个数）
        
        while (!queue.isEmpty()) {
            level++;  // 进入下一层
            int size = queue.size();
            
            // 处理当前层的所有节点
            for (int i = 0; i < size; i++) {
                int remainder = queue.poll();
                
                // 尝试减去每个完全平方数
                for (int square : squares) {
                    int next = remainder - square;
                    
                    if (next == 0) {
                        return level;  // 找到答案，返回层数
                    }
                    
                    if (next > 0 && !visited.contains(next)) {
                        visited.add(next);
                        queue.offer(next);
                    }
                    
                    if (next < 0) {
                        break;  // 平方数已大于余数，后续也不用尝试
                    }
                }
            }
        }
        
        return level;
    }
}
```

### 复杂度分析

-   **时间复杂度：** O(n × √n)，但实际运行通常比DP快
    
-   **空间复杂度：** O(n)
    

### 适用场景

-   需要找到**最少个数**的问题，BFS天然适合
    
-   实际面试中展示多种解题思路的加分项

## 解法四：数学定理（最优解）⭐

### 算法思路

基于**拉格朗日四平方和定理**和**勒让德三平方和定理**[1][2]：

1.  **四平方和定理**：任何正整数都可以表示为最多4个完全平方数之和
    
2.  **三平方和定理**：当且仅当 `n ≠ 4^k × (8m + 7)` 时，n 可以表示为3个完全平方数之和
    

**判断流程：**

1.  如果 n 本身是完全平方数 → 返回 1
    
2.  如果 n 满足 `n = 4^k × (8m + 7)` 形式 → 返回 4
    
3.  如果 n 可以表示为两个完全平方数之和 → 返回 2
    
4.  否则 → 返回 3
    

### 代码实现

```java
class Solution {
    public int numSquares(int n) {
        // 步骤1：检查是否本身就是完全平方数
        if (isPerfectSquare(n)) {
            return 1;
        }
        
        // 步骤2：检查是否满足四平方定理的特殊形式 n = 4^k * (8m + 7)
        // 先不断除以4，直到不能整除
        int temp = n;
        while (temp % 4 == 0) {
            temp /= 4;
        }
        // 检查是否满足 8m + 7 的形式
        if (temp % 8 == 7) {
            return 4;
        }
        
        // 步骤3：检查是否可以表示为两个完全平方数之和
        // 枚举第一个平方数，检查剩余部分是否为完全平方数
        for (int i = 1; i * i <= n; i++) {
            if (isPerfectSquare(n - i * i)) {
                return 2;
            }
        }
        
        // 步骤4：根据三平方和定理，剩余情况答案为3
        return 3;
    }
    
    // 辅助函数：判断一个数是否为完全平方数
    private boolean isPerfectSquare(int n) {
        int sqrt = (int) Math.sqrt(n);
        return sqrt * sqrt == n;
    }
}
```

### 复杂度分析

-   **时间复杂度：** O(√n)，主要是步骤3的枚举
    
-   **空间复杂度：** O(1)
    

### 适用场景

-   **最优解法**，时间和空间效率最高
    
-   适合对数学有深入理解的候选人展示
    
-   实际生产环境中追求极致性能时使用

## 面试与实战建议

### 面试场景

1.  **优先展示动态规划解法**
    
    -   这是面试官最期望看到的标准解法
        
    -   能体现对背包问题的理解
        
    -   代码容易解释，思路清晰
        
2.  **主动提及BFS思路**
    
    -   展示多角度思考能力
        
    -   说明"最少个数"问题BFS天然适合
        
3.  **数学解法作为加分项**
    
    -   如果知道四平方和定理，可以提及
        
    -   不必强求完整实现，但知道存在更优解体现知识广度
        

### 实际工作场景

1.  **数据规模较小时（n ≤ 10^4）**
    
    -   动态规划足够，代码易维护
        
    -   团队其他成员容易理解
        
2.  **数据规模较大或性能敏感**
    
    -   使用数学定理解法
        
    -   注意添加注释说明数学原理
        
3.  **扩展场景**
    
    -   如果完全平方数列表可变，DP更灵活
        
    -   如果需要记录具体组合方案，BFS更方便回溯
        

### 关键技巧总结

1.  **完全背包问题识别**：物品可重复使用，求最小/最大值
    
2.  **BFS求最短路径**：求"最少步数"类问题的通用思路
    
3.  **数学定理应用**：某些问题存在数学捷径，平时需积累
    
4.  **剪枝优化**：无论哪种解法，及时剪枝能大幅提升性能
