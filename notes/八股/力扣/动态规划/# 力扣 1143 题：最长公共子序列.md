# 力扣 1143 题：最长公共子序列

## 题目描述

给定两个字符串 `text1` 和 `text2`，返回这两个字符串的最长公共子序列的长度。

**子序列**指的是：在不改变字符相对顺序的情况下，删除某些字符（也可以不删除）后得到的新字符串。

例如：

-   `"abcde"` 的子序列有 `"ace"`、`"abc"`、`"de"` 等
    
-   `"aec"` 不是 `"abcde"` 的子序列，因为相对顺序变了
    

示例：

```text
输入：text1 = "abcde", text2 = "ace"
输出：3
解释：最长公共子序列是 "ace"，长度为 3
```

---

## 一、核心思路

这道题是经典的二维动态规划题。

关键在于定义状态：

设 `dp[i][j]` 表示：

-   `text1` 的前 `i` 个字符
    
-   和 `text2` 的前 `j` 个字符
    

它们的**最长公共子序列长度**。

这里的“前 i 个字符”是一个非常常见的 DP 定义方式，可以让边界更自然。

---

## 二、状态转移分析

假设当前考虑：

-   `text1.charAt(i - 1)`
    
-   `text2.charAt(j - 1)`
    

因为 `dp` 下标通常比字符串下标多 1，所以字符位置要写成 `i - 1` 和 `j - 1`。

### 情况 1：两个字符相同

如果：

```java
text1.charAt(i - 1) == text2.charAt(j - 1)
```

说明这两个字符可以作为公共子序列的最后一个字符，因此：

```java
dp[i][j] = dp[i - 1][j - 1] + 1
```

含义是：

-   前面的最长公共子序列长度来自两边都少一个字符的情况
    
-   再加上当前这个相同字符
    

---

### 情况 2：两个字符不同

如果当前字符不同，那么这两个字符不能同时作为公共子序列的末尾。

此时有两种选择：

1.  不使用 `text1` 当前字符<br>即看 `dp[i - 1][j]`
    
2.  不使用 `text2` 当前字符<br>即看 `dp[i][j - 1]`
    

取较大值：

```java
dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1])
```

---

## 三、为什么不需要 `dp[i - 1][j - 1]` 参与取最大值

很多人第一次写时会疑惑：

字符不相等时，为什么不是：

```java
dp[i][j] = Math.max(dp[i - 1][j - 1], Math.max(dp[i - 1][j], dp[i][j - 1]))
```

实际上没必要。

因为：

-   `dp[i - 1][j]` 一定不会小于 `dp[i - 1][j - 1]`
    
-   `dp[i][j - 1]` 也一定不会小于 `dp[i - 1][j - 1]`
    

原因是：多考虑一个字符，最长公共子序列长度不可能变小。

所以只需要：

```java
dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1])
```

## 四、解法一：二维动态规划

### 适用场景

-   最常规写法
    
-   最适合面试讲解
    
-   最容易推导和 debug
    
-   如果后面要恢复具体的公共子序列内容，也更方便从二维表回溯
    

### Java 代码

```java
import java.util.*;

class Solution {
    public int longestCommonSubsequence(String text1, String text2) {
        int m = text1.length();
        int n = text2.length();

        // dp[i][j] 表示 text1 前 i 个字符和 text2 前 j 个字符的最长公共子序列长度
        int[][] dp = new int[m + 1][n + 1];

        // 从 1 开始遍历，dp[0][*] 和 dp[*][0] 默认为 0，表示有一个串为空
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                // 当前两个字符相同
                if (text1.charAt(i - 1) == text2.charAt(j - 1)) {
                    // 可以在前一个状态基础上加 1
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                } else {
                    // 当前字符不同，只能舍弃其中一个字符，取较大值
                    dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
                }
            }
        }

        // 最终答案是两个完整字符串的最长公共子序列长度
        return dp[m][n];
    }
}
```

## 五、手动推演一个例子

以：

```text
text1 = "abcde"
text2 = "ace"
```

为例。

构造 `dp[6][4]`，因为长度分别是 5 和 3，所以多一行一列表示空串。

### 初始状态

第一行、第一列都为 0：

```text
      ""  a  c  e
""    0   0  0  0
a     0
b     0
c     0
d     0
e     0
```

### 填表过程

#### i = 1，text1[0] = 'a'

-   j = 1，text2[0] = 'a'，相同<br>`dp[1][1] = dp[0][0] + 1 = 1`
    
-   j = 2，text2[1] = 'c'，不同<br>`dp[1][2] = max(dp[0][2], dp[1][1]) = 1`
    
-   j = 3，text2[2] = 'e'，不同<br>`dp[1][3] = 1`
    

#### i = 2，text1[1] = 'b'

-   和 `a/c/e` 都不相同
    
-   这一行结果都保持为 1
    

#### i = 3，text1[2] = 'c'

-   j = 1，不同，`dp[3][1] = 1`
    
-   j = 2，相同，`dp[3][2] = dp[2][1] + 1 = 2`
    
-   j = 3，不同，`dp[3][3] = max(dp[2][3], dp[3][2]) = 2`
    

#### i = 4，text1[3] = 'd'

-   都不同
    
-   结果延续
    

#### i = 5，text1[4] = 'e'

-   j = 3，相同，`dp[5][3] = dp[4][2] + 1 = 3`
    

最终答案：

```text
dp[5][3] = 3
```

## 六、解法二：滚动数组优化

二维 DP 的状态转移只依赖：

-   左上 `dp[i - 1][j - 1]`
    
-   上方 `dp[i - 1][j]`
    
-   左方 `dp[i][j - 1]`
    

所以可以把空间从 `O(mn)` 优化到 `O(n)`。

---

### 适用场景

-   字符串长度较大，想节省空间
    
-   面试中如果先讲二维 DP，再主动优化，会很加分
    
-   只求长度，不需要恢复具体子序列时非常合适
    

---

### 关键技巧

在一维数组里：

-   `dp[j]` 在更新前，表示上一行的 `dp[i - 1][j]`
    
-   `dp[j - 1]` 在更新后，表示当前行的 `dp[i][j - 1]`
    
-   还需要一个变量 `prev` 保存更新前的 `dp[i - 1][j - 1]`
    

这个 `prev` 是很多人最容易写错的地方。

---

### Java 代码

```java
import java.util.*;

class Solution {
    public int longestCommonSubsequence(String text1, String text2) {
        int m = text1.length();
        int n = text2.length();

        // 为了减少空间，可以让较短字符串作为列
        if (n > m) {
            return longestCommonSubsequence(text2, text1);
        }

        // dp[j] 表示当前处理到某一行时，前 i 个字符和 text2 前 j 个字符的 LCS 长度
        int[] dp = new int[n + 1];

        for (int i = 1; i <= m; i++) {
            int prev = 0; // 表示左上角 dp[i - 1][j - 1]
            for (int j = 1; j <= n; j++) {
                int temp = dp[j]; // 先保存旧的 dp[j]，它是上方 dp[i - 1][j]

                if (text1.charAt(i - 1) == text2.charAt(j - 1)) {
                    // 相同则取左上角 + 1
                    dp[j] = prev + 1;
                } else {
                    // 不同则取上方和左方的较大值
                    dp[j] = Math.max(dp[j], dp[j - 1]);
                }

                // 更新 prev，供下一列使用
                prev = temp;
            }
        }

        return dp[n];
    }
}
```

## 七、解法三：递归 + 记忆化搜索

这是从“定义问题”角度最自然的一种写法。

定义函数：

```java
dfs(i, j)
```

表示：

-   `text1` 从下标 `i` 开始
    
-   `text2` 从下标 `j` 开始
    

它们的最长公共子序列长度。

### 状态转移

-   如果某一个串走到末尾，返回 0
    
-   如果 `text1[i] == text2[j]`<br>`dfs(i, j) = 1 + dfs(i + 1, j + 1)`
    
-   否则<br>`dfs(i, j) = max(dfs(i + 1, j), dfs(i, j + 1))`
    

如果不加记忆化，会有大量重复计算，时间复杂度会爆炸。

---

### 适用场景

-   更适合理解“问题分解”
    
-   写起来思路清晰
    
-   面试里如果你擅长递归表达，可以先讲这个，再转成 DP
    

---

### Java 代码

```java
import java.util.*;

class Solution {
    private int[][] memo;
    private String text1;
    private String text2;

    public int longestCommonSubsequence(String text1, String text2) {
        int m = text1.length();
        int n = text2.length();

        this.text1 = text1;
        this.text2 = text2;
        this.memo = new int[m][n];

        // 用 -1 表示还没计算过
        for (int i = 0; i < m; i++) {
            Arrays.fill(memo[i], -1);
        }

        return dfs(0, 0);
    }

    private int dfs(int i, int j) {
        // 只要有一个串走到末尾，就没有公共子序列了
        if (i == text1.length() || j == text2.length()) {
            return 0;
        }

        // 已经计算过，直接返回
        if (memo[i][j] != -1) {
            return memo[i][j];
        }

        int ans;
        if (text1.charAt(i) == text2.charAt(j)) {
            // 当前字符相同，可以同时选择
            ans = 1 + dfs(i + 1, j + 1);
        } else {
            // 当前字符不同，尝试跳过 text1[i] 或 text2[j]
            ans = Math.max(dfs(i + 1, j), dfs(i, j + 1));
        }

        memo[i][j] = ans;
        return ans;
    }
}
```

## 八、三种解法对比

| 解法  | 时间复杂度 | 空间复杂度 | 优点  | 缺点  |
| --- | --- | --- | --- | --- |
| 递归 + 记忆化 | `O(mn)` | `O(mn)` | 思维自然，易于从问题定义出发 | 递归有栈开销 |
| 二维 DP | `O(mn)` | `O(mn)` | 最稳、最经典、最好理解 | 空间较大 |
| 滚动数组 DP | `O(mn)` | `O(n)` | 节省空间 | 写法更容易出错 |

> 按面试表达顺序，通常建议：
> 
> 1.  先讲递归定义
>     
> 2.  再讲二维 DP 转化
>     
> 3.  最后讲滚动数组优化
>     

不过按工程实现的稳妥程度，最推荐二维 DP；按空间表现，最推荐滚动数组。

## 九、边界条件与易错点

### 1\. `dp` 数组大小为什么是 `m + 1` 和 `n + 1`

因为需要多一行一列表示“空字符串”的情况。

这样：

-   `dp[0][j] = 0`
    
-   `dp[i][0] = 0`
    

都天然成立，不需要单独写很多判断。

---

### 2\. 字符访问为什么是 `i - 1` 和 `j - 1`

因为：

-   `dp[i][j]` 表示前 `i` 个字符
    
-   Java 字符串下标从 0 开始
    

所以第 `i` 个字符的下标是 `i - 1`。

---

### 3\. 不要把“子序列”写成“子串”

这道题是**最长公共子序列**，不是**最长公共子串**。

两者区别非常重要：

-   子序列：可以不连续，只要相对顺序不变
    
-   子串：必须连续
    

对应的 DP 转移完全不同。

---

### 4\. 滚动数组里 `prev` 更新顺序不能错

错误写法通常是：

-   先更新了 `prev`
    
-   再用它去算当前值
    

这样会把“左上角”覆盖掉。

正确顺序是：

1.  先保存 `dp[j]` 到 `temp`
    
2.  用 `prev` 参与当前计算
    
3.  最后令 `prev = temp`

## 十、如果题目要求输出具体的最长公共子序列

1143 只要求长度，但实际面试里经常会追问：

> 如果要返回具体的一个最长公共子序列，怎么办？

做法是：

1.  先用二维 DP 求长度表
    
2.  再从 `dp[m][n]` 倒推回去
    

### 回溯规则

从 `(m, n)` 开始：

-   如果 `text1[i - 1] == text2[j - 1]`<br>说明这个字符在答案中，加入结果，然后 `i--, j--`
    
-   否则：
    
    -   如果 `dp[i - 1][j] >= dp[i][j - 1]`，则 `i--`
        
    -   否则 `j--`
        

最后得到的是逆序，要反转一下。

---

### Java 代码：返回一个最长公共子序列

```java
import java.util.*;

class Solution {
    public String getOneLCS(String text1, String text2) {
        int m = text1.length();
        int n = text2.length();

        int[][] dp = new int[m + 1][n + 1];

        // 先计算 LCS 长度表
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (text1.charAt(i - 1) == text2.charAt(j - 1)) {
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                } else {
                    dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
                }
            }
        }

        // 从右下角回溯具体序列
        StringBuilder sb = new StringBuilder();
        int i = m, j = n;

        while (i > 0 && j > 0) {
            if (text1.charAt(i - 1) == text2.charAt(j - 1)) {
                // 当前字符属于某个 LCS
                sb.append(text1.charAt(i - 1));
                i--;
                j--;
            } else if (dp[i - 1][j] >= dp[i][j - 1]) {
                // 往较大的方向走
                i--;
            } else {
                j--;
            }
        }

        // 回溯得到的是逆序，需要翻转
        return sb.reverse().toString();
    }
}
```

## 十一、面试中怎么讲这题

一个比较完整、清晰的表达模板如下：

### 1\. 先说问题性质

这是一个典型的二维动态规划问题，因为状态由两个字符串的位置共同决定。

### 2\. 再定义状态

`dp[i][j]` 表示 `text1` 前 `i` 个字符与 `text2` 前 `j` 个字符的最长公共子序列长度。

### 3\. 分情况讨论转移

-   若 `text1[i - 1] == text2[j - 1]`，则<br>`dp[i][j] = dp[i - 1][j - 1] + 1`
    
-   否则<br>`dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])`
    

### 4\. 说明初始化

`dp[0][j] = dp[i][0] = 0`，表示有一个字符串为空时，公共子序列长度为 0。

### 5\. 给出复杂度

-   时间复杂度：`O(mn)`
    
-   空间复杂度：`O(mn)`<br>可优化为 `O(n)`
    

这样表述一般会让面试官觉得你不是“背模板”，而是真的理解了状态定义和转移来源。

## 十三、面试和工作场景建议

### 面试建议

1.  **优先讲二维 DP**
    
    -   最稳
        
    -   不容易错
        
    -   方便展示你对状态转移的理解
        
2.  **主动补充滚动数组优化**
    
    -   能体现你对空间优化有意识
        
    -   但要确保自己真的能写对
        
3.  **区分 LCS 和最长公共子串**
    
    -   这是高频考点
        
    -   面试官很喜欢顺手追问
        
4.  **准备好“如何输出具体序列”的追问**
    
    -   很多面试到这里不会停在“只求长度”
        

---

### 工作建议

1.  **工程里先考虑可维护性**
    
    -   如果数据规模不大，优先二维 DP，代码更直观
        
    -   不要为了省一点空间牺牲太多可读性
        
2.  **大输入时注意内存**
    
    -   如果两个字符串都很长，二维数组可能会占用较多内存
        
    -   此时滚动数组更合适
        
3.  **如果需要输出路径/方案，保留完整状态**
    
    -   滚动数组虽然省空间，但不方便回溯具体答案
        
    -   这种时候二维 DP 更实用

## 十四、最终推荐写法

如果你是在：

-   **刷题**
    
-   **面试**
    
-   **学习 DP**
    

最推荐先掌握下面这个二维 DP 版本。

```java
import java.util.*;

class Solution {
    public int longestCommonSubsequence(String text1, String text2) {
        int m = text1.length();
        int n = text2.length();

        // dp[i][j] 表示 text1 前 i 个字符和 text2 前 j 个字符的最长公共子序列长度
        int[][] dp = new int[m + 1][n + 1];

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (text1.charAt(i - 1) == text2.charAt(j - 1)) {
                    // 当前字符相同，可以接在前面的公共子序列后面
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                } else {
                    // 当前字符不同，只能舍弃一个字符，取两种情况中的较大值
                    dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
                }
            }
        }

        return dp[m][n];
    }
}
```
