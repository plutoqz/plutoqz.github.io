# 力扣 72 题：编辑距离

## 题目理解

给定两个字符串 `word1` 和 `word2`，你需要计算将 `word1` 转换成 `word2` 所使用的最少操作数。

允许的操作只有三种：

1.  插入一个字符
    
2.  删除一个字符
    
3.  替换一个字符
    

这道题是经典动态规划题，核心在于：

-   每一步只关心当前两个前缀的最优解
    
-   当前状态可以由更短前缀的状态转移而来
    
-   属于“两个字符串匹配类 DP”中的代表题
    

---

## 一、动态规划本质

设：

-   `dp[i][j]` 表示：`word1` 的前 `i` 个字符，转换成 `word2` 的前 `j` 个字符，所需要的最少操作数
    

这里的“前 `i` 个字符”指的是子串：

-   `word1[0...i-1]`
    
-   `word2[0...j-1]`
    

---

## 二、状态转移分析

假设当前处理到：

-   `word1` 的第 `i` 个字符，实际下标是 `i - 1`
    
-   `word2` 的第 `j` 个字符，实际下标是 `j - 1`
    

### 1）当两个字符相等时

如果：

```java
word1.charAt(i - 1) == word2.charAt(j - 1)
```

那么最后一个字符不需要额外操作，直接继承前面的结果：

```java
dp[i][j] = dp[i - 1][j - 1];
```

因为最后一个字符已经匹配好了。

---

### 2）当两个字符不相等时

就要考虑三种操作：

#### a. 删除

把 `word1` 当前字符删掉：

```java
dp[i][j] = dp[i - 1][j] + 1;
```

含义：

-   先把 `word1` 前 `i-1` 个字符变成 `word2` 前 `j` 个字符
    
-   再删除 `word1` 的第 `i` 个字符
    

---

#### b. 插入

往 `word1` 后面插入一个字符，使其匹配 `word2[j-1]`：

```java
dp[i][j] = dp[i][j - 1] + 1;
```

含义：

-   先让 `word1` 前 `i` 个字符变成 `word2` 前 `j-1` 个字符
    
-   再插入 `word2[j-1]`
    

---

#### c. 替换

把 `word1[i-1]` 替换成 `word2[j-1]`：

```java
dp[i][j] = dp[i - 1][j - 1] + 1;
```

含义：

-   先让前面部分匹配
    
-   再替换最后一个字符
    

---

### 综合转移方程

如果字符相等：

```java
dp[i][j] = dp[i - 1][j - 1];
```

如果字符不等：

```java
dp[i][j] = Math.min(
    Math.min(dp[i - 1][j], dp[i][j - 1]),
    dp[i - 1][j - 1]
) + 1;
```

---

## 三、边界条件

### 1）`dp[0][0] = 0`

空串转空串，不需要操作。

### 2）`dp[i][0] = i`

把 `word1` 前 `i` 个字符变成空串，只能全部删除。

### 3）`dp[0][j] = j`

把空串变成 `word2` 前 `j` 个字符，只能不断插入。

这三个边界是整道题最重要的基础。

---

## 四、解法一：二维动态规划

### 适用场景

-   面试中最推荐先写这个版本
    
-   逻辑最清晰
    
-   不容易出错
    
-   适合第一次掌握编辑距离
    

---

### Java 代码

```java
import java.util.*;

class Solution {
    public int minDistance(String word1, String word2) {
        int m = word1.length();
        int n = word2.length();

        // dp[i][j] 表示 word1 前 i 个字符转换成 word2 前 j 个字符的最少操作数
        int[][] dp = new int[m + 1][n + 1];

        // 初始化：word1 前 i 个字符转换为空串，需要删除 i 次
        for (int i = 0; i <= m; i++) {
            dp[i][0] = i;
        }

        // 初始化：空串转换成 word2 前 j 个字符，需要插入 j 次
        for (int j = 0; j <= n; j++) {
            dp[0][j] = j;
        }

        // 按前缀长度逐步计算
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                // 如果当前字符相同，则不需要新增操作
                if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
                    dp[i][j] = dp[i - 1][j - 1];
                } else {
                    // 三种操作：删除、插入、替换，取最小值后 +1
                    dp[i][j] = Math.min(
                        Math.min(dp[i - 1][j], dp[i][j - 1]),
                        dp[i - 1][j - 1]
                    ) + 1;
                }
            }
        }

        return dp[m][n];
    }
}
```

## 五、二维 DP 推演示例

以：

```text
word1 = "horse"
word2 = "ros"
```

为例。

最终答案是 `3`，一种操作方式：

1.  `horse` -> `rorse`，替换 `h` 为 `r`
    
2.  `rorse` -> `rose`，删除一个 `r`
    
3.  `rose` -> `ros`，删除 `e`
    

---

### DP 表含义

`dp[i][j]` 表示 `word1` 前 `i` 个字符到 `word2` 前 `j` 个字符的最小编辑距离。

我们构造 `(m+1) x (n+1)` 表。

---

### 初始化

```text
      ""  r  o  s
""    0  1  2  3
h     1
o     2
r     3
s     4
e     5
```

---

### 逐步填表思路

#### `i = 1`，字符 `h`

-   `j = 1`，`h != r`
    
    -   删除：`dp[0][1] + 1 = 2`
        
    -   插入：`dp[1][0] + 1 = 2`
        
    -   替换：`dp[0][0] + 1 = 1`
        
    -   所以 `dp[1][1] = 1`
        
-   `j = 2`，`h != o`
    
    -   `dp[1][2] = 2`
        
-   `j = 3`，`h != s`
    
    -   `dp[1][3] = 3`
        

---

#### `i = 2`，字符 `o`

-   `j = 1`，`o != r`
    
    -   `dp[2][1] = 2`
        
-   `j = 2`，`o == o`
    
    -   `dp[2][2] = dp[1][1] = 1`
        
-   `j = 3`，`o != s`
    
    -   `dp[2][3] = 2`
        

---

#### `i = 3`，字符 `r`

-   `j = 1`，`r == r`
    
    -   `dp[3][1] = dp[2][0] = 2`
        
-   `j = 2`，`r != o`
    
    -   `dp[3][2] = 2`
        
-   `j = 3`，`r != s`
    
    -   `dp[3][3] = 2`
        

---

#### `i = 4`，字符 `s`

-   `j = 1`，`s != r`
    
    -   `dp[4][1] = 3`
        
-   `j = 2`，`s != o`
    
    -   `dp[4][2] = 3`
        
-   `j = 3`，`s == s`
    
    -   `dp[4][3] = dp[3][2] = 2`
        

---

#### `i = 5`，字符 `e`

-   `j = 1`，`e != r`
    
    -   `dp[5][1] = 4`
        
-   `j = 2`，`e != o`
    
    -   `dp[5][2] = 4`
        
-   `j = 3`，`e != s`
    
    -   `dp[5][3] = 3`
        

最终：

```text
      ""  r  o  s
""    0  1  2  3
h     1  1  2  3
o     2  2  1  2
r     3  2  2  2
s     4  3  3  2
e     5  4  4  3
```

答案为：

```text
dp[5][3] = 3
```

## 六、解法二：一维动态规划（空间优化）

二维 DP 的状态转移只依赖：

-   当前行左边 `dp[i][j-1]`
    
-   上一行当前列 `dp[i-1][j]`
    
-   左上角 `dp[i-1][j-1]`
    

所以可以压缩成一维数组。

---

### 适用场景

-   需要优化空间复杂度时
    
-   面试中作为“进一步优化”非常加分
    
-   对 DP 依赖关系理解较深时使用
    

---

### 优化难点

一维压缩时，`dp[j]` 在不同时间点含义不同：

-   更新前：表示上一行的 `dp[i-1][j]`
    
-   更新后：表示当前行的 `dp[i][j]`
    

同时还需要一个变量记录左上角旧值 `dp[i-1][j-1]`。

---

### Java 代码

```java
import java.util.*;

class Solution {
    public int minDistance(String word1, String word2) {
        int m = word1.length();
        int n = word2.length();

        // dp[j] 表示当前处理到某一行时，word1 前 i 个字符到 word2 前 j 个字符的最小操作数
        int[] dp = new int[n + 1];

        // 初始化第 0 行：空串转换为 word2 前 j 个字符
        for (int j = 0; j <= n; j++) {
            dp[j] = j;
        }

        for (int i = 1; i <= m; i++) {
            // pre 表示左上角 dp[i-1][j-1]
            int pre = dp[0];
            // 当前行第 0 列：word1 前 i 个字符转换为空串
            dp[0] = i;

            for (int j = 1; j <= n; j++) {
                // temp 保存更新前的 dp[j]，也就是上一行当前列 dp[i-1][j]
                int temp = dp[j];

                if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
                    // 字符相同，继承左上角
                    dp[j] = pre;
                } else {
                    // dp[j]     -> 上方    dp[i-1][j]
                    // dp[j - 1] -> 左方    dp[i][j-1]
                    // pre       -> 左上角  dp[i-1][j-1]
                    dp[j] = Math.min(
                        Math.min(dp[j], dp[j - 1]),
                        pre
                    ) + 1;
                }

                // pre 更新为下一列所需的左上角
                pre = temp;
            }
        }

        return dp[n];
    }
}
```

## 七、时间复杂度与空间复杂度

### 解法一：二维 DP

-   时间复杂度：`O(m * n)`
    
-   空间复杂度：`O(m * n)`
    

### 解法二：一维 DP

-   时间复杂度：`O(m * n)`
    
-   空间复杂度：`O(n)`
    

其中：

-   `m = word1.length()`
    
-   `n = word2.length()`
    

---

## 八、为什么这题不能用贪心

很多人初看会想：

-   字符相同就跳过
    
-   不同就直接替换
    
-   或者优先删除 / 插入
    

这类贪心策略都不可靠，因为局部最优不一定导向全局最优。

### 例子

```text
word1 = "intention"
word2 = "execution"
```

某一步你觉得替换更优，但从整体来看，也许先删除再插入会带来后续更优匹配。

编辑距离问题的本质是：<br>当前决策会影响后续字符串对齐方式，因此必须枚举三种可能，并取全局最优，这正是动态规划擅长的地方。

---

## 九、面试中如何讲清楚这题

面试回答建议按下面顺序说：

### 1\. 明确定义状态

`dp[i][j]` 表示 `word1` 前 `i` 个字符转换成 `word2` 前 `j` 个字符的最少操作数。

### 2\. 解释最后一步分类

对于 `word1[i-1]` 和 `word2[j-1]`：

-   相等：不需要操作，来自 `dp[i-1][j-1]`
    
-   不等：考虑
    
    -   删除：`dp[i-1][j] + 1`
        
    -   插入：`dp[i][j-1] + 1`
        
    -   替换：`dp[i-1][j-1] + 1`
        

### 3\. 说明边界

-   `dp[i][0] = i`
    
-   `dp[0][j] = j`
    

### 4\. 说明遍历顺序

因为 `dp[i][j]` 依赖上、左、左上，所以按 `i` 从小到大，`j` 从小到大遍历。

### 5\. 再补充空间优化

如果时间够，可以主动说能压缩到一维。

这个讲法很标准，面试官通常会认可你对 DP 的理解是完整的。

## 十、常见错误

### 1）下标错位

`dp[i][j]` 对应的是前 `i`、前 `j` 个字符，所以取字符时必须用：

```java
word1.charAt(i - 1)
word2.charAt(j - 1)
```

不能直接写 `i`、`j`。

---

### 2）初始化漏掉第 0 行 / 第 0 列

这题最常见 bug 就是没有正确初始化：

```java
dp[i][0] = i;
dp[0][j] = j;
```

---

### 3）把插入和删除搞反

其实可以这样记：

-   `dp[i - 1][j] + 1`：删掉 `word1[i-1]`
    
-   `dp[i][j - 1] + 1`：向 `word1` 插入 `word2[j-1]`
    

很多人会在口头解释时混乱，但代码里只要根据状态含义推就不会错。

---

### 4）一维压缩时覆盖了左上角

如果直接覆盖 `dp[j]`，就会丢失上一行的数据。<br>必须用：

-   `pre` 保存左上角
    
-   `temp` 保存当前格更新前的旧值
    

---

## 十一、和其他字符串 DP 题的联系

这题和下面几类题很像：

### 1\. 最长公共子序列

-   都是两个字符串的二维 DP
    
-   都依赖左、上、左上
    

区别是：

-   编辑距离求最小操作数
    
-   LCS 求最长匹配长度
    

---

### 2\. 删除操作使两个字符串相同

本质上也是双字符串 DP，只是允许操作种类不同。

---

### 3\. 正则表达式匹配

也是字符串 DP，但状态设计和转移条件更复杂。

---

## 十二、实际工作中的意义

编辑距离在真实业务里非常常见，例如：

### 1）搜索纠错

用户输入：

-   `iphnoe`
    
-   `appel`
    

系统可以根据编辑距离判断其最接近的单词。

### 2）输入法联想 / 拼写检查

比较用户输入和词典中单词的“相似程度”。

### 3）模糊匹配

在日志检索、命令推荐、客服问答、OCR 文本纠错中，编辑距离常用于粗粒度文本相似判断。

不过在真实工业场景里，纯编辑距离通常只适合：

-   短字符串
    
-   字典规模不太大
    
-   需要可解释性的场景
    

如果文本更长、语义更复杂，一般会结合：

-   Trie
    
-   BK-Tree
    
-   倒排索引
    
-   向量检索
    
-   机器学习排序模型

## 十三、面试和工程实践建议

### 面试建议

1.  **先写二维 DP，确保正确**
    
    -   这是最稳的版本
        
    -   面试官最看重的是思路完整和边界正确
        
2.  **写完主动分析优化空间**
    
    -   表示你不只是会套模板，而是真的理解依赖关系
        
3.  **一定把三种操作和状态含义讲一致**
    
    -   很多人代码是对的，但口头解释混乱，会减分
        
4.  **拿一个小例子手推**
    
    -   比如 `horse -> ros`
        
    -   能体现你对转移方程的掌握
        

---

### 工程实践建议

1.  **如果只求距离值，用一维 DP 更实用**
    
    -   节省内存
        
2.  **如果需要恢复具体编辑路径，用二维 DP**
    
    -   因为可以回溯操作过程
        
3.  **超长字符串时要注意性能**
    
    -   `O(m * n)` 对很长文本开销很大
        
    -   可以考虑剪枝、带阈值的编辑距离、分块策略
        
4.  **注意 Unicode / 多字节字符问题**
    
    -   Java `charAt()` 对某些 Unicode 扩展字符并不总是语义级“一个字符”
        
    -   如果业务涉及 emoji、特殊语言字符，可能需要按 code point 处理

## 十四、如果面试官追问：如何输出具体操作序列

如果不仅要求最小编辑距离，还要输出具体操作步骤，可以在二维 DP 基础上回溯：

-   从 `dp[m][n]` 倒推到 `dp[0][0]`
    
-   如果字符相等，往左上走
    
-   如果当前值来自：
    
    -   `dp[i-1][j] + 1`，说明执行了删除
        
    -   `dp[i][j-1] + 1`，说明执行了插入
        
    -   `dp[i-1][j-1] + 1`，说明执行了替换
        

这是很多业务系统里真正需要的能力，比如文本纠错要告诉用户“改了哪里”。

下面给一个可回溯版本。

## 十五、扩展解法：二维 DP + 回溯输出操作路径

### 适用场景

-   不仅要最小距离，还要知道如何修改
    
-   工程里做拼写纠错、diff 展示时很有用
    

---

### Java 代码

```java
import java.util.*;

class Solution {
    public int minDistance(String word1, String word2) {
        int m = word1.length();
        int n = word2.length();

        int[][] dp = new int[m + 1][n + 1];

        for (int i = 0; i <= m; i++) {
            dp[i][0] = i;
        }
        for (int j = 0; j <= n; j++) {
            dp[0][j] = j;
        }

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
                    dp[i][j] = dp[i - 1][j - 1];
                } else {
                    dp[i][j] = Math.min(
                        Math.min(dp[i - 1][j], dp[i][j - 1]),
                        dp[i - 1][j - 1]
                    ) + 1;
                }
            }
        }

        return dp[m][n];
    }

    // 扩展方法：返回一种编辑操作路径
    public List<String> getEditPath(String word1, String word2) {
        int m = word1.length();
        int n = word2.length();

        int[][] dp = new int[m + 1][n + 1];

        for (int i = 0; i <= m; i++) {
            dp[i][0] = i;
        }
        for (int j = 0; j <= n; j++) {
            dp[0][j] = j;
        }

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
                    dp[i][j] = dp[i - 1][j - 1];
                } else {
                    dp[i][j] = Math.min(
                        Math.min(dp[i - 1][j], dp[i][j - 1]),
                        dp[i - 1][j - 1]
                    ) + 1;
                }
            }
        }

        List<String> operations = new ArrayList<>();
        int i = m, j = n;

        while (i > 0 || j > 0) {
            if (i > 0 && j > 0 && word1.charAt(i - 1) == word2.charAt(j - 1)) {
                // 字符相同，不需要操作，左上移动
                i--;
                j--;
            } else if (i > 0 && dp[i][j] == dp[i - 1][j] + 1) {
                // 来自删除
                operations.add("删除 word1 的字符 '" + word1.charAt(i - 1) + "'");
                i--;
            } else if (j > 0 && dp[i][j] == dp[i][j - 1] + 1) {
                // 来自插入
                operations.add("插入字符 '" + word2.charAt(j - 1) + "'");
                j--;
            } else if (i > 0 && j > 0 && dp[i][j] == dp[i - 1][j - 1] + 1) {
                // 来自替换
                operations.add("将 '" + word1.charAt(i - 1) + "' 替换为 '" + word2.charAt(j - 1) + "'");
                i--;
                j--;
            }
        }

        Collections.reverse(operations);
        return operations;
    }
}
```

## 十六、总结

这题的核心就一句话：

> `dp[i][j]` 表示两个字符串前缀的最小编辑距离，当前字符相等则继承左上角，不等则在删除、插入、替换三种操作中取最小值加一。

你需要真正掌握的点有：

1.  **状态定义**
    
2.  **三种操作如何映射到 DP 转移**
