# 力扣 300 题：最长递增子序列

## 题目概述

给你一个整数数组 `nums`，返回其中 **最长严格递增子序列** 的长度。

-   **子序列**：可以删除数组中的某些元素，也可以一个都不删，但不能改变剩余元素的相对顺序。
    
-   **严格递增**：后一个数必须 **大于** 前一个数，不能相等。
    

例如：

-   `nums = [10,9,2,5,3,7,101,18]`，答案是 `4`
    
-   一个最长递增子序列可以是 `[2,3,7,101]`
    

---

## 核心难点

这题最容易混淆的点有两个：

1.  **子序列不是子数组**
    
    -   子数组要求连续
        
    -   子序列不要求连续
        
2.  **严格递增**
    
    -   `1,2,2,3` 里，两个 `2` 不能同时选进严格递增序列
        

---

## 解法总览

按照“次优到最优”的顺序：

1.  **动态规划**：`O(n^2)`
    
    -   容易理解
        
    -   面试中适合先讲这个，体现扎实基本功
        
2.  **贪心 + 二分查找**：`O(n log n)`
    
    -   最优解
        
    -   高频面试写法
        
    -   需要理解其正确性，不只是背模板
        

---

## 解法一：动态规划 `O(n^2)`

### 适用场景

-   想先快速建立正确思路
    
-   数据规模不大
    
-   面试时作为第一层解法，便于自然过渡到最优解
    

---

### 思路

定义：

-   `dp[i]` 表示 **以** `nums[i]` **结尾** 的最长严格递增子序列长度
    

那么对于每个 `i`，我们去看前面的每个 `j`：

-   如果 `nums[j] < nums[i]`
    
-   说明 `nums[i]` 可以接在 `nums[j]` 后面
    
-   则有转移： `dp[i] = max(dp[i], dp[j] + 1)`
    

初始值：

-   每个元素自己都可以单独成为一个长度为 `1` 的递增子序列
    
-   所以 `dp[i] = 1`
    

最终答案：

-   `max(dp[i])`
    

---

### 状态转移公式

$$dp[i] = \max(dp[i], dp[j] + 1), \quad \text{当 } j < i \text{ 且 } nums[j] < nums[i]$$

---

### 代码实现（Java）

```java
import java.util.Arrays;

class Solution {
    public int lengthOfLIS(int[] nums) {
        int n = nums.length;
        int[] dp = new int[n];
        Arrays.fill(dp, 1); // 每个元素至少可以单独形成长度为 1 的递增子序列

        int ans = 1;

        for (int i = 0; i < n; i++) {
            for (int j = 0; j < i; j++) {
                // 只有前面的数比当前数小，才能接到当前数前面
                if (nums[j] < nums[i]) {
                    dp[i] = Math.max(dp[i], dp[j] + 1);
                }
            }
            // 维护全局最大值
            ans = Math.max(ans, dp[i]);
        }

        return ans;
    }
}
```

### 示例推演

以 `nums = [10,9,2,5,3,7,101,18]` 为例：

初始：

```text
dp = [1,1,1,1,1,1,1,1]
```

逐个计算：

-   `i = 0`，`10`，前面没人，`dp[0]=1`
    
-   `i = 1`，`9`，前面 `10` 不小于 `9`，`dp[1]=1`
    
-   `i = 2`，`2`，前面都不小于 `2`，`dp[2]=1`
    
-   `i = 3`，`5`
    
    -   `2 < 5`，所以 `dp[3] = dp[2] + 1 = 2`
        
-   `i = 4`，`3`
    
    -   `2 < 3`，所以 `dp[4] = 2`
        
-   `i = 5`，`7`
    
    -   `2 < 7`，可得 `2`
        
    -   `5 < 7`，可得 `3`
        
    -   `3 < 7`，也可得 `3`
        
    -   所以 `dp[5] = 3`
        
-   `i = 6`，`101`
    
    -   可以接在很多前面元素后，最长可得 `4`
        
-   `i = 7`，`18`
    
    -   也可以接在 `7` 后面，得到 `4`
        

最终：

```text
dp = [1,1,1,2,2,3,4,4]
```

答案为 `4`

---

### 复杂度分析

-   时间复杂度：`O(n^2)`
    
-   空间复杂度：`O(n)`
    

---

### 常见错误

#### 1\. 把“以 i 结尾”理解成“前 i 个元素中的答案”

这是最常见的定义混乱。

-   `dp[i]` 一定要定义清楚
    
-   这里是：**必须以** `nums[i]` **结尾**
    

这样状态转移才自然。

#### 2\. 条件写成 `nums[j] <= nums[i]`

题目要求是 **严格递增**，所以必须写：

```java
nums[j] < nums[i]
```

#### 3\. 忘记初始化为 1

因为每个元素自己都能形成长度为 1 的子序列。

## 解法二：贪心 + 二分查找 `O(n log n)`

### 适用场景

-   追求最优时间复杂度
    
-   面试进阶写法
    
-   数组长度较大时更合适
    

---

## 先理解贪心含义

我们维护一个数组 `tails`：

-   `tails[len]` 表示：**长度为** `len + 1` **的递增子序列中，结尾元素的最小可能值**
    

这个定义很关键。

它不是一个真实的 LIS 序列，而是一个“最优结尾记录表”。

为什么要让结尾尽可能小？

因为结尾越小，后面越容易接上更大的数，越有机会形成更长的递增子序列。

---

### 贪心策略

遍历每个数 `num`：

1.  在 `tails` 中找到第一个 **大于等于** `num` 的位置
    
2.  用 `num` 替换它
    
    -   因为更小的结尾更优
        
3.  如果 `num` 比 `tails` 中所有元素都大
    
    -   说明可以让最长长度加一
        
    -   直接追加到末尾
        

---

### 为什么是“找第一个 >= num 的位置”？

因为题目要求的是 **严格递增**。

-   如果当前 `num` 和某个结尾相等，那么不能让长度变长
    
-   只能替换掉那个位置，保持相同长度但让结尾不变或更优
    

这本质上是在做一个 **lower\_bound**。

---

## 代码实现（Java）

```java
import java.util.Arrays;

class Solution {
    public int lengthOfLIS(int[] nums) {
        int n = nums.length;
        int[] tails = new int[n]; // tails[i] 表示长度为 i + 1 的递增子序列的最小结尾
        int size = 0; // 当前已构造出的最长递增子序列长度

        for (int num : nums) {
            int left = 0;
            int right = size;

            // 在 [0, size) 中二分查找第一个 >= num 的位置
            while (left < right) {
                int mid = left + (right - left) / 2;
                if (tails[mid] < num) {
                    left = mid + 1;
                } else {
                    right = mid;
                }
            }

            // left 就是第一个 >= num 的位置
            tails[left] = num;

            // 如果 num 比当前所有结尾都大，说明最长长度可以 +1
            if (left == size) {
                size++;
            }
        }

        return size;
    }
}
```

## 为什么这个贪心是对的

这个算法最容易在“我会写，但我讲不清”上失分。面试里最好这么解释。

### 核心不变量

对于任意长度 `k`：

-   `tails[k - 1]` 始终表示长度为 `k` 的递增子序列中，**最小可能的结尾值**
    

如果我们让一个长度相同的子序列结尾更小，那么它只会更优秀，不会更差。

因为：

-   长度没变
    
-   结尾更小
    
-   后面可以接上的数只会更多
    

所以当一个新的 `num` 来了，如果它能把某个长度的结尾替换得更小，那一定值得替换。

---

### 为什么替换不会丢失最优答案

假设某个位置原来是 `x`，现在被更小或相等的 `num` 替换：

-   对当前长度来说，仍然存在一个长度不变的递增子序列
    
-   而且结尾更小
    
-   未来扩展能力更强
    

因此替换不会影响最终最长长度，反而更有利。

---

### 为什么 `tails` 本身不一定是真实 LIS

例如：

```text
nums = [0, 8, 4, 12, 2]
```

处理中间过程可能得到：

```text
tails = [0, 2, 12]
```

这个 `tails` 恰好是一个递增数组，但它并不保证对应原数组中的真实一条完整子序列构造路径。

它只是记录“每种长度下最优的结尾值”。

所以这题如果只求长度，这个方法非常好； 如果还要输出具体序列，就需要额外记录前驱信息。

---

## 复杂度分析

-   时间复杂度：`O(n log n)`
    
    -   每个元素做一次二分查找
        
-   空间复杂度：`O(n)`
    

---

## 两种解法对比

| 解法  | 时间复杂度 | 空间复杂度 | 优点  | 缺点  |
| --- | --- | --- | --- | --- |
| 动态规划 | `O(n^2)` | `O(n)` | 易理解、易推导 | 数据大时较慢 |
| 贪心 + 二分 | `O(n log n)` | `O(n)` | 最优复杂度，面试高频 | 正确性不容易讲清 |

## 面试中推荐的回答顺序

建议按这个节奏讲：

### 第一层：先给出 DP

这样面试官会看到你不是只会背最优模板，而是真的理解题目。

可这样表达：

> 定义 `dp[i]` 为以 `nums[i]` 结尾的 LIS 长度，枚举 `j < i`，若 `nums[j] < nums[i]`，则 `dp[i] = max(dp[i], dp[j] + 1)`。

### 第二层：再优化到 `O(n log n)`

然后自然过渡：

> 上述做法时间复杂度是 `O(n^2)`，如果数据更大，可以维护一个 `tails` 数组，用贪心保留每种长度下最小的结尾，再配合二分更新，复杂度优化到 `O(n log n)`。

### 第三层：解释“为什么替换是合理的”

这一层往往决定面试表现是否优秀。

## 扩展问题 1：如果题目要求“最长非递减子序列”怎么办？

这时条件从“严格递增”变成“可以相等”。

### DP 改法

把：

```java
nums[j] < nums[i]
```

改成：

```java
nums[j] <= nums[i]
```

### 二分改法

原题是严格递增，所以找的是第一个 `>= num` 的位置。

如果是非递减，则应找第一个 `> num` 的位置，也就是 **upper\_bound**。

这个区别非常重要，是面试高频追问。

---

## 扩展问题 2：如果不仅要求长度，还要求输出具体 LIS 序列

这时单纯的 `tails` 数组不够，需要额外维护：

1.  `prev[i]`
    
    -   记录 `nums[i]` 在 LIS 中的前驱下标
        
2.  `pos[len]`
    
    -   记录长度为 `len` 的递增子序列当前最优结尾元素的下标
        

这样在最终找到最长长度后，可以沿着 `prev` 反向回溯出完整序列。

这属于 LIS 的进阶版本，很多公司会在“求长度”的基础上继续追问。

---

## 扩展问题 3：这题和“最长连续递增子序列”有什么区别？

这也是很容易被问到的点。

### 本题：最长递增子序列

-   可以不连续
    
-   典型做法：DP / 贪心 + 二分
    

### 最长连续递增子序列

-   必须连续
    
-   通常一次遍历即可
    
-   思维难度明显更低
    

所以一看到“子序列”，就要先警觉：**不要求连续**。

## 边界情况

### 1\. 只有一个元素

```text
nums = [5]
```

答案是 `1`

### 2\. 全部相同

```text
nums = [2,2,2,2]
```

因为要求严格递增，答案是 `1`

### 3\. 严格递减

```text

```

答案也是 `1`

### 4\. 已经严格递增

```text
nums = [1,2,3,4,5]
```

答案是 `5`

## 面试和实际工作中的建议

### 1\. 面试里不要一上来就只背 `tails`

如果你直接写出 `O(n log n)` 模板，但解释不清为什么是对的，面试官通常会继续追问，容易暴露“会写不会讲”。

更稳妥的方式是：

-   先讲 DP，证明你理解定义和转移
    
-   再讲贪心 + 二分，体现优化能力
    

这比单纯背模板表现更好。

---

### 2\. 要特别注意“严格递增”和“非严格递增”的二分边界差异

很多人模板一背就错：

-   严格递增：找第一个 `>= num`
    
-   非递减：找第一个 `> num`
    

这个区别在面试里很常见。

---

### 3\. 在业务开发中，这题本身未必直接出现，但思想非常常用

LIS 的核心价值不只在这道题，而在于它背后的模式：

-   动态规划建模
    
-   贪心正确性证明
    
-   二分查找优化
    
-   “维护最优状态表示，而不是维护真实路径”
    

这些思想会迁移到很多真实问题中，例如：

-   日志或事件流中的最优趋势分析
    
-   排序后转化为 LIS 的二维问题
    
-   调度、链式关系、版本演进问题
    

---

### 4\. 如果线上代码需要可维护性，优先选择更容易解释的版本

在真实工作里，未必所有场景都必须上 `O(n log n)`。

如果：

-   数据量不大
    
-   团队成员需要快速理解
    
-   可维护性优先
    

那么 `O(n^2)` DP 版本可能更合适。

如果：

-   数据量大
    
-   性能敏感
    
-   这是核心路径
    

那么再选择贪心 + 二分版本。

这类“根据场景做权衡”的表达，在面试里也会加分。

## 最后给出一个推荐记忆模板

### DP 版记忆点

-   `dp[i]`：以 `i` 结尾
    
-   初始化全 1
    
-   枚举 `j < i`
    
-   如果 `nums[j] < nums[i]`，就尝试转移
    
-   答案取全局最大
    

### 贪心 + 二分版记忆点

-   `tails[k]`：长度为 `k + 1` 的递增子序列最小结尾
    
-   对每个 `num`
    
    -   找第一个 `>= num` 的位置
        
    -   替换它
        
    -   如果没找到，就追加
        
-   最终 `size` 就是答案

## 推荐提交版本

如果是在力扣直接提交，推荐使用最优解版本：

```java
import java.util.Arrays;

class Solution {
    public int lengthOfLIS(int[] nums) {
        int[] tails = new int[nums.length];
        int size = 0;

        for (int num : nums) {
            int left = 0;
            int right = size;

            // 二分查找第一个 >= num 的位置
            while (left < right) {
                int mid = left + (right - left) / 2;
                if (tails[mid] < num) {
                    left = mid + 1;
                } else {
                    right = mid;
                }
            }

            // 用更小的结尾更新对应长度的最优状态
            tails[left] = num;

            // 如果插入位置在当前末尾，说明 LIS 长度加 1
            if (left == size) {
                size++;
            }
        }

        return size;
    }
}
```

## 一句话总结

这题本质上是：

-   用 **DP** 求“以每个位置结尾的最优解”
    
-   再进一步用 **贪心 + 二分** 压缩状态，只保留“每种长度下最小的结尾”，从而把时间复杂度从 `O(n^2)` 优化到 `O(n log n)`。
