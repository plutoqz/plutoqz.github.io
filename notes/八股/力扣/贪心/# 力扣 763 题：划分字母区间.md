# 力扣 763 题：划分字母区间

## 题意与约束

给定字符串 `s`，将其划分为尽可能多的片段，使得同一字母最多出现在一个片段中，且按顺序拼接仍为原字符串；返回每个片段的长度列表。[1][2][3]<br>`1 <= s.length <= 500`，且仅由小写字母组成。[2][3]

---

## 解法一（次优）：字符区间 + 合并区间

### 思路

-   对每个字符统计 **首次出现位置** 与 **最后出现位置**，形成区间 `[first, last]`。
    
-   将所有字符区间按起点排序并合并重叠区间。
    
-   每个合并后的区间就是一个合法片段，其长度为 `end - start + 1`。<br>这种做法等价于先把“必须在同一段内的字符范围”合并，再得到最终分段。
    

### 复杂度

-   时间：`O(n + k log k)`，`k <= 26`（字符种类数）。
    
-   空间：`O(k)`。
    

### 适用场景

-   需要复用“区间合并”思路的场景；
    
-   字符集不固定（如更大字符范围），想先抽象成区间再处理。
    

### Java 实现

```java
import java.util.*;

class Solution {
    public List<Integer> partitionLabels(String s) {
        List<Integer> res = new ArrayList<>();
        if (s == null || s.length() == 0) return res; // 空串保护

        int n = s.length();
        int[] first = new int[26];
        int[] last = new int[26];
        Arrays.fill(first, -1);
        Arrays.fill(last, -1);

        for (int i = 0; i < n; i++) {
            int idx = s.charAt(i) - 'a';
            if (first[idx] == -1) first[idx] = i; // 记录首次出现
            last[idx] = i; // 记录最后出现
        }

        List<int[]> intervals = new ArrayList<>();
        for (int i = 0; i < 26; i++) {
            if (first[i] != -1) {
                intervals.add(new int[]{first[i], last[i]}); // 生成区间
            }
        }

        intervals.sort(Comparator.comparingInt(a -> a[0])); // 按起点排序
        int curStart = intervals.get(0)[0];
        int curEnd = intervals.get(0)[1];

        for (int i = 1; i < intervals.size(); i++) {
            int[] in = intervals.get(i);
            if (in[0] <= curEnd) {
                curEnd = Math.max(curEnd, in[1]); // 合并重叠区间
            } else {
                res.add(curEnd - curStart + 1); // 记录片段长度
                curStart = in[0];
                curEnd = in[1];
            }
        }
        res.add(curEnd - curStart + 1); // 加入最后一段
        return res;
    }
}
```

## 解法二（最优）：贪心一次扫描

### 思路

-   先记录每个字符的 **最后出现位置**。[1][2][3]
    
-   从左到右扫描，用 `end` 表示当前片段必须覆盖到的最远位置：<br>`end = max(end, last[当前字符])`。
    
-   当扫描索引 `i == end`，说明当前片段内所有字符的最后出现位置都被覆盖，可以切分。[1][2][3]
    

### 复杂度

-   时间：`O(n)`（两次线性扫描）。
    
-   空间：`O(1)`（固定 26 个字符）。
    

### 适用场景

-   面试首选、性能最优；
    
-   字符集较小且固定（如小写字母）。
    

### Java 实现

```java
import java.util.*;

class Solution {
    public List<Integer> partitionLabels(String s) {
        List<Integer> res = new ArrayList<>();
        int[] last = new int[26];

        for (int i = 0; i < s.length(); i++) {
            last[s.charAt(i) - 'a'] = i; // 记录每个字母最后出现位置
        }

        int start = 0, end = 0;
        for (int i = 0; i < s.length(); i++) {
            end = Math.max(end, last[s.charAt(i) - 'a']); // 更新当前段最远边界
            if (i == end) { // 到达边界即可切分
                res.add(end - start + 1);
                start = end + 1; // 开启下一段
            }
        }
        return res;
    }
}
```

---

## 细节技巧

-   `end` 必须用 `max` 更新，否则可能提前切分导致同字符跨段。
    
-   若字符集不固定，可用 `HashMap<Character, Integer>` 代替数组。
    
-   片段长度统一用 `end - start + 1`，避免 off-by-one。
    

---

## 面试/工作建议

-   面试中应强调不变量：**当前片段必须覆盖已出现字符的最后位置**，因此在 `i == end` 时切分是安全且最早的切分，能保证片段数最大。[1][2][3]
    
-   真实工程中如果字符集扩大（如 Unicode），选择 `HashMap`；若固定为小写字母，数组实现最稳。
    
-   建议口头快速验证示例：
    
    -   `ababcbacadefegdehijhklij` → `[9,7,8]`。[1][2][3]
        
    -   `eccbbbbdec` → `[10]`。[1][2][3]
