# 力扣 207 题：课程表

## 算法思路

本题的本质是判断一个**有向图是否存在环**。如果图中存在环，意味着存在类似“A依赖B，B依赖A”的死循环，必然无法完成所有课程。解决此类有向图依赖或判环问题的标准解法是**拓扑排序**。

我们可以通过两种主流方式来实现拓扑排序与判环：

1.  **深度优先搜索（DFS）**：通过记录节点的访问状态，如果在同一条搜索路径上遇到了处于“访问中”的节点，说明存在环。
    
2.  **广度优先搜索（BFS / Kahn算法）**：通过统计节点的“入度”（即前置依赖的数量）。每次仅将入度为0的节点（无依赖或依赖已满足的课程）加入队列并消除它的出边。若最终能够处理的节点数等于总课程数，说明无环。
    

---

## 解法一：深度优先搜索（DFS）

基于DFS的解法主要利用一个状态数组来标记每个节点。递归深度等于最长的依赖链长度。

### 适用场景

适用于图的边比较稀疏，且对代码简洁度要求较高的场景。缺点是在图深度极大时可能面临栈溢出（Stack Overflow）风险。

### 代码实现

```java
import java.util.List;
import java.util.ArrayList;

class Solution {
    public boolean canFinish(int numCourses, int[][] prerequisites) {
        // 构建邻接表：index代表课程，list存放依赖该课程的后续课程
        List<List<Integer>> adjacency = new ArrayList<>();
        for (int i = 0; i < numCourses; i++) {
            adjacency.add(new ArrayList<>());
        }
        for (int[] cp : prerequisites) {
            adjacency.get(cp[1]).add(cp[0]);
        }
        
        // 状态数组 flags
        // 0: 未被访问
        // 1: 正在当前DFS路径中被访问（遇到1说明发现了环）
        // 2: 已经访问完毕且该节点及其所有后续节点都不存在环
        int[] flags = new int[numCourses];
        
        // 遍历所有节点，确保非连通图也能全覆盖
        for (int i = 0; i < numCourses; i++) {
            if (!dfs(adjacency, flags, i)) {
                return false; // 一旦发现环，直接返回false
            }
        }
        return true;
    }

    private boolean dfs(List<List<Integer>> adjacency, int[] flags, int i) {
        if (flags[i] == 1) return false; // 遇到当前访问路径中的节点，说明有环
        if (flags[i] == 2) return true;  // 遇到已知安全的节点，直接返回
        
        // 标记为当前DFS路径访问中
        flags[i] = 1; 
        
        // 递归访问所有后续课程
        for (Integer j : adjacency.get(i)) {
            if (!dfs(adjacency, flags, j)) {
                return false;
            }
        }
        
        // 当前节点及其所有后续节点均安全，将状态置为2
        flags[i] = 2; 
        return true;
    }
}
```

## 解法二：广度优先搜索（Kahn 算法 / 最优解）

基于BFS的拓扑排序（Kahn算法）是一种更加符合常理的直观思路：“从不需要前置条件的课开始学，学完一门，就解锁下一门，看最后能不能全学完”。

### 适用场景

生产环境和工程应用中最常使用的图算法之一。迭代的实现方式避免了递归深度的问题，同时可以轻松地改造成“多线程并发处理无依赖任务”的模型。

### 代码实现

```java
import java.util.List;
import java.util.ArrayList;
import java.util.Queue;
import java.util.LinkedList;

class Solution {
    public boolean canFinish(int numCourses, int[][] prerequisites) {
        // indegrees 记录每个课程的入度（需要多少门前置课程）
        int[] indegrees = new int[numCourses];
        // 构建邻接表
        List<List<Integer>> adjacency = new ArrayList<>();
        for (int i = 0; i < numCourses; i++) {
            adjacency.add(new ArrayList<>());
        }
        
        // 填充入度表和邻接表
        for (int[] cp : prerequisites) {
            indegrees[cp[0]]++; // 课程 cp[0] 的前置条件+1
            adjacency.get(cp[1]).add(cp[0]); // 课程 cp[1] 能够解锁 cp[0]
        }
        
        Queue<Integer> queue = new LinkedList<>();
        // 寻找所有的起点：将所有入度为 0 的课程（无前置条件）入队
        for (int i = 0; i < numCourses; i++) {
            if (indegrees[i] == 0) {
                queue.offer(i);
            }
        }
        
        int count = 0; // 记录已学习的课程数
        
        // BFS 进行拓扑排序
        while (!queue.isEmpty()) {
            int pre = queue.poll();
            count++; // 每出队一个说明学完一门课
            
            // 遍历刚刚学完的课能解锁的后续课程
            for (int cur : adjacency.get(pre)) {
                indegrees[cur]--; // 解锁一个前置条件
                // 如果该后续课程的所有前置条件都已满足，则入队
                if (indegrees[cur] == 0) {
                    queue.offer(cur);
                }
            }
        }
        
        // 比较完成的课程数与总课程数
        return count == numCourses;
    }
}
```

## 实际工作与面试场景下的建议

1.  **面试表达技巧**：
    
    -   看到“前置依赖”、“任务调度”的字眼，必须条件反射出\*\*“拓扑排序”**与**“有向图”\*\*。
        
    -   面试时优先手写 **BFS (Kahn算法)**。不仅因为它不容易引发爆栈，而且它的“入度出度”概念非常容易通过白板画图向面试官解释清楚，代码逻辑更清晰。
        
    -   主动提到时间复杂度与空间复杂度均为 $O(V+E)$（$V$ 为节点数，$E$ 为边数）。
        
2.  **实际工程应用**：
    
    -   **依赖打包器**：比如前端的 Webpack / Vite，后端的 Maven / Gradle 在解决包依赖树时，底层逻辑完全就是该题的扩展。
        
    -   **任务流调度（DAG）**：如 Airflow、大数据平台的工作流引擎。当多个批处理任务存在上下游依赖时，系统会剥离出所有入度为 0 的任务，丢进线程池**并发执行**。当某任务完成，通知其下游任务`indegree - 1`，归零后自动触发执行。BFS的逻辑能够天然匹配这种高并发的任务调度场景。 [1] [[LeetCode in Python] 207 (M) course schedule 课程表](https://www.cnblogs.com/journeyonmyway/p/12702667.html)
