# 力扣 994 题：腐烂的橘子

## 题目描述

在给定的 `m x n` 网格 `grid` 中，每个单元格可以有以下三个值之一：

-   值 `0` 代表空单元格
    
-   值 `1` 代表新鲜橘子
    
-   值 `2` 代表腐烂的橘子
    

每分钟，腐烂的橘子周围 4 个方向上相邻的新鲜橘子都会腐烂。返回直到单元格中没有新鲜橘子为止所必须经过的最小分钟数。如果不可能，返回 `-1` [1][2]。

**示例 1：**

```text
输入：[[2,1,1],[1,1,0],[0,1,1]]
输出：4
```

**示例 2：**

```text
输入：[[2,1,1],[0,1,1],[1,0,1]]
输出：-1
解释：左下角的橘子永远不会腐烂
```

**示例 3：**

```text
输入：[[0,2]]
输出：0
解释：初始就没有新鲜橘子
```

## 解法一：DFS（不推荐，仅用于理解）

### 思路分析

DFS 可以遍历所有可达的新鲜橘子，但**无法保证得到最短时间**。因为 DFS 是深度优先，可能绕远路，不适合求最小分钟数的问题 [4]。

### 代码实现

```java
import java.util.HashMap;
import java.util.Map;

class Solution {
    private int[][] grid;
    private Map<String, Integer> memo; // 记录每个位置被腐烂的最短时间
    
    public int orangesRotting(int[][] grid) {
        this.grid = grid;
        this.memo = new HashMap<>();
        int rows = grid.length;
        int cols = grid[0].length;
        int maxTime = 0;
        
        // 遍历所有新鲜橘子，计算被腐烂的最短时间
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                if (grid[i][j] == 1) {
                    int time = dfs(i, j, 0);
                    if (time == Integer.MAX_VALUE) {
                        return -1; // 有橘子无法被腐烂
                    }
                    maxTime = Math.max(maxTime, time);
                }
            }
        }
        return maxTime;
    }
    
    private int dfs(int row, int col, int time) {
        // 边界检查
        if (row < 0 || row >= grid.length || col < 0 || col >= grid[0].length) {
            return Integer.MAX_VALUE;
        }
        // 遇到空单元格
        if (grid[row][col] == 0) {
            return Integer.MAX_VALUE;
        }
        // 遇到腐烂橘子，返回当前时间
        if (grid[row][col] == 2) {
            return time;
        }
        
        String key = row + "," + col;
        // 避免循环访问（需要更复杂的visited处理，此处简化）
        if (memo.containsKey(key) && memo.get(key) <= time) {
            return memo.get(key);
        }
        
        // 标记为已访问（临时）
        int original = grid[row][col];
        grid[row][col] = 3; // 临时标记
        
        // 向四个方向搜索
        int minTime = Integer.MAX_VALUE;
        minTime = Math.min(minTime, dfs(row - 1, col, time + 1));
        minTime = Math.min(minTime, dfs(row + 1, col, time + 1));
        minTime = Math.min(minTime, dfs(row, col - 1, time + 1));
        minTime = Math.min(minTime, dfs(row, col + 1, time + 1));
        
        grid[row][col] = original; // 恢复
        memo.put(key, minTime);
        return minTime;
    }
}
```

### 适用场景

-   ❌ **不推荐用于此题**：无法保证最短路径
    
-   ✅ 仅适用于需要遍历所有可能性的场景
    

### 复杂度分析

-   时间复杂度：O(4^(m×n))，存在大量重复计算
    
-   空间复杂度：O(m×n)，递归栈深度

## 解法二：多源 BFS（最优解法）⭐

### 思路分析

这是典型的**多源 BFS** 问题 [2][4]：

1.  **初始化**：遍历网格，统计新鲜橘子数量，将所有腐烂橘子坐标加入队列
    
2.  **层序遍历**：每一轮处理当前队列中的所有腐烂橘子（代表同一分钟）
    
3.  **扩散腐烂**：每个腐烂橘子使其上下左右相邻的新鲜橘子腐烂，新腐烂的橘子加入队列
    
4.  **结果判断**：BFS 结束后，若还有新鲜橘子剩余返回 `-1`，否则返回分钟数
    

**为什么 BFS 是最优的？**

-   BFS 按层遍历，先搜索到的结点一定是距离最近的 [1][4]
    
-   所有初始腐烂橘子同时开始扩散，确保分钟计数准确 [2]
    
-   时间复杂度与网格大小成正比 [2]
    

### 代码实现

```java
import java.util.ArrayDeque;
import java.util.Queue;

class Solution {
    public int orangesRotting(int[][] grid) {
        int rows = grid.length;                      // 获取行数
        int cols = grid[0].length;                   // 获取列数
        int minutes = 0;                             // 记录经过的分钟数
        int freshCount = 0;                          // 统计新鲜橘子数量
        
        Queue<int[]> queue = new ArrayDeque<>();     // BFS队列，存储腐烂橘子坐标
        
        // 方向数组：上、下、左、右
        int[][] directions = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};
        
        // 第一步：遍历网格，统计新鲜橘子，将所有腐烂橘子入队
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                if (grid[i][j] == 1) {
                    freshCount++;                    // 统计新鲜橘子
                } else if (grid[i][j] == 2) {
                    queue.offer(new int[]{i, j});    // 腐烂橘子入队
                }
            }
        }
        
        // 特殊情况：没有新鲜橘子，直接返回0
        if (freshCount == 0) {
            return 0;
        }
        
        // 第二步：多源BFS，模拟腐烂扩散过程
        while (freshCount > 0 && !queue.isEmpty()) {
            int levelSize = queue.size();            // 当前层的橘子数量（同一分钟）
            
            // 处理当前层的所有腐烂橘子
            for (int i = 0; i < levelSize; i++) {
                int[] cur = queue.poll();            // 取出当前腐烂橘子
                int curRow = cur[0];
                int curCol = cur[1];
                
                // 向四个方向扩散
                for (int[] dir : directions) {
                    int newRow = curRow + dir[0];    // 新位置行坐标
                    int newCol = curCol + dir[1];    // 新位置列坐标
                    
                    // 检查边界和是否为新鲜橘子
                    if (newRow >= 0 && newRow < rows && 
                        newCol >= 0 && newCol < cols && 
                        grid[newRow][newCol] == 1) {
                        
                        grid[newRow][newCol] = 2;    // 标记为腐烂
                        freshCount--;                // 新鲜橘子减一
                        queue.offer(new int[]{newRow, newCol}); // 新腐烂橘子入队
                    }
                }
            }
            
            minutes++;                               // 完成一轮，分钟数加一
        }
        
        // 第三步：判断是否所有橘子都腐烂
        return freshCount == 0 ? minutes : -1;
    }
}
```

### 适用场景

-   ✅ **最短路径/最小步数问题**
    
-   ✅ **分层扩散类问题**（如病毒传播、洪水填充）
    
-   ✅ **多源起点同时扩散**
    

### 复杂度分析

-   时间复杂度：O(m×n)，每个单元格最多被访问一次 [2]
    
-   空间复杂度：O(m×n)，最坏情况下队列存储所有腐烂橘子 [2]

## 关键细节与技巧

### 1\. 层序遍历的正确写法

```java
int levelSize = queue.size();    // 必须先记录当前层大小
for (int i = 0; i < levelSize; i++) {
    // 处理当前层元素
}
minutes++;                       // 层处理完后分钟数加一
```

**注意**：不能在 for 循环中直接用 `queue.size()`，因为队列大小在循环中会变化 [1][5]。

### 2\. 方向数组简化代码

```java
int[][] directions = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};
for (int[] dir : directions) {
    int newRow = curRow + dir[0];
    int newCol = curCol + dir[1];
}
```

避免写 4 个重复的 if 判断，代码更简洁 [2][3]。

### 3\. 提前终止优化

```java
while (freshCount > 0 && !queue.isEmpty())
```

当新鲜橘子数为 0 时提前结束 BFS，无需继续遍历 [2]。

### 4\. 边界情况处理

-   初始没有新鲜橘子：返回 0
    
-   有新鲜橘子但无法被腐烂：返回 -1
    
-   网格只有 1 行或 1 列：BFS 依然适用

## 面试与实战建议

### 面试场景

1.  **先说思路**：明确这是多源 BFS 问题，解释为什么不用 DFS [4]
    
2.  **强调关键点**：层序遍历、新鲜橘子计数、多源同时扩散
    
3.  **主动提优化**：方向数组、提前终止、边界处理
    
4.  **复杂度分析**：必须给出时间和空间复杂度
    

### 实际工作场景

1.  **类似应用场景**：
    
    -   病毒/信息传播模拟
        
    -   洪水填充算法
        
    -   游戏地图扩散效果
        
    -   网络故障传播分析
        
2.  **代码可复用性**：
    
    -   将方向数组提取为常量
        
    -   将边界检查封装为独立方法
        
    -   考虑使用不可变坐标类替代 int[]
        
3.  **扩展思考**：
    
    -   如果网格很大（1000×1000），考虑使用并行 BFS
        
    -   如果需要记录每个橘子的腐烂时间，可用 HashMap 存储
        
    -   如果有多个独立区域，可先做连通性分析
        

### 常见错误

1.  ❌ 忘记统计初始新鲜橘子数量
    
2.  ❌ 层序遍历时直接用 `queue.size()` 作为循环条件
    
3.  ❌ 忘记检查边界条件
    
4.  ❌ 没有处理初始就没有新鲜橘子的情况
    
5.  ❌ 分钟数计算错误（最后一轮没有新橘子腐烂也加了 1）
