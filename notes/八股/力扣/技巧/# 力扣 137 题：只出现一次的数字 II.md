# 力扣 137 题：只出现一次的数字 II

## 题目核心

给定一个整数数组 `nums`，除某个元素只出现 **1 次** 外，其余每个元素都恰好出现 **3 次**，找出那个只出现一次的元素。

这题的关键在于：

-   普通计数很好做；
    
-   但题目的进阶要求是：
    
    -   时间复杂度 `O(n)`
        
    -   额外空间复杂度 `O(1)`
        

下面按 **“容易想到但不满足进阶 → 满足进阶 → 最优技巧”** 的顺序来讲。

---

## 解法一：排序

### 思路

排序后，相同元素会排在一起。

由于除了一个数只出现 1 次，其他数都出现 3 次，所以排完序以后，正常情况应该是：

-   `x x x`
    
-   `y y y`
    
-   `z z z`
    

唯一出现 1 次的数字会打破这个三连块结构。

可以每次从下标 `i` 开始检查：

-   如果 `nums[i] != nums[i + 1]`，说明 `nums[i]` 就是答案；
    
-   否则这个数是一个三连块的开头，可以直接 `i += 3` 跳过去。
    

### 适用场景

-   先快速写出正确解；
    
-   面试中先给一个基础方案；
    
-   允许修改原数组。
    

### 复杂度

-   时间复杂度：`O(n log n)`
    
-   空间复杂度：Java 的 `Arrays.sort(int[])` 递归栈通常为 `O(log n)`
    

### 代码

```java
import java.util.Arrays;

class Solution {
    public int singleNumber(int[] nums) {
        Arrays.sort(nums); // 排序后，相同元素会聚集在一起

        int n = nums.length;
        if (n == 1) {
            return nums[0]; // 只有一个元素时，直接返回
        }

        for (int i = 0; i < n - 1; i += 3) {
            if (nums[i] != nums[i + 1]) {
                return nums[i]; // 当前元素没有和后一个组成重复块，说明它只出现一次
            }
        }

        return nums[n - 1]; // 如果前面都正常，答案只能在最后
    }
}
```

## 解法二：HashMap 计数

### 思路

用 `HashMap<Integer, Integer>` 统计每个数字出现的次数。

最后遍历 `map`，找到计数为 `1` 的那个数字即可。

这是最直观、最稳定的写法。

### 适用场景

-   面试中先给出正确版本；
    
-   业务代码里需要顺手统计频次；
    
-   不要求 `O(1)` 额外空间。
    

### 复杂度

-   时间复杂度：`O(n)`
    
-   空间复杂度：`O(n)`
    

### 代码

```java
import java.util.HashMap;
import java.util.Map;

class Solution {
    public int singleNumber(int[] nums) {
        Map<Integer, Integer> countMap = new HashMap<>();

        for (int num : nums) {
            countMap.put(num, countMap.getOrDefault(num, 0) + 1); // 统计每个数字出现次数
        }

        for (Map.Entry<Integer, Integer> entry : countMap.entrySet()) {
            if (entry.getValue() == 1) {
                return entry.getKey(); // 找到只出现一次的数字
            }
        }

        return -1; // 题目保证有解，理论上不会走到这里
    }
}
```

## 解法三：按位统计

### 思路

这是这题最经典、也最推荐在面试里讲的解法。

核心思想是：

-   一个 `int` 有 32 位；
    
-   对于每一位，统计数组里这一位上 `1` 出现了多少次；
    
-   因为其他数字都出现 3 次，所以这些重复数字对某一位的贡献一定是 **3 的倍数**；
    
-   对 `3` 取模后，剩下的就是那个只出现一次的数字在这一位上的值。
    

### 举例

假设某一位上：

-   重复数字贡献了 `3k` 个 `1`
    
-   唯一数字如果这一位是 `1`，总数就变成 `3k + 1`
    
-   `count % 3 == 1`，说明答案这一位是 `1`
    

### 为什么负数也成立

负数在 Java 里也是按 **32 位二进制补码** 存储的。<br>所以我们逐位统计时，负数同样适用。

注意：

-   一定要遍历 **32 位**，不能只遍历 31 位；
    
-   最高位是符号位，也要一起统计。
    

### 适用场景

-   面试最推荐；
    
-   满足题目进阶要求；
    
-   可读性比状态机更好。
    

### 复杂度

-   时间复杂度：`O(32n)`，也就是 `O(n)`
    
-   空间复杂度：`O(1)`
    

### 代码

```java
class Solution {
    public int singleNumber(int[] nums) {
        int ans = 0;

        for (int i = 0; i < 32; i++) { // int 一共 32 位
            int bitCount = 0;

            for (int num : nums) {
                bitCount += (num >>> i) & 1; // 统计第 i 位上有多少个 1
            }

            if (bitCount % 3 != 0) {
                ans |= (1 << i); // 这一位属于只出现一次的数字
            }
        }

        return ans;
    }
}
```

### 关键细节

1.  `>>>` 是无符号右移，取位时更直观。
    
2.  `1 << 31` 在 Java 中是合法的，结果是负数，但这正好可以正确还原负数答案。
    
3.  这类“其他元素出现 k 次，只有一个元素出现 1 次”的题，按位统计非常容易推广。

## 解法四：状态机 / 位运算压缩（综合最优）

### 思路

这是比“按位统计”更进一步的写法，本质上是把“每一位出现次数对 3 取模”的过程，压缩成两个整数状态。

定义：

-   `ones`：记录当前出现次数 `% 3 == 1` 的那些位
    
-   `twos`：记录当前出现次数 `% 3 == 2` 的那些位
    

对某一位来说，状态变化如下：

-   `00`：出现次数 `% 3 == 0`
    
-   `01`：出现次数 `% 3 == 1`
    
-   `10`：出现次数 `% 3 == 2`
    

当当前位再来一个 `1` 时，状态循环：

-   `00 -> 01`
    
-   `01 -> 10`
    
-   `10 -> 00`
    

把这个状态转移并行地作用到 32 位上，就得到下面两句：

```java
ones = (ones ^ num) & ~twos;
twos = (twos ^ num) & ~ones;
```

最后剩在 `ones` 里的，就是只出现一次的数字。

### 为什么最后返回 `ones`

因为：

-   出现 3 次的那些位，最终会回到 `00`
    
-   只出现 1 次的那些位，最终会停留在 `01`
    

也就是只会保存在 `ones` 中。

### 适用场景

-   面试中需要展示位运算功底；
    
-   追求最优常数；
    
-   熟悉状态压缩思想后，这个写法非常漂亮。
    

### 复杂度

-   时间复杂度：`O(n)`
    
-   空间复杂度：`O(1)`
    

### 代码

```java
class Solution {
    public int singleNumber(int[] nums) {
        int ones = 0;
        int twos = 0;

        for (int num : nums) {
            ones = (ones ^ num) & ~twos; // 记录出现 1 次的位
            twos = (twos ^ num) & ~ones; // 记录出现 2 次的位，注意这里用的是更新后的 ones
        }

        return ones; // 最终只出现一次的位保存在 ones 中
    }
}
```

### 这一写法最容易错的地方

更新顺序不能随便换。

必须先更新 `ones`，再用更新后的 `ones` 更新 `twos`。<br>如果顺序写反，状态转移就乱了。

## 几种解法对比

| 解法  | 时间复杂度 | 空间复杂度 | 是否满足进阶 | 可读性 | 面试表现 |
| --- | --- | --- | --- | --- | --- |
| 排序  | `O(n log n)` | `O(log n)` | 否   | 高   | 一般  |
| HashMap | `O(n)` | `O(n)` | 否   | 很高  | 基础分 |
| 按位统计 | `O(n)` | `O(1)` | 是   | 高   | 很好  |
| 状态机 | `O(n)` | `O(1)` | 是   | 中   | 最亮眼 |

## 常见错误

### 1\. 只统计 31 位，漏掉符号位

如果数组里有负数，答案可能错误。<br>必须统计完整的 32 位。

### 2\. 看到位运算就直接写异或

这题和“只出现一次的数字 I”不同。

-   那题是“其他数字都出现 2 次”，异或能直接消掉；
    
-   这题是“其他数字都出现 3 次”，单纯异或不行。
    

### 3\. 状态机写法更新顺序错

这题最常见的 bug 之一。

### 4\. 排序解法忽略“会修改原数组”

如果原数组后续还要使用，排序可能不合适。

## 面试建议

### 1\. 最稳妥的回答顺序

建议这样答：

1.  先说 `HashMap`，快速说明你能立刻得到正确解；
    
2.  主动补一句：题目进阶要求 `O(1)` 空间；
    
3.  给出“按位统计”作为主解；
    
4.  如果面试官继续追问位运算能力，再讲“状态机”。
    

这个顺序很自然，也最容易拿到完整分。

### 2\. 面试里最推荐主讲哪一个

首推 **按位统计**：

-   满足进阶要求；
    
-   不难证明；
    
-   比状态机更容易让面试官听懂；
    
-   代码也不容易写错。
    

### 3\. 什么时候讲状态机更合适

如果面试偏底层、偏位运算、偏算法竞赛风格，可以主动补充状态机写法。<br>这会显得你不仅会做题，而且理解得更深。

## 实际工作场景建议

1.  **业务代码优先可读性**
    
    -   如果不是极致性能场景，按位统计通常比状态机更适合团队协作。
        
    -   状态机虽然短，但不熟悉的人维护成本更高。
        
2.  **需要推广到“其他数字出现 k 次”**
    
    -   按位统计法最好扩展。
        
    -   只需要把 `3` 改成 `k`，再根据余数判断目标数字的每一位。
        
3.  **底层/高性能场景**
    
    -   状态机写法常数更优，单次扫描，适合追求性能的场景。

## 延伸：这题真正要掌握的能力

这题不是单纯记一个模板，而是掌握下面两个核心思想：

### 思想一：按位独立处理

很多整数题都可以把一个 `int` 拆成 32 个二进制位分别处理。<br>一旦你接受“每一位互不影响”这个视角，很多位运算题就会变简单。

### 思想二：出现次数取模

-   出现 2 次：常见做法是异或
    
-   出现 3 次：常见做法是按位计数 `% 3`
    
-   出现 k 次：继续推广成 `% k`
    

本题的状态机解法，本质也是在做“位上的计数取模 3”，只是把 32 次位枚举压缩掉了。
