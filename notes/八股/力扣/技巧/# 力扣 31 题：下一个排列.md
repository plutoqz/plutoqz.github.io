
# 力扣 31 题：下一个排列

## 题意

给你一个整数数组 `nums`，需要**原地**把它改成“字典序中的下一个更大排列”。

如果当前排列已经是最大排列，比如 `[3,2,1]`，那么就改成最小排列 `[1,2,3]`。

### 例如
- `[1,2,3] -> [1,3,2]`
- `[3,2,1] -> [1,2,3]`
- `[1,1,5] -> [1,5,1]`

题目的核心要求有两个：

1. **必须原地修改**
2. **额外空间尽量是常数级**

---

## 核心规律

这题本质是在求：**当前排列的下一个字典序排列**。

### 什么叫“下一个”？

比如所有排列按字典序排好：

- `123`
- `132`
- `213`
- `231`
- `312`
- `321`

那么 `132` 的下一个就是 `213`。

### 怎么找到这个“刚刚大一点点”的排列？

要想让结果比当前大，但又不能大太多，思路一定是：

1. **尽量不动左边高位**
2. **从右边开始找可以变大的位置**
3. **只把这个位置增大一点点**
4. **后面的部分变成最小的样子**

这就是这题的本质。

---

## 解法一：交换后对后缀排序（次优，容易想到）

### 思路

先从右往左找第一个满足：

```text
nums[i] < nums[i + 1]
```

的位置 `i`，这个位置叫“拐点”或者“枢轴”。

为什么要找它？

因为从右往左看，如果一段是单调不升的，比如：

```text
5 4 4 3 1
```

那这段已经是这个后缀的**最大排列**了，想在这段内部找更大的排列是不可能的。  
所以必须往左找到第一个可以“增加”的位置。

找到 `i` 之后：

1. 在右侧找一个 **比 `nums[i]` 大的最小值**
2. 交换这两个数
3. 把 `i+1` 到末尾这一段重新排成升序，这样整体才是“刚刚大一点点”

---

### 代码实现

```java
import java.util.Arrays;

class Solution {
    public void nextPermutation(int[] nums) {
        int n = nums.length;
        int i = n - 2;

        // 1. 从右往左找到第一个下降点：nums[i] < nums[i + 1]
        while (i >= 0 && nums[i] >= nums[i + 1]) {
            i--;
        }

        // 2. 如果没找到，说明整个数组已经是降序，直接变成最小排列
        if (i < 0) {
            Arrays.sort(nums); // 次优解里直接整体排序
            return;
        }

        int j = n - 1;

        // 3. 从右往左找到第一个大于 nums[i] 的元素
        //    由于后缀原本是降序，这样找到的就是“刚好大一点”的数
        while (j > i && nums[j] <= nums[i]) {
            j--;
        }

        // 4. 交换枢轴和目标元素
        swap(nums, i, j);

        // 5. 将后缀排成升序，保证整体是下一个最小更大排列
        //    注意：Arrays.sort 的右边界是开区间
        Arrays.sort(nums, i + 1, n);
    }

    private void swap(int[] nums, int i, int j) {
        int temp = nums[i];
        nums[i] = nums[j];
        nums[j] = temp;
    }
}
```

---

### 复杂度分析

- 时间复杂度：`O(n log n)`
  - 找拐点：`O(n)`
  - 找交换位置：`O(n)`
  - 排序后缀：`O(n log n)`
- 空间复杂度：
  - 如果严格按题目要求看，`Arrays.sort` 对原始类型虽然很省空间，但**面试里通常不认为这是严格常数空间解法**
  - 所以这不是最优答案

---

- 你刚开始理解这题
- 想先写一个容易想到、容易验证正确性的版本
- 面试时如果一开始没想到最优解，可以先说出这个思路，再继续优化

## 解法二：交换后直接反转后缀（最优，面试标准解）

这才是这题真正的经典解法。

---

### 关键结论

当我们从右往左找到第一个 `nums[i] < nums[i+1]` 时，  
`i+1` 到末尾这一段一定是**单调不升**的。

例如：

```text
1 3 5 4 2
    ^ ^
```

这里 `i = 1`，因为 `3 < 5`。  
而右边 `5 4 2` 明显是降序的。

---

### 为什么交换后可以直接“反转”，不用排序？

因为：

1. 原后缀本来就是降序
2. 我们从右往左找第一个比 `nums[i]` 大的数去交换
3. 交换之后，后缀依然保持降序结构
4. 要让后缀变成最小，只需要把它**反转成升序**即可

所以不需要排序，直接双指针反转就行。

这一步把时间复杂度从 `O(n log n)` 优化到了 `O(n)`。

---

### 最优解完整思路

#### 第 1 步：从右往左找第一个下降点 `i`
找到第一个满足：

```text
nums[i] < nums[i + 1]
```

的位置。

#### 第 2 步：如果找不到
说明整个数组都是降序，比如 `[5,4,3,2,1]`，当前已经是最大排列。  
此时直接把整个数组反转成升序即可。

#### 第 3 步：从右往左找第一个大于 `nums[i]` 的数 `j`
因为右侧后缀是降序，所以从右往左第一个大于 `nums[i]` 的数，就是**刚好比它大一点的数**。

#### 第 4 步：交换 `nums[i]` 和 `nums[j]`

#### 第 5 步：反转 `i+1` 到末尾
让后缀变成最小升序排列。

---

### 代码实现

```java
class Solution {
    public void nextPermutation(int[] nums) {
        int n = nums.length;
        int i = n - 2;

        // 1. 从右往左找到第一个下降点
        while (i >= 0 && nums[i] >= nums[i + 1]) {
            i--;
        }

        // 2. 如果找到了下降点，就在右边找刚好比它大的数
        if (i >= 0) {
            int j = n - 1;

            // 从右往左找第一个大于 nums[i] 的元素
            while (j > i && nums[j] <= nums[i]) {
                j--;
            }

            // 3. 交换，让前缀变大一点点
            swap(nums, i, j);
        }

        // 4. 将后缀反转成升序
        //    如果 i == -1，相当于反转整个数组
        reverse(nums, i + 1, n - 1);
    }

    private void swap(int[] nums, int i, int j) {
        int temp = nums[i];
        nums[i] = nums[j];
        nums[j] = temp;
    }

    private void reverse(int[] nums, int left, int right) {
        while (left < right) {
            swap(nums, left, right);
            left++;
            right--;
        }
    }
}
```

---

### 复杂度分析

- 时间复杂度：`O(n)`
  - 找拐点一次
  - 找交换位置一次
  - 反转一次
- 空间复杂度：`O(1)`

这就是题目要求下的最优解。

## 手动推演一遍

以 `nums = [1,3,5,4,2]` 为例。

### 第一步：找下降点
从右往左看：

- `4 > 2`
- `5 > 4`
- `3 < 5`，找到了下降点 `i = 1`

所以：
- 枢轴是 `3`
- 后缀是 `[5,4,2]`

### 第二步：从右往左找第一个大于 3 的数
从右往左看：

- `2` 不行
- `4` 可以

所以 `j = 3`

### 第三步：交换
交换 `3` 和 `4`

得到：

```text
[1,4,5,3,2]
```

### 第四步：反转后缀
把 `[5,3,2]` 反转成升序 `[2,3,5]`

最终结果：

```text
[1,4,2,3,5]
```

这就是原排列的下一个字典序排列。

---

## 为什么这个算法一定正确

可以从“最小增量”角度理解。

### 1. 为什么要从右往左找第一个下降点？

因为右边那段已经是降序，说明它已经是“这个后缀能形成的最大排列”。  
如果你只在这段后缀内部折腾，不可能得到更大的整体排列。  
所以必须改动更左边的位置。

而从右往左找到的第一个可改位置，意味着：

- 左边前缀尽量不变
- 改动位置尽量靠右

这保证了“变大得最少”。

---

### 2. 为什么要找“右边刚好比它大的数”？

如果你换一个太大的数，结果会跳过很多排列。  
为了得到“下一个”，只能让当前位置增大得尽量少。

而由于后缀是降序，从右往左第一个比它大的数，就是“最小的大数”。

---

### 3. 为什么交换后要让后缀变成升序？

前缀已经确定变大了，接下来为了让整个数组仍然尽可能小，  
后缀必须取最小排列，也就是升序。

---

## 易错点与细节技巧

### 1. 比较符号很容易写错

找下降点时必须写：

```java
while (i >= 0 && nums[i] >= nums[i + 1])
```

不是 `>`，而是 `>=`。  
因为相等的时候也属于“非递增”，仍然要继续往左找。

---

### 2. 找交换位置时必须找“严格大于”

```java
while (nums[j] <= nums[i])
```

必须是 `<=`。  
因为目标是找到**第一个严格大于**枢轴的元素。

---

### 3. `i == -1` 是非常重要的边界情况

比如 `[3,2,1]`：

- 从右往左找不到下降点
- 说明已经是最大排列
- 应该直接反转整个数组，得到最小排列 `[1,2,3]`

---

### 4. 不要把“反转后缀”和“排序后缀”混淆

- 排序后缀：也能做，但不是最优
- 反转后缀：利用了“后缀本来就是降序”这个性质，所以更快

---

### 5. 这题对重复元素完全适用

比如 `[1,1,5]`：

- 找到下降点 `1`
- 找到右侧第一个大于它的 `5`
- 交换后得到 `[1,5,1]`

重复元素不会破坏算法逻辑。

## 面试中建议怎么讲

如果这是面试题，建议你按下面这个顺序说：

### 第一层：先说核心规律
“下一个排列就是在保证整体变大的前提下，让增量尽可能小。  
所以应该从右往左找第一个还能增大的位置。”

### 第二层：再说步骤
1. 从右往左找第一个 `nums[i] < nums[i+1]`
2. 再从右往左找第一个 `nums[j] > nums[i]`
3. 交换 `i` 和 `j`
4. 把 `i+1` 到末尾反转

### 第三层：补一句关键证明
“因为 `i+1` 到末尾原本是降序，所以交换后把后缀反转成升序，就能得到最小的更大排列。”

这样回答会显得你不仅会写，而且真正理解了字典序本质。

## 实际工作场景建议

### 1. 如果你要不断生成下一个排列
这个函数非常适合做排列枚举：

- 先把数组排序成最小排列
- 每次调用一次 `nextPermutation`
- 就能按字典序遍历所有排列

这在一些搜索、组合测试、穷举校验中很常见。

---

### 2. 不要去生成“所有排列再排序”
虽然能做，但复杂度是阶乘级，实际工程里几乎不可用。  
这题的价值就在于：**只求下一个，不求所有**，所以可以做到线性时间。

---

### 3. Java 中如果换成对象数组或集合，要注意排序开销
如果你写的是 `Integer[]`、`List<Integer>`，再调用对象排序，  
空间和时间行为会和原始类型数组不同。  
这在面试里如果题目强调“常数空间”，最好还是用 `int[] + reverse` 的写法。

---

## 一句话记忆模板

这题可以浓缩成一句话：

**从右往左找第一个下降点，交换成右边刚好更大的数，再把后缀反转成最小升序。**

这句话基本就是整题的完整解法。
