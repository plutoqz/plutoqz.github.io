# 力扣 98：验证二叉搜索树

给你一棵二叉树，判断它是否是合法的二叉搜索树（BST）。

BST 的定义不是“左孩子比根小、右孩子比根大”这么局部，而是：

-   左子树 **所有节点** 都 `< 根节点`
    
-   右子树 **所有节点** 都 `> 根节点`
    
-   左右子树本身也都必须是 BST
    
-   题目中通常要求 **严格递增**，所以 **不允许重复值**
    

---

## 容易踩坑的点

很多人会写出这种错误逻辑：

```java
if (root.left != null && root.left.val >= root.val) return false;
if (root.right != null && root.right.val <= root.val) return false;
```

这只能检查当前节点和左右孩子的关系，**不能检查整棵子树的范围约束**。

例如：

```text
    5
   / \
  1   6
     / \
    3   7
```

这里 `3 < 5`，但它出现在 `5` 的右子树里，所以整棵树不是 BST。<br>只看父子关系会误判为 true。

---

# 解法一：中序遍历存入数组后判断是否严格递增

## 思路

BST 的一个重要性质：

> 对 BST 做中序遍历，得到的序列一定是严格递增的。

所以可以：

1.  对整棵树做中序遍历
    
2.  把结果放进 `List<Integer>`
    
3.  再遍历这个数组，判断是否严格递增
    

这是最容易想到、最好写的方案，但用了额外 `O(n)` 空间。

## 适用场景

-   刚做这题，先写对
    
-   面试时先快速给出一个正确思路
    
-   对 BST 和中序遍历关系还不够熟时
    

## 代码实现

```java
import java.util.ArrayList;
import java.util.List;

class Solution {
    public boolean isValidBST(TreeNode root) {
        List<Integer> inorderList = new ArrayList<>();
        inorder(root, inorderList);

        // 中序结果必须严格递增
        for (int i = 1; i < inorderList.size(); i++) {
            if (inorderList.get(i) <= inorderList.get(i - 1)) {
                return false;
            }
        }
        return true;
    }

    private void inorder(TreeNode node, List<Integer> inorderList) {
        if (node == null) {
            return;
        }

        inorder(node.left, inorderList);   // 先遍历左子树
        inorderList.add(node.val);         // 记录当前节点
        inorder(node.right, inorderList);  // 再遍历右子树
    }
}
```

## 复杂度分析

-   时间复杂度：`O(n)`
    
-   空间复杂度：`O(n)`<br>主要是中序结果数组占用的空间

# 解法二：中序遍历过程中直接比较前驱节点

## 思路

既然 BST 中序遍历后一定严格递增，那就没必要把所有值都存下来。

可以在中序遍历过程中维护一个“前一个访问到的节点值”：

-   当前值必须 `> 前一个值`
    
-   否则就不是 BST
    

这样就把解法一的额外数组优化掉了。

这是面试中很常见的写法。

## 适用场景

-   已经知道 BST 的中序性质
    
-   想在空间上优于“存数组”
    
-   面试里比较常见，写法不长
    

## 代码实现

```java
import java.util.*;

class Solution {
    private Long prev = null;  // 记录中序遍历的前一个值，使用 Long 避免边界问题

    public boolean isValidBST(TreeNode root) {
        return inorderCheck(root);
    }

    private boolean inorderCheck(TreeNode node) {
        if (node == null) {
            return true;
        }

        // 先检查左子树
        if (!inorderCheck(node.left)) {
            return false;
        }

        // 当前值必须严格大于前一个中序值
        if (prev != null && node.val <= prev) {
            return false;
        }

        // 更新前驱值
        prev = (long) node.val;

        // 再检查右子树
        return inorderCheck(node.right);
    }
}
```

## 复杂度分析

-   时间复杂度：`O(n)`
    
-   空间复杂度：`O(h)`<br>`h` 是树高，递归栈最坏 `O(n)`，平衡树时约 `O(log n)`
    

## 细节技巧

-   `prev` 最好用 `Long`，避免和 `Integer.MIN_VALUE` 之类边界值冲突
    
-   判断必须是 `<=`，因为 BST 要求严格递增，**重复值不合法**

# 解法三：递归传上下界

## 思路

这是最符合 BST 定义本身的做法，也是我最推荐的做法。

对每个节点，都有一个合法范围：

-   它必须 `>` 下界 `lower`
    
-   它必须 `<` 上界 `upper`
    

然后递归地更新范围：

-   左子树：范围变成 `(lower, 当前节点值)`
    
-   右子树：范围变成 `(当前节点值, upper)`
    

这样可以直接解决“右子树某个节点虽然比父节点小，但比根节点还小”的问题。

### 为什么这是最优思路

因为它不是依赖“中序递增”这个结果去判断，而是**直接在递归过程中维护 BST 的定义**。

面试里如果你能主动说出“局部比较不够，必须传递祖先范围约束”，通常会比较加分。

## 适用场景

-   面试推荐写法
    
-   对 BST 定义理解更深入
    
-   处理“全局范围约束”最直观
    

## 代码实现

```java
import java.util.*;

class Solution {
    public boolean isValidBST(TreeNode root) {
        // 使用 long 作为边界，避免 Integer.MIN_VALUE / MAX_VALUE 的边界问题
        return dfs(root, Long.MIN_VALUE, Long.MAX_VALUE);
    }

    private boolean dfs(TreeNode node, long lower, long upper) {
        if (node == null) {
            return true;
        }

        // 当前节点值必须严格位于 (lower, upper) 之间
        if (node.val <= lower || node.val >= upper) {
            return false;
        }

        // 左子树所有节点都必须 < node.val
        if (!dfs(node.left, lower, node.val)) {
            return false;
        }

        // 右子树所有节点都必须 > node.val
        if (!dfs(node.right, node.val, upper)) {
            return false;
        }

        return true;
    }
}
```

## 复杂度分析

-   时间复杂度：`O(n)`
    
-   空间复杂度：`O(h)`<br>递归栈深度为树高
    

---

# 三种解法对比

| 解法  | 时间复杂度 | 空间复杂度 | 特点  |
| --- | --- | --- | --- |
| 中序遍历 + 数组 | `O(n)` | `O(n)` | 最容易理解，适合入门 |
| 中序遍历 + 前驱比较 | `O(n)` | `O(h)` | 常见面试写法，空间更优 |
| 上下界递归 | `O(n)` | `O(h)` | 最贴近 BST 定义，最推荐 |

# 面试时怎么讲更好

## 1\. 先指出错误直觉

先说：

> 不能只比较当前节点和左右孩子，因为 BST 需要满足整棵子树的全局范围约束。

这句话能说明你不是在机械刷题，而是真的理解了问题。

---

## 2\. 再给两个正确方向

你可以这样组织表达：

### 方向一：利用中序遍历严格递增

-   BST 中序遍历后一定严格递增
    
-   可以存数组判断
    
-   也可以边遍历边比较前驱值，优化空间
    

### 方向二：直接维护合法范围

-   每个节点都要落在祖先传下来的 `(lower, upper)` 区间内
    
-   左子树更新上界，右子树更新下界
    
-   这是对 BST 定义的直接实现
    

---

## 3\. 推荐最终提交上下界写法

如果是正式面试，我更建议你最后落到“上下界递归”这版，因为它体现了：

-   你理解了 BST 的本质
    
-   你能处理全局约束
    
-   你能规避 `Integer` 边界问题

# 实际工作/面试建议

## 工作里

如果这是业务代码而不是算法题，优先考虑：

-   **可读性**
    
-   **边界稳定性**
    
-   **容易维护**
    

因此通常会偏向“上下界递归”这种表达更直接的写法。

---

## 面试里

建议答题顺序：

1.  先说局部比较会错
    
2.  给出中序递增性质
    
3.  再给出更本质的“上下界递归”
    
4.  主动补充：
    
    -   重复值不允许，所以用严格比较
        
    -   用 `long` 处理边界更安全
        

这个回答结构会比较完整。

# 最推荐提交版本

```java
import java.util.*;

class Solution {
    public boolean isValidBST(TreeNode root) {
        return dfs(root, Long.MIN_VALUE, Long.MAX_VALUE);
    }

    private boolean dfs(TreeNode node, long lower, long upper) {
        if (node == null) {
            return true;
        }

        if (node.val <= lower || node.val >= upper) {
            return false;
        }

        return dfs(node.left, lower, node.val) 
            && dfs(node.right, node.val, upper);
    }
}
```

# 补充：如果面试官追问“非递归怎么写”

可以用栈做中序遍历，维护前驱值。

```java
import java.util.ArrayDeque;
import java.util.Deque;

class Solution {
    public boolean isValidBST(TreeNode root) {
        Deque<TreeNode> stack = new ArrayDeque<>();
        TreeNode curr = root;
        Long prev = null;

        while (curr != null || !stack.isEmpty()) {
            // 一路向左入栈
            while (curr != null) {
                stack.push(curr);
                curr = curr.left;
            }

            curr = stack.pop();

            // 中序遍历时，当前值必须严格大于前一个值
            if (prev != null && curr.val <= prev) {
                return false;
            }
            prev = (long) curr.val;

            // 转向右子树
            curr = curr.right;
        }

        return true;
    }
}
```

这个写法适合面试官继续追问“递归改迭代”。
