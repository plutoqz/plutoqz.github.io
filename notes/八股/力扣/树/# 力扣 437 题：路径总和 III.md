# 力扣 437 题：路径总和 III

## 题意概述

给定二叉树和目标和 `targetSum`，统计从任意节点出发、向下（父到子方向）的一条路径，其路径节点值之和等于目标和的路径数量。

---

## 解法一（次优）：以每个节点为起点的 DFS 枚举

**适用场景**：节点数较小或需要易懂实现时。<br>**时间复杂度**：`O(n^2)` 最坏，树退化为链表。<br>**空间复杂度**：`O(h)` 递归栈。

**思路**

-   以每个节点作为路径起点，向下 DFS，统计以该节点为起点的路径和。
    
-   对每个节点重复一遍。
    

```java
import java.util.*;

class Solution {
    public int pathSum(TreeNode root, int targetSum) {
        if (root == null) return 0;
        // 以当前节点为起点的路径数 + 左子树 + 右子树
        return countFrom(root, targetSum) 
            + pathSum(root.left, targetSum) 
            + pathSum(root.right, targetSum);
    }

    private int countFrom(TreeNode node, long target) {
        if (node == null) return 0;
        int count = 0;
        if (node.val == target) count++; // 当前路径满足
        // 继续向下累加
        count += countFrom(node.left, target - node.val);
        count += countFrom(node.right, target - node.val);
        return count;
    }
}
```

## 解法二（最优）：前缀和 + 回溯

**适用场景**：大规模数据、面试首选。<br>**时间复杂度**：`O(n)`<br>**空间复杂度**：`O(h)` 递归栈 + 哈希表

**思路**

-   维护从根到当前节点路径和 `currSum`。
    
-   若存在某个前缀和 `currSum - targetSum`，说明从该前缀之后到当前节点的路径和为 `targetSum`。
    
-   使用 `HashMap` 统计前缀和出现次数。
    
-   递归时入栈计数，回溯时出栈减计数。
    

```java
import java.util.*;

class Solution {
    public int pathSum(TreeNode root, int targetSum) {
        Map<Long, Integer> prefixCount = new HashMap<>();
        prefixCount.put(0L, 1); // 空路径前缀和
        return dfs(root, 0L, targetSum, prefixCount);
    }

    private int dfs(TreeNode node, long currSum, int targetSum, Map<Long, Integer> prefixCount) {
        if (node == null) return 0;

        currSum += node.val; // 更新当前前缀和
        int res = prefixCount.getOrDefault(currSum - targetSum, 0); // 关键计数

        // 记录当前前缀和
        prefixCount.put(currSum, prefixCount.getOrDefault(currSum, 0) + 1);

        // 继续遍历子树
        res += dfs(node.left, currSum, targetSum, prefixCount);
        res += dfs(node.right, currSum, targetSum, prefixCount);

        // 回溯：移除当前前缀和影响
        prefixCount.put(currSum, prefixCount.get(currSum) - 1);

        return res;
    }
}
```

---

## 关键细节与技巧

-   使用 `long` 防止路径和溢出（节点值范围可能较大）。
    
-   前缀和哈希表必须在回溯时减计数，否则会污染兄弟路径。
    
-   `prefixCount.put(0,1)` 是处理从根开始的路径关键。
    

---

## 实际工作/面试建议

-   面试中优先讲解前缀和+回溯，强调“从根到当前”的前缀计数思想。
    
-   若面试官担心空间，说明哈希表大小与树高相关（受路径长度限制）。
    
-   实战中注意 `long` 溢出与回溯清理，这是常见 bug 点。
