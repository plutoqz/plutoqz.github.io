## 199\. 二叉树的右视图

### 题意

给你一棵二叉树的根节点 `root`，返回从右侧看到的节点值，按从上到下的顺序返回。

### 核心本质

这题本质上就是：

-   **每一层只取 1 个节点**
    
-   这个节点是这一层 **最右边** 的节点
    

所以无论你用：

-   **层序遍历 BFS**
    
-   **深度优先 DFS**
    

都可以做出来。

下面按**空间利用 + 工程稳定性综合**，从次优到最优排序。

---

## 解法一：层序遍历 BFS（次优，最直观，面试第一反应）

### 思路

按层遍历二叉树，每一层遍历结束时，记录这一层最后一个节点的值即可。

如果一层内节点按 **从左到右** 的顺序出队，那么这一层**最后出队**的节点就是右视图节点。

### 适用场景

-   面试里第一版答案，最好讲清楚
    
-   题目本质是“按层处理”时，BFS 非常自然
    
-   后续如果题目要求“每层最大值 / 每层平均值 / 锯齿层序遍历”，BFS 很容易扩展
    

### 复杂度

-   时间复杂度：`O(n)`
    
-   空间复杂度：`O(w)`<br>`w` 是二叉树的最大层宽
    

### 代码

> 下面代码是**独立可提交版本**，LeetCode 已内置 `TreeNode` 定义。

```java
import java.util.ArrayDeque;
import java.util.ArrayList;
import java.util.List;
import java.util.Queue;

class Solution {
    public List<Integer> rightSideView(TreeNode root) {
        List<Integer> ans = new ArrayList<>();
        if (root == null) {
            return ans; // 空树直接返回空列表
        }

        Queue<TreeNode> queue = new ArrayDeque<>();
        queue.offer(root);

        while (!queue.isEmpty()) {
            int size = queue.size(); // 固定当前层节点数，避免和下一层混在一起

            for (int i = 0; i < size; i++) {
                TreeNode node = queue.poll();

                if (i == size - 1) {
                    ans.add(node.val); // 当前层最后一个节点，就是右视图节点
                }

                if (node.left != null) {
                    queue.offer(node.left); // 先左后右入队
                }
                if (node.right != null) {
                    queue.offer(node.right);
                }
            }
        }

        return ans;
    }
}
```

## 解法二：右优先递归 DFS（更优，代码更短）

### 思路

右视图的关键是：**每一层最先被看到的节点**。

如果我们按：

1.  当前节点
    
2.  右子树
    
3.  左子树
    

的顺序去递归，那么在某个深度上，**第一次到达该深度**的节点，一定就是这一层从右侧能看到的节点。

### 为什么成立

因为你总是先走右边，所以该深度最先访问到的节点，就是这一层最靠右的可见节点。

### 适用场景

-   面试里展示你对“深度”视角的理解
    
-   树高度不大时，递归写法非常简洁
    
-   想用更少的辅助结构时，递归 DFS 很合适
    

### 复杂度

-   时间复杂度：`O(n)`
    
-   空间复杂度：`O(h)`<br>`h` 是树高，递归栈深度
    

### 代码

```java
import java.util.ArrayList;
import java.util.List;

class Solution {
    public List<Integer> rightSideView(TreeNode root) {
        List<Integer> ans = new ArrayList<>();
        dfs(root, 0, ans);
        return ans;
    }

    private void dfs(TreeNode node, int depth, List<Integer> ans) {
        if (node == null) {
            return; // 递归终止条件
        }

        if (depth == ans.size()) {
            ans.add(node.val); // 该深度第一次到达，说明它就是右视图节点
        }

        dfs(node.right, depth + 1, ans); // 先递归右子树
        dfs(node.left, depth + 1, ans);  // 再递归左子树
    }
}
```

### 关键技巧

-   `depth == ans.size()` 表示：这一层还没放过答案
    
-   必须 **先右后左**
    
-   如果你写成先左后右，这段逻辑得到的就更像“左视图”

## 解法三：右优先迭代 DFS（最优，工程更推荐）

### 思路

核心思想和递归 DFS 一样：

-   仍然是“先访问右边”
    
-   仍然是“每层第一次访问到的节点就是答案”
    

不同点在于：<br>不用系统递归栈，而是手动维护一个栈，避免深树情况下的 `StackOverflowError`。

### 为什么通常更推荐

-   保留了 DFS 常见的较好空间表现：`O(h)`
    
-   避免 Java 递归过深导致栈溢出
    
-   工程上更稳
    

### 适用场景

-   树深度未知
    
-   真实业务代码更关注稳定性
    
-   面试中想进一步体现“递归可改迭代”的能力
    

### 复杂度

-   时间复杂度：`O(n)`
    
-   空间复杂度：`O(h)`
    

### 代码

```java
import java.util.ArrayDeque;
import java.util.ArrayList;
import java.util.Deque;
import java.util.List;

class Solution {
    private static class NodeState {
        TreeNode node;
        int depth;

        NodeState(TreeNode node, int depth) {
            this.node = node;
            this.depth = depth;
        }
    }

    public List<Integer> rightSideView(TreeNode root) {
        List<Integer> ans = new ArrayList<>();
        if (root == null) {
            return ans; // 空树直接返回
        }

        Deque<NodeState> stack = new ArrayDeque<>();
        stack.push(new NodeState(root, 0));

        while (!stack.isEmpty()) {
            NodeState cur = stack.pop();
            TreeNode node = cur.node;
            int depth = cur.depth;

            if (depth == ans.size()) {
                ans.add(node.val); // 该层第一次访问到的节点，就是右视图节点
            }

            if (node.left != null) {
                stack.push(new NodeState(node.left, depth + 1)); // 先压左
            }
            if (node.right != null) {
                stack.push(new NodeState(node.right, depth + 1)); // 后压右，保证右子树先弹出
            }
        }

        return ans;
    }
}
```

### 关键技巧

因为栈是 **后进先出**：

-   想让 **右子树先处理**
    
-   就要 **先压左，再压右**
    

这是迭代 DFS 最容易写反的地方。

## 三种解法对比

| 解法  | 时间复杂度 | 空间复杂度 | 优点  | 缺点  | 适合场景 |
| --- | --- | --- | --- | --- | --- |
| BFS 层序遍历 | O(n) | O(w) | 最直观，最好讲 | 宽树时队列可能较大 | 面试第一版、按层处理 |
| 递归 DFS | O(n) | O(h) | 代码短，思路漂亮 | 深树可能栈溢出 | 面试进阶、树不深 |
| 迭代 DFS | O(n) | O(h) | 工程稳定，避免递归栈风险 | 代码稍复杂 | 工程推荐、深度未知 |

## 容易出错的点

### 1\. 把“右视图”误解成一路走右孩子

这是错的。

例如：

```text
    1
   / \
  2   3
   \
    5
```

右视图是 `[1, 3, 5]`，因为第三层虽然没有更右的节点，但 `5` 仍然可见。

---

### 2\. BFS 没有提前记录当前层大小

错误写法会让下一层节点混进当前层，导致层边界错乱。

正确做法是：

```java
int size = queue.size();
for (int i = 0; i < size; i++) {
    ...
}
```

---

### 3\. DFS 遍历顺序写反

如果你用了：

```java
if (depth == ans.size()) ans.add(node.val);
```

那必须是：

```java
先右后左
```

否则得到的不是右视图。

---

### 4\. Java 里优先用 `ArrayDeque`

-   `Queue`/`Deque` 场景下，优先用 `ArrayDeque`
    
-   通常比 `LinkedList` 更轻量

## 面试表达建议

### 如果是面试，推荐这样答

1.  先说本质：**每层取最右节点**
    
2.  第一版给出 **BFS**
    
    -   最稳
        
    -   最容易验证正确性
        
3.  然后主动补充 **右优先 DFS**
    
    -   展示你对“层”和“深度”两种视角都理解
        
4.  最后再补一句：
    
    -   Java 递归过深可能栈溢出
        
    -   工程里更倾向 **迭代 DFS** 或 **BFS**
        

这样会显得你不是只会“刷题模板”，而是知道不同写法的取舍。

## 工作场景建议

### 1\. 如果系统里树深度不可控

优先：

-   **BFS**
    
-   **迭代 DFS**
    

少用递归 DFS，避免极端数据触发 `StackOverflowError`。

### 2\. 如果后续还要统计“每层信息”

比如：

-   每层节点数
    
-   每层最大值
    
-   每层平均值
    
-   层序输出
    

直接选 **BFS**，因为模型一致，代码更容易维护。

### 3\. 如果只关心“路径 / 深度 / 某层首次出现”

优先考虑 **DFS** 因为这类问题天然更适合“深度”视角。

---

## 一句话总结

这题最核心的认知是：

-   **BFS：每层取最后一个**
    
-   **右优先 DFS：每层取第一个到达的**
    

如果只考虑面试通过率，先写 **BFS**；<br>如果考虑工程稳定性和深树安全，优先 **迭代 DFS**。
