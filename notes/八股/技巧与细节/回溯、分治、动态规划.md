## 一、回溯算法 (Backtracking)

### 📌 基础模板

回溯算法本质是**多叉树的遍历问题**，核心框架如下 [1]：

```python
def backtrack(路径, 选择列表):
    if 满足结束条件:
        结果集.add(路径)
        return
    
    for 选择 in 选择列表:
        # 做选择
        路径.add(选择)
        选择列表.remove(选择)
        
        # 递归
        backtrack(路径, 选择列表)
        
        # 撤销选择（回溯）
        路径.remove(选择)
        选择列表.add(选择)
```

**三个关键要素** [1]：

| 要素  | 含义  |
| --- | --- |
| **路径** | 已经做出的选择 |
| **选择列表** | 当前可以做的选择 |
| **结束条件** | 到达决策树底层，无法再做选择 |

### 🔁 常见变式

| 变式类型 | 特点  | 典型题目 |
| --- | --- | --- |
| **子集树** | 每个元素选/不选，2ⁿ 个叶子节点 [2] | 子集问题、0-1背包 |
| **排列树** | n! 个叶子节点，元素顺序重要 [2] | 全排列、N皇后 |
| **带剪枝** | 提前判断无效路径，减少搜索 [2] | 数独、图着色 |
| **去重处理** | 处理重复元素，避免重复解 [1] | 含重复元素的排列组合 |

### 💡 典型例题

-   全排列 (LeetCode 46) [1]
    
-   N皇后问题 [2]
    
-   组合总和、子集问题 [7]

## 二、分治算法 (Divide and Conquer)

### 📌 基础模板

分治法核心思想是\*\*"分而治之"\*\*，三步流程 [2][6]：

```python
def divide_conquer(问题):
    # 1. 分解 (Divide)
    if 问题规模足够小:
        return 直接求解(问题)
    
    # 2. 解决 (Conquer)
    子问题列表 = 分解(问题)
    子结果列表 = []
    for 子问题 in 子问题列表:
        子结果列表.append(divide_conquer(子问题))
    
    # 3. 合并 (Merge)
    return 合并(子结果列表)
```

### 🔁 常见变式

| 变式类型 | 特点  | 典型题目 |
| --- | --- | --- |
| **归并排序** | 二分分解，有序合并 [2] | 排序、逆序对 |
| **快速排序** | 轴值划分，递归排序 [2] | 排序、第K大元素 |
| **棋盘覆盖** | 四等分递归 [2] | 特殊棋盘问题 |
| **最近点对** | 平面分治，合并时特殊处理 [2] | 几何问题 |
| **循环赛日程** | 迭代分治 [2] | 比赛安排 |

### 💡 典型例题

-   归并排序、快速排序 [2]
    
-   Pow(x, n) 快速幂
    
-   多数组第K小元素

## 三、动态规划 (Dynamic Programming)

### 📌 基础模板

动态规划需要满足**三个特征** [4][5]：

1.  **最优子结构**：问题的最优解包含子问题的最优解
    
2.  **无后效性**：当前状态不受未来状态影响
    
3.  **重叠子问题**：子问题会被重复计算
    

**两种实现方式** [4][5]：

#### 方式一：自顶向下（递归+备忘录）

```python
memo = {}
def dp(状态):
    if 状态 in memo:
        return memo[状态]
    if base_case:
        return 基础值
    
    结果 = 状态转移方程(状态)
    memo[状态] = 结果
    return 结果
```

#### 方式二：自底向上（迭代填表）

```python
# 1. 初始化 dp 数组
dp = [0] * (n + 1)
dp[base_case] = 基础值

# 2. 状态转移
for i in range(1, n + 1):
    dp[i] = 状态转移方程(dp[i-1], dp[i-2], ...)

# 3. 返回结果
return dp[n]
```

### 🔁 常见变式

| 变式类型 | 特点  | 典型题目 |
| --- | --- | --- |
| **线性DP** | 一维状态转移 [4] | 爬楼梯、最大子数组 |
| **二维DP** | 二维状态表 [2][4] | 最长公共子序列、编辑距离 |
| **背包问题** | 容量约束 [2][7] | 0-1背包、完全背包 |
| **区间DP** | 区间合并 [7] | 石子合并、戳气球 |
| **状态压缩DP** | 用位运算表示状态 [9] | 旅行商问题、铺砖问题 |
| **树形DP** | 树上状态转移 [7] | 树的最大独立集 |
| **数位DP** | 数字位处理 [7] | 统计特定数字 |

### 💡 典型例题

-   最小路径和 (LeetCode) [4]
    
-   硬币兑换 [4]
    
-   最长递增子序列 [7]
    
-   股票买卖问题 [7]

## 四、三种算法的对比与联系

| 维度  | 回溯  | 分治  | 动态规划 |
| --- | --- | --- | --- |
| **核心思想** | 枚举所有解，剪枝优化 [1] | 分而治之，合并结果 [2] | 状态转移，避免重复计算 [4] |
| **时间复杂度** | 通常指数级 O(n!) [1] | O(n log n) 居多 [2] | 多项式级 O(n²) 居多 [4] |
| **空间复杂度** | O(n) 递归栈 [1] | O(log n) 递归栈 [2] | O(n²) 状态表 [5] |
| **适用场景** | 求所有解、排列组合 [2] | 可独立分解的子问题 [6] | 最优解、重叠子问题 [5] |
| **是否重叠子问题** | ❌ 无 [1] | ❌ 通常无 [5] | ✅ 有 [4] |
| **实现方式** | 递归+回溯 [1] | 递归 [6] | 递归/迭代 [4] |

### 🔗 三者联系 [6]

```text
        递归
         │
    ┌────┼────┐
    │    │    │
  分治  回溯  动态规划
    │         │
    └────┬────┘
         │
    都可转化为迭代
```

-   **回溯 → 动态规划**：当回溯发现大量重复子问题时，可用 DP 优化 [4]
    
-   **分治 → 动态规划**：当分治的子问题重叠时，可转为 DP [5]
    
-   **共同点**：都可用递归实现，都涉及子问题分解 [6]
