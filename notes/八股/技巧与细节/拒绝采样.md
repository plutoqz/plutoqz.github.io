# 拒绝采样

---

# 一、核心问题与陷阱

-   你有：`rand5()`，返回 1..5 **均匀**。
    
-   你要：`rand7()`，返回 1..7 **均匀**。
    

**陷阱：不能直接** `rand5() % 7`<br>因为 5 不是 7 的倍数，取模会偏置。

---

# 二、标准解法：拒绝采样（Rejection Sampling）

## ✅ 思路

-   用 rand5 拼出更大的均匀空间
    
-   找到一个能被 7 整除的范围
    
-   超出范围的结果丢弃重来
    

## ✅ 具体构造

两次 rand5 生成 0..24（25 个状态）：

```text
x = rand5() - 1  // 0..4
y = rand5() - 1  // 0..4
v = x*5 + y      // 0..24（均匀）
```

25 个结果中，最大能整除 7 的部分是 0..20（21个），刚好 21 是 7 的倍数。

**算法：**

```pseudo
function rand7():
    while true:
        x = rand5() - 1
        y = rand5() - 1
        v = x*5 + y  // 0..24
        if v < 21:
            return (v % 7) + 1
```

## ✅ 正确性

-   `v` 在 0..24 均匀
    
-   0..20 每个数出现概率相等
    
-   0..20 映射到 1..7（每个值正好出现 3 次）
    
-   所以 1..7 均匀
    

## ✅ 期望调用次数

-   单次成功概率：21/25
    
-   每次用 2 次 rand5
    
-   期望调用次数：<br>[ 2 \* \\frac{1}{21/25} = 2 \* \\frac{25}{21} = \\frac{50}{21} \\approx 2.38 ]

# 三、思维升级：一般化方法（randA → randB）

## ✅ 通用模板（拒绝采样）

**目标：用 randA 得到 randB**

1.  选择 k，使 (A^k \\ge B)
    
2.  用 k 个 randA 拼出 0..A^k-1
    
3.  设 `m = floor(A^k / B) * B`
    
4.  若 `v < m`，返回 `v % B + 1`<br>否则丢弃重试
    

### 数学解释

-   `A^k` 个等概率结果
    
-   `m` 是最大的 `B` 的倍数
    
-   保证均匀分配
    

### 伪代码

```pseudo
function randB():
    while true:
        v = 0
        for i in 1..k:
            v = v*A + (randA()-1)
        if v < m:
            return (v % B) + 1
```

# 四、效率提升：利用“剩余信息（recycling）”

在标准拒绝采样中，被拒绝的那部分其实也均匀，可以“回收”再拼接，不必完全丢弃。

## 例子：rand5 → rand7 的回收版

第一轮：

-   0..24
    
-   0..20 接受
    
-   21..24（4 个数）回收
    

第二轮：

-   用 4 个数 × rand5 → 0..19（20 个）
    
-   取 0..13（14 个）可整除 7
    
-   余数继续回收…
    

这种“**逐步回收残差**”可以把期望调用次数进一步降低（接近理论下界）。<br>这是更高级的技巧，实际面试中非必须，但展示思考深度。

# 五、另一路：先造“公平硬币”，再用二进制

## 步骤

1.  用 rand5 造公平 coin：
    
    -   {1,2} → 0
        
    -   {4,5} → 1
        
    -   3 丢弃
        
2.  用 3 个 coin 生成 0..7
    
3.  若是 7 则重来
    

### 缺点

-   平均调用次数更高（约 4.3 次 rand5）
    
-   但逻辑清晰、便于推广

# 六、通用思维框架（建议牢记）

### ✅ 1. **扩大样本空间**

用多次 randA 生成 (A^k) 个等概率状态。

### ✅ 2. **匹配整除**

选一个能被 B 整除的最大范围，然后取模。

### ✅ 3. **拒绝采样保证无偏**

只要拒绝的是“超出范围”的那部分，就不会偏。

### ✅ 4. **考虑效率**

-   选择适当 k 减少拒绝概率
    
-   可回收剩余提高效率
    

### ✅ 5. **对比熵极限**

randA 每次提供 (\\log\_2 A) bits<br>目标需要 (\\log\_2 B) bits<br>理论下界：<br>[ \\text{最少期望次数} \\ge \\frac{\\log B}{\\log A} ]
